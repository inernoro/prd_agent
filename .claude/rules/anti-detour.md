# 少绕路原则（Anti-Detour）

> 用户是懒惰的、没准备好数据的。能一步做完的，绝不让他走多步、绝不逼他先去准备一个文件再回来。核心一个字：**少绕路**。

本规则补强 `zero-friction-input.md`（输入零摩擦）与 `quickstart-zero-friction.md`（启动零摩擦），把视角从"别让用户发呆"升级为"**别让用户为了喂数据而离开当前流程**"。

---

## 0. 一句话

> 任何需要用户提供数据的地方，默认假设：用户手头**没有**现成文件、**懒得**去准备、希望**就地**敲两下就完事。所以——**优先给一个可就地编辑的入口**（粘贴框 / 可编辑文本域 / 智能预填），上传文件只是补充而非唯一路径。

---

## 1. 三条硬规则

### 规则 A：能一步，不要两步

如果一件事可以在当前页面/弹窗内一次做完，就**不要**拆成"先去 X 页准备 → 回来 Y 页引用"。

- ❌ "请先在设置页上传 .env，再回到这里创建项目"
- ✅ 创建弹窗里**直接**有一个"粘贴环境变量"文本域，`KEY=VALUE` 一行一条，当场解析

### 规则 B：除非天然只能是文件，否则必须给"可就地编辑"入口

判定：**这份数据用户脑子里/剪贴板里能直接产出吗？**

| 数据 | 能否就地编辑 | 应提供 |
|---|---|---|
| 环境变量（KEY=VALUE） | 能（粘贴 .env 文本 / key-value 行） | **粘贴框 + key-value 行**，上传 .env 作为补充 |
| 启动命令 / Docker 镜像 | 能（一行命令） | 可编辑输入框 + 智能默认 |
| 初始化 SQL | 能（贴一段 SQL） | 文本域 |
| 数据库名 | 能（敲一个词） | 输入框 + 默认值 |
| 大型二进制 / 证书 / 压缩包 | **不能**（天然是文件） | 上传（此时上传是合理的唯一路径） |
| 整个代码仓库 | 不能就地敲 | Git URL / 仓库选择器（已是零摩擦） |

只有最后两类"天然是文件"才允许 upload-only。其余一律先给就地编辑入口。

### 规则 C：不全靠用户，也不全靠文件——智能预填兜底

"少绕路"不等于"把空白框丢给用户"。两头都要避免：

- 不要**纯空白手输**（违反零摩擦）
- 不要**强制上传文件**（违反少绕路）
- 正解：**智能默认/检测 + 可编辑**。能从仓库/上下文推断的（运行时、启动命令、端口、连接串）先填好，用户改的是"差异"而非"全部"

---

## 2. 智能默认的鲁棒性要求（防写死）

预填的默认值**必须考虑泛化**，不能只对一种技术栈成立：

- 默认命令/镜像若按"某个语言"写死（如只对 Node 合理的 `pnpm install && pnpm start`），换个栈就成噪音 → 必须**按所选运行时给对应默认**，且**首选 `auto` 检测**（读 `package.json` / `requirements.txt` / `go.mod` / `cds-compose.yml` 等真实信号）
- 默认值只是"起点"，**永远可被用户覆盖**（可编辑字段，不是只读展示）
- 检测不到时给"最常见"的保守默认 + 明确提示"这是猜的，请核对"，而不是给一个只对作者本人项目成立的命令
- 判定：**把这个默认值放到一个完全不同的项目里，它是"基本能用/明显要改"还是"驴唇不对马嘴"？** 后者就是写死、泛化差，必须改

---

## 3. 自查清单（设计任何"要用户给数据"的输入时）

- [ ] 这份数据用户能就地敲/粘贴吗？能 → 必须有可编辑入口，upload 只作补充
- [ ] 我有没有逼用户"先去别处准备文件再回来"？有 → 砍掉这一步，挪到当前流程里
- [ ] 我有没有给空白框？有 → 加智能默认/检测预填
- [ ] 我的默认值换个技术栈还成立吗？不成立 → 按运行时分流 + auto 检测
- [ ] 复杂项目（monorepo / 多服务 / 自定义构建 / 多个同类数据库）这条路走得通吗？走不通 → 补齐，别假设用户项目很简单

---

## 4. 与其他规则的关系

- `zero-friction-input.md`：管"别让用户面对空白发呆"（输入焦虑）
- `quickstart-zero-friction.md`：管"启动脚本大包大揽"（环境焦虑）
- **本规则**：管"别让用户为喂数据离开当前流程去准备文件"（绕路焦虑）
- `no-rootless-tree.md`：智能默认必须有根（真实检测），不能编造

三者共同目标：用户在任何一步都**不需要中断当前心流去别处准备东西**。

---

## 5. 历史背景

2026-06-02 用户验收 CDS 一键部署时连续指出：(1) 环境变量只能上传文件——"用户真的有环境变量文件吗？"；(2) Node 运行时默认命令写死、换个栈泛化差；(3) "能一步做完的不要走多步"。本规则把"少绕路"从隐性期待固化为显性约束。
