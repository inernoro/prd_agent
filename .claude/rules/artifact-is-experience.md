# 产物即体验（Artifact-is-the-Experience）

> 用户等待时，屏幕上必须是**产物本身在生长**，而不是"关于产物的状态描述"。
> 状态面板、进度条、spinner、诊断日志全都只配做配角；主视觉永远留给产物。

## 历史背景（本规则的由来，2026-06-10）

MD 转 PPT 一天内连续三次撞上同一堵墙，用户三次纠偏其实是同一句话：

1. **80 秒静止 spinner**：后端一直在推流式 delta，前端 `onDelta: () => {}` 原地丢弃，用户对着"正在精修中...已等待 80s"干等。
2. **主题换皮**：前端往 iframe 注入 `!important` CSS 覆盖层把 AI 的设计盖掉换色——用户纠偏："皮肤指的是 AI 参照去画动态的 HTML 页面，而不是换一个皮。"
3. **进度卡不等于产物**：第一版修复做了"第 N 页已生成"的进度卡面板，用户再次追问"其他平台是这样做的吗？为什么不朝他们的方式设计？"——Gamma/Kimi 的本质是**真实页面逐张渲染**，看的就是产物本身。最终改为：流式 HTML 每闭合一个 `<section>` 立即真实渲染进实况 iframe。

三次教训合成一个判定句：**用户在等的那个东西，能不能此刻就让他看见一部分？能就渲染它，不能才允许用状态文案过渡。**

## 强制规则

### 1. 等待期主视觉 = 产物的实时形态

- 生成类功能（PPT / 图片 / 报告 / 网页）：流式产物**解析到哪渲染到哪**（已闭合的页、已成形的段落、已出图的部分），主区域放产物，状态行放角落。
- 产物第一个可渲染片段出现之前：用**产物形状的骨架**（skeleton 模拟一页幻灯/一张图/一段文档）过渡，不用通用 spinner 居中。
- 流式原文（代码尾巴 / token 流）只能做次要信息，且产物片段一旦可渲染就退位。

### 2. 禁止"状态剧场"替代产物

以下都算状态剧场，单独出现即不合格：

- 居中 spinner + 按秒数轮换的猜测文案（"正在排版与收尾..."）
- 只有阶段步骤条 / 诊断日志（`[connection][profile][create]`裸标签）
- "第 N 页已生成"之类**关于**产物的卡片，而产物本身一帧都没показ

它们可以作为配角存在（角落状态行、产物未到达时的过渡），不可做主视觉。

### 3. 外观/风格类参数 = 生成参照，不是后处理

- 风格/主题/皮肤的唯一语义：**AI 生成时参照的设计语言**（提示词里的设计 token、字体、版式气质）。
- 禁止用前端 CSS 覆盖层"纠正/替换"AI 产物的外观（换皮）。切换风格 = 让 AI 按新风格重绘产物。

### 4. 等待中的固有开销要"藏进用户已有的动作里"

环境启动、会话创建这类藏不掉的物理耗时，挪到用户正在做别的事的时间窗里做（如：大纲确认期间预热 Agent 会话）。判定：用户主动操作的间隙里，系统是不是在闲着？

### 5. 架构命题冲突时，问用户，不替用户保守

当"工程稳妥"与用户的产品命题（如 agent-native、吃自己狗粮）冲突时，**禁止默默选稳妥路线**。要么按命题做，要么把取舍摆出来问。历史教训：用户说"默认使用 cds agent"被理解成"默认 MAP 直出 + Agent 实验"，错把稳妥当忠实。

## 交付前自审

- [ ] 等待期主视觉是产物（或产物形状的骨架），不是 spinner/状态卡？
- [ ] 产物的第一个可渲染片段到达后，立即渲染了它？
- [ ] 风格/外观参数走的是生成参照，没有任何后处理覆盖层？
- [ ] 固有启动开销有没有藏进用户的阅读/确认动作里？
- [ ] 有没有在哪个取舍上替用户选了"稳妥"而没问？

## 与其他规则的关系

- `CLAUDE.md §6`（禁止空白等待）是底线；本规则是上限——不只"有东西在动"，动的必须是产物。
- `closed-loop-acceptance.md`（验收必须闭环）是同一原则在验收侧的投影：验收的终点也是"产物可见且正确"。
- `chief-designer-usability.md` 第一原则（快启动/边等边看）由本规则给出"看什么"的答案：看产物。
- `zero-friction-input.md` 管输入侧，本规则管输出侧，合起来：**用户的注意力永远落在内容上，不落在系统状态上。**
