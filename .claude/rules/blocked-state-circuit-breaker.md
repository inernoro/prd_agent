# 卡死熔断（Blocked-State Circuit Breaker）

> Agent 撞上「自己无法提供的外部输入」时，必须熔断并升级，禁止用进度文档 / 脚本 / gate 的反复重写来掩盖卡死。本规则唯一目标：把「卡 35 小时还在重写计划」变成「卡 1 小时就停下问人」。

---

## 历史背景（真实发生过）

2026-05-17 ~ 05-18，`codex/cds-agent-workbench-ui` 分支：

- 开 branch 第 6 小时（05-17 07:29）撞上 runtime-pool 外部依赖（需要可 `docker pull` 的 sidecar 镜像 + enabled remote host，agent 自己造不出来）。
- 此后 **~35 小时**没有结构性逃出，节奏不降（10–15 提交/小时，含凌晨），共 361 个 branch-only 提交。
- 头号文件是 `doc/plan.cds-agent-official-sdk-migration.md`，**被重写 118 次**；最近 150 个提交里 124 个是文档/脚本/纯测试，真实功能代码停在 05-18 18:36 之后零产出。
- Agent 自己的台账写了 8 条「兜圈」复盘，写完继续兜。

结论：判断没错（架构上拒绝把 agent runtime 塞进 branch preview 的捷径是对的），但**对的判断 + 错的行为**——该停下抛决策，却 grinding 35 小时 + 42 个脚本。本规则把这条隐形坑显式化。

---

## 一、熔断触发条件（满足任一即触发）

- 同一外部 blocker（需要用户 / 外部系统提供、agent 自身无法产出的输入）跨 **≥ 8 个提交** 或 **≥ 2 小时** 未消除，且期间 `prd-api/src` `prd-admin/src` `prd-desktop/src` 等真实功能代码**净零进展**
- 同一份 `plan.*` / `status.*` / `report.*` 文档被重写 **≥ 5 次**，而 blocker 状态未翻转
- 自己的台账 / 复盘里出现「兜圈 / going in circles / 报告不是修复 / 重复」字样 **≥ 2 次**

判定「净零进展」只看两件事：blocker 状态是否翻转、真实功能代码是否净增。脚本数量、gate 数量、handoff 数量、文档行数**一律不计入进展**。

## 二、触发后强制动作（按序）

1. **立即停止**提交进度看板 / readiness 脚本 / handoff bundle / gate / lifecycle-overview 这类「不消除 blocker」的产物
2. 发出**一条**合并后的 blocker 升级，一次说清最小外部输入清单；**禁止反复重发同一个 ask**，禁止把同一个升级拆成几十个脚本
3. 切换到**不依赖该 blocker 的纯代码工作**并行推进；若确无可并行项 → **挂起等待用户**，不得空转提交
4. 每个「真实状态变化」才允许更新一次进度文档；**禁止无状态变化的文档校准**

## 三、明令禁止的反模式

| 反模式 | 表现 |
|--------|------|
| 进度剧场 | 用脚本 / gate / manifest / lifecycle-overview 的数量制造「在推进」的假象 |
| 计划 thrashing | 重写自己的 plan / status 次数超过真实状态变化次数 |
| 不眠节奏 | 撞 blocker 后仍维持高频提交且功能 delta 为零（尤其凌晨连续提交） |
| 哑巴 grinding | 把「我无法做到 X」写进文档却不停下、不升级、继续刷脚本 |

## 四、与既有规则的关系

- 补 `zero-friction-input.md` 的盲区：那条管「别让用户发呆」，本条管「别让 agent 自己发呆兜圈」
- 与 CLAUDE.md §8.1「自测优先」互补：自测路径穷尽后，若 blocker 确属外部不可自产，**必须升级而非 grinding**，二者不冲突
- 落地「报告不是修复」：报告 / 脚本 / gate 数量不计进展，只有 blocker 状态翻转或功能代码净增才算
- 与 `no-rootless-tree.md` 呼应：缺什么明确暴露 + 借用外力（升级给人），不假装能自己造

## 五、自查清单（长任务每跨 8 个提交自检一次）

- [ ] 最近 8 个提交里，真实功能代码（非 doc/scripts/纯 test）是否净增？
- [ ] 我是否在反复重写同一份 plan/status/report？重写次数 > 状态翻转次数？
- [ ] 当前 blocker 是不是「我自己造不出来、必须外部给」的输入？
- [ ] 如果是：我是否已经发过**一条**清晰升级？发过就该停 / 切纯代码 / 挂起，而不是再写第 N 个脚本
- [ ] 任意一项命中，立即执行第二节的强制动作，不要再提交进度类产物
