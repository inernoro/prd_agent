# 内容区填满画布（Content Fills the Canvas）

> 用户每次说"这块又小又窄 / 怎么这么多留白 / 不大气"，背后都是同一条原则没守住：
> **承载主产物（预览 / 详情 / 结果 / 画布）的区域必须填满可用空间并占据视觉主导，
> 禁止把内容塞进一个内容尺寸的小盒子，让四周大片底色留白。**
> 触发：编辑任何用户直接观看的界面（`prd-admin/src/**/*.tsx`、`prd-desktop/src/**/*.tsx`、`cds/web/src/**/*.tsx`、`cds/web/src/index.css`）。

---

## 核心主张

一屏里"用户要看的那个东西"必须是这屏**最大、最显眼、填满**的元素。如果产物只占一小块、其余是空白底色，就是不合格——空间越大越要给产物，不是给留白。

判定口诀：**把这屏截图，问"主产物是不是这屏里最大、填满可用区域的元素？"** 不是 → 返工。

---

## 三条硬约束

### 1. 主产物区必须 flex-1 占满，不能塌成内容高度

承载预览 / 详情 / 结果的容器必须真正填满它的可用宽高，而不是"有多少内容占多少地方"。

- 容器链：页面根 `h-full min-h-0 flex flex-col` → 内容行 `flex-1 min-h-0` → 产物区 `flex-1 min-h-0`。
- 产物本体（`iframe` / `canvas` / `img` / markdown 渲染 / 列表）：`h-full w-full`（或 `flex-1 min-h-0 overflow-auto`），让它 stretch 填满 pane。
- **iframe 高度绝不能由内容决定**——必须显式 `height:100%` 撑满，短内容时是"高容器 + 顶部内容"，不是"矮容器贴着内容"。

### 2. 高度必须从外壳一路传到产物（断一层就全塌）

`h-full` 只在父级有确定高度时才生效。外壳容器（`<main>` / Workspace / 布局包裹层）若只有 `max-width + margin:auto` 而没有高度/flex，子级 `h-full` 解析不到高度 → 整条链塌成内容高度（这正是 2026-06-21 CDS 验收报告页右侧预览"又小又窄"的根因）。

- 全高页面的外壳要么 `display:flex;flex-direction:column;min-height:0;height:100%`，要么是撑满的 grid item。
- CDS 新栈用 `.cds-workspace--fill` 修饰类承载这件事（`cds/web/src/index.css`）；prd-admin/desktop 走 `full-height-layout.md` 的 `h-full min-h-0 flex flex-col` 链。
- 禁止 `calc(100vh - Npx)` 魔数补高度（见 `full-height-layout.md`）。

### 3. 主从布局：产物是主角，列表/边栏是配角

列表 + 预览、树 + 详情、缩略图 + 大图这类主从布局：

- 预览/详情/产物区 `flex-1`，**占主导宽度**（宽屏下视觉权重 ≥60%）。
- 列表/边栏固定或受限宽度（`shrink-0` + `w-[Npx]` / `lg:w-[360px]`），不要和产物平分秋色。
- 文本类产物可保留可读性上限（`max-width` ~1100px）+ 居中，但**容器仍要填满**，宽屏下用横向 padding 收口而不是让容器缩成窄条。

## 交付前自审

- [ ] 主产物区填满可用宽高了吗？（不是内容高度的小盒子）
- [ ] 高度从外壳一路传到产物了吗？（外壳有 flex/height，没断层）
- [ ] 主从布局里产物占主导比例了吗？（列表是配角，不平分）
- [ ] 短内容/空数据时，容器仍然撑满、没出现大片底色留白吗？
- [ ] 截图自检：主产物是这屏最大、最显眼的元素吗？

任一不达标，回去填满，不许交"又小又窄"。

---

## 与其他规则的关系

- `full-height-layout.md`：管"页面撑满视口高度"（外壳层）；本规则进一步管"产物在撑满的页面里也要填满并占主导"（内容层）。两者是同一件事的外/内两层。
- `artifact-is-experience.md`：产物即体验——本规则保证产物拿到的是**主舞台**，不是角落。
- `miduo-review-lens.md` 镜头 2「比例美感」+ `chief-designer-usability.md` 第三原则「够明显/不遮挡」：本规则是它们的可执行落地。
- `expectation-management.md`：填满的产物区让用户一眼知道"这就是结果"，也是一种预期管理。

## 历史背景

2026-06-21 用户看 CDS 自托管验收报告页，右侧预览只占一个窄小白盒、下方大片留白，问"这是一个什么原则能指导好，类似的问题很多，我不希望每次都说"。根因是 `.cds-main`/`.cds-workspace` 外壳无高度，页面 `h-full` 链断裂。修复：新增 `.cds-workspace--fill` 修饰类 + 本规则固化"内容填满画布"为通用约束。
