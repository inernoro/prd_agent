# AI 辅助缺陷修复

通过缺陷分享链接，按标准工作流自动化分析和修复缺陷。

## 触发词

- "修复缺陷"、"解决缺陷"、"辅助修复"
- "ai fix defects"、"ai resolve"
- 用户提供缺陷分享的提示词内容时

## 认证配置

所有 API 请求需要认证 Header，从环境变量读取：

```bash
# 必须设置
AI_ACCESS_KEY=你的密钥
```

Header 格式：
```
X-AI-Access-Key: $AI_ACCESS_KEY
X-AI-Impersonate: 用户名
```

> Header 名称是 `X-AI-Access-Key`（注意大小写和连字符），环境变量名是 `AI_ACCESS_KEY`。

## 标准工作流（6 阶段）

### 阶段 1：获取缺陷数据

```
GET {viewUrl}
```

读取响应中的缺陷列表、附件截图、AI 评分、API 端点说明。

### 阶段 2：制定修复计划（必须）

**在动手之前，必须先列出完整的修复清单**：

1. 逐条分析每个缺陷
2. 标记安全修复 vs 有风险的修复
3. **有争议或破坏性的修改必须先和人类确认，不要自行决定**

输出格式示例：
```markdown
## 修复清单

| # | 缺陷编号 | 标题 | 修复方案 | 风险等级 |
|---|---------|------|---------|---------|
| 1 | DEF-2026-0017 | 暂无可用生图模型 | 修复模型池查询逻辑 | 安全 |
| 2 | DEF-2026-0039 | 缺陷追踪优化 | 重构状态流转 | ⚠️ 需确认 |
```

### 阶段 3：通过评论沟通（全程）

```
POST {viewUrl}/comments
Content-Type: application/json

{
  "agentName": "你的名称",
  "items": [
    { "defectId": "...", "content": "评论内容（Markdown）" }
  ]
}
```

**必须发表评论的时机**：
- 开始修复前：发表修复计划
- 遇到阻碍时：说明问题和需要人类确认的事项
- 修复完成后：**详细说明验收方式**

### 阶段 4：提交分析报告（可选）

```
POST {viewUrl}/report
Content-Type: application/json

{
  "agentName": "你的名称",
  "items": [
    {
      "defectId": "...",
      "confidenceScore": 85,
      "analysis": "根因分析",
      "fixSuggestion": "修复方案"
    }
  ]
}
```

### 阶段 5：执行修复

根据计划修改代码。注意：
- 遵循代码库的现有规范和架构模式
- 不做超出缺陷范围的修改

### 阶段 6：标记修复完成

**先在评论中说明验收方式，然后调用**：

```
POST {viewUrl}/fix-status
Content-Type: application/json

{
  "items": [
    {
      "defectId": "...",
      "agentName": "你的名称",
      "resolution": "修复说明（含验收方式）"
    }
  ]
}
```

> 会自动标记为「AI 自动解决」并通知缺陷提交者。

## 安全协作规则

1. **先列清单再动手** — 修复前必须列出所有修复方案
2. **有争议的找人确认** — 以下情况必须暂停并询问人类：
   - 删除代码或文件
   - 修改 API 接口签名
   - 架构层面的变更
   - 影响其他模块的修改
   - 不确定根因的缺陷
3. **全程发评论** — 通过评论接口保持信息透明
4. **说明验收方式** — 标记完成时必须告诉提交者如何验证

## 评论模板

### 修复计划评论
```markdown
## 🔧 修复计划

我已分析完所有缺陷，以下是修复方案：

| 缺陷 | 方案 | 风险 |
|------|------|------|
| DEF-xxx | ... | 安全 |

如有疑问请回复，否则我将按此计划执行。
```

### 修复完成评论
```markdown
## ✅ 修复完成

### 修复内容
- DEF-xxx: 修复了 xxx

### 验收方式
1. 打开 xxx 页面
2. 执行 xxx 操作
3. 预期结果：xxx

如验收不通过，请重新打开缺陷并说明问题。
```
