# CN Brief Summary — 中文简明总结

在所有面向用户的最终回复末尾，固定追加一段「200字以内中文总结」，让非技术用户也能快速理解。

## 适用场景

- 用户要求“通俗解释”“小白可懂”
- 任务包含代码改动、排查、上线、文档更新
- 需要让用户快速确认“做了什么、下一步是什么”
- 默认场景：任何最终答复（除纯工具输出）

## 执行流程

创建最终回复时，按以下步骤执行：

1. 先给完整专业结论（正常技术说明）
2. 在末尾追加二级标题：`简明总结（200字内）`
3. 总结必须包含 5 个点（可合并成1段）：
   - 我怎么理解你的需求
   - 我如何判断与思考
   - 我采用了什么办法
   - 我完成了什么结果
   - 下一步建议或待你确认事项
4. 字数限制：`<= 200` 中文字（尽量 140-180 字）
5. 语言风格：口语化、无术语堆砌、无夸张表述

## 输出模板

```markdown
简明总结（200字内）
我理解你的目标是……。我先检查了……，判断关键点在……。随后用……方式完成了……，现在结果是……。下一步建议你……，如果你同意我就继续……
```

## 质量自检

- [ ] 是否放在最终回复末尾
- [ ] 是否控制在 200 字以内
- [ ] 是否覆盖 5 个关键信息点
- [ ] 是否让非技术用户读得懂
