## 关键更新脉络

```mermaid
timeline
    title 关键更新脉络 W{WEEK_NUM} ({MONDAY_SHORT} ~ {SUNDAY_SHORT})
    section {MM-DD} {周X}
      {事件1简述} : {补充说明}
      {事件2简述} : {补充说明}
    section {MM-DD} {周X}
      {事件3简述} : {补充说明}
    ...
```

> **生成规则**：
> - 每个 `section` 对应一个活跃日（提交数 >= 3），格式为 `{MM-DD} {周X}`（如 `02-10 周一`）
> - 每天最多 3 个事件，按重要程度降序
> - 事件主文本 <= 10 字中文，冒号后补充说明 <= 15 字
> - 跳过无实质提交的日期（纯文档/merge commit 日）
> - 如果该周合并了 PR，优先从 PR 标题提炼事件
