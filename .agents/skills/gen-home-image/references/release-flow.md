# 测试与正式发布

## 测试环境

1. 确认所有本地验收通过并记录 `git status --short`。
2. 提交到专用分支并推送同一分支，让 CDS 创建或更新预览环境。
3. 锁定预览域名和提交 SHA；验证 `/ai-toolbox` 深链，不只验证首页。
4. 在浅色与深色模式分别检查紧凑、标准和大图布局；刷新页面确认静态资源可长期加载。

## 正式环境确认门

正式发布属于高风险外部变更。即使用户希望自动化，也必须在发布前明确展示并确认：CDS 项目身份与正式域名、来源分支及不可变 commit SHA、测试环境验证结果、上一个健康生产 SHA 与回滚方式、本次仅包含的文件范围。

确认后，完整读取仓库 `.claude/skills/cds-release/SKILL.md` 并严格执行其 preflight、publish、verify、rollback 流程。不得用临时产物、未提交工作树或可移动分支名作为正式发布来源。

## 生产验证

- 正式域名返回预期 HTML，而不是预览或登录错误页。
- 新增 WebP 静态资源返回成功并具备正确内容类型。
- `/ai-toolbox` 深链可刷新。
- 明暗主题各抽查至少一张新增图；失败立即停止并回滚。
