# 模型网关权威教程

这本书带你从一个业务配置完全空的租户开始，依次完成团队、Provider、模型、默认模型池、appCaller、接入密钥、安全测试、请求查询、提示词治理、费用核对和生产接入。平台运维需要先安全交付首个 Owner 账号；公网登录页不会让陌生人自助领取 Owner。每章都从上一章的结果继续，每个编号步骤下方直接放对应圈选图，不需要在正文与章末图库之间来回滚动。

## 阅读路线

1. 基础篇，第 0 至 13 章：完成第一条可定位的安全测试请求。
2. 中级篇，第 14 至 24 章：让团队稳定、安全地使用并能追溯费用与操作。
3. 高级篇，第 25 至 32 章：完成隔离自检、四协议验收、治理和生产回滚准备。

## 全书共同规则

- 当前租户只由登录会话或接入 key 在服务端确定。请求中不要填写 `tenantId`。
- 每个接入方、环境和用途使用不同的 key。没有 key 就不能调用，也不能形成可靠审计。
- PromptPolicy 首版只用于 chat 和 vision。日志只记录策略 id、版本和 hash。
- unknown cost 表示当前没有可信金额，不能当作 0。
- CNY 和 USD 没有可审计汇率时分别展示，不能直接相加。
- 默认使用安全测试和假上游；每类真实协议最多验证一次，不做批量付费测试。

## 连续示例

全书使用“教程咖啡店”作为租户，使用“客服组”和“内容组”作为团队。你会配置“教程假上游”“教程聊天模型”“教程视觉模型”“默认对话池”和“默认视觉池”，再创建 `tutorial.gateway-book::chat` 与 `tutorial.gateway-book::vision` 两个 appCaller。生产操作时请在名称末尾增加自己的唯一短后缀，避免和其他练习者冲突。

## 如果你只想知道租户能用多少

先记住一句话：租户像一间独立办公室，既隔离数据，也提供跨所有业务的预算和速率总闸；appCaller 与 key 再把额度细分到“谁在做什么”和“从哪把钥匙进来”。

1. 到“预算与用量”设置租户总月预算、单次原子预占和总 RPM。
2. 到“appCaller”给每项业务设置更细的月预算、单次预算预占和每分钟上限。
3. 到“接入密钥”给每个接入方的 key 设置每分钟上限，需要停用谁就撤销谁的 key。
4. 回到“预算与用量”核对整个当前租户用了多少、费用是否可信、哪些请求还是 unknown cost。

租户总预算不是各 appCaller 预算的求和，而是独立的服务端原子硬限制。租户、appCaller 和 key 任一层触顶都会拒绝请求。第 4、14、22 章会一步步带你完成分层管理。
