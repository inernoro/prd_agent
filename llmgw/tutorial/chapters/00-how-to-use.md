# 第 0 章 这本书怎么用

## 你在做什么

你将跟着一条不丢步骤、不换练习数据的路线，从空白租户走到第一条可审计请求。基础配置由 Owner 完成；讲到权限边界时，会明确让你另开 Developer 或 Viewer 会话，但不会要求你把唯一 Owner 降权。故事中的店叫“教程咖啡店”，有“客服组”和“内容组”，上游叫“教程假上游”，模型叫“教程聊天模型”和“教程视觉模型”。书中出现的名字是便于辨认的短名；实际多人同时练习时，管理员可能在末尾加一段编号，这是正常的。

## 为什么要做

模型网关页面很多，如果只随便点几个菜单，很容易留下有模型却没有池、有密钥却没有 appCaller 的“半成品”。本书每章都以上一章的结果为起点，并告诉你成功标志和恢复办法。你不需要记住技术地址，也不要自己猜字段。基础篇先把第一条安全请求跑通，中级篇再学团队协作和密钥轮换，高级篇最后做隔离、费用和生产检查。

## 开始前检查

- 准备一个可以登录 Gateway 控制台的账号。创建租户和路由配置最好使用 Owner；Admin 可完成大多数配置；Developer 只能管理所属团队的 appCaller 与限定密钥。
- 准备一个密码保管工具。书中不会展示真实密钥，创建时出现的明文只能看一次。
- 确认练习只用隔离环境和假上游，不改生产共享池，不测试批量付费模型。
- 浏览器能打开 MAP，并能从 MAP 进入模型网关控制台。手机也能阅读，但首次配置建议用桌面宽屏。

## 看到什么算成功

你已经知道完整路线，能说出本书的练习租户、两个团队、两个模型和两个默认池；你也知道密码、Provider 通讯密钥与 `gwk_` 接入密钥都不能写进教程记录。最重要的是，你接受“每章确认成功后再继续”的规则。

## 失败怎么办

- 没有控制台账号：联系目标租户的 Owner 或 Admin 在“团队与成员”中创建独立账号，不要借用他人账号。
- 只能看到很少的左侧菜单：查看顶部当前租户和自己的角色；Viewer、Billing、Developer 的可见范围本来就不同。
- 不知道是否在测试环境：先停止操作，让管理员确认租户用途和 Gateway 地址；不要拿生产租户试书中数据。
- 页面名称有细微变化：以页面中的中文说明和成功提示为准，先不要用地址栏猜路径。

## 本章小结

这本书是一条连续操作流，不是互不相干的功能说明。你的练习证据只保留名称、状态和 requestId，所有秘密都留在安全系统中。
