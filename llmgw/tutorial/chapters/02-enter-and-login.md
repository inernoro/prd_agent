# 第 2 章 找到入口并登录

## 你在做什么

你将从 MAP 的正常导航进入 Gateway 控制台，并使用独立的控制台账号登录。登录后先确认当前用户、当前租户和角色，不急着创建配置。

## 为什么要做

MAP 账号、Gateway 控制台账号和 `gwk_` 应用接入密钥是三种不同身份。混用会造成“明明有 key 却登录不了”或把应用密钥暴露给人的问题。从产品入口进入还能避免记错域名，并让返回 MAP 的路径清楚可见。

## 开始前检查

- 已得到本租户 Owner 或 Admin 单独提供的控制台用户名与初始密码。
- 初始密码只用于首次验证，不在聊天、截图或工单中转发。
- 浏览器地址栏显示可信的 MAP 域名；不要使用别人发来的陌生登录页。

## 跟我做

1. 登录 MAP，回到首页的模型兼容区域，点击“打开模型网关控制台”。等待 Gateway 登录页显示“管理租户、密钥、路由、请求与费用”。
2. 阅读登录框旁说明，确认这里明确写着 MAP 账号与 Gateway 控制台账号彼此独立。
3. 输入管理员给你的控制台用户名和初始密码，提交登录。
4. 如果页面要求首次改密，输入管理员单独提供的初始口令，再设置只属于自己的新密码；新密码至少 12 位。
5. 进入控制台后查看顶部用户菜单和租户切换器，核对用户名、当前租户以及角色。
6. 登录后的顶部“文档”负责打开说明；如需返回 MAP，使用浏览器返回或重新打开 MAP 首页。不要把“文档”误当成返回按钮。

## 看到什么算成功

左侧出现与你角色匹配的导航，顶部只承担品牌、租户切换、搜索、文档和用户菜单。你能看到当前租户名称；如果账号属于多个租户，切换器只列出已有成员关系的租户。刷新页面后仍保持已登录状态，且不会再次显示初始密码。

## 失败怎么办

- 提示账号或密码错误：检查是否误用了 MAP 密码或 `gwk_` 密钥；连续失败时联系租户管理员核对账号，不要要求重置其他人的密码。
- 登录后要求改密：这是首次登录保护，按页面完成；若旧口令未知，让创建该账号的管理员重新走成员管理流程，不能猜公共默认口令。
- 登录后没有目标租户：你尚未加入该租户，联系 Owner 或 Admin 创建成员关系；请求里填写 `tenantId` 不能解决。
- 页面反复跳回登录：记录出现时间和页面提示，联系管理员检查账号状态或会话是否被强制失效。

## 本章小结

人使用控制台账号登录，应用使用接入密钥调用 Gateway。租户范围由服务器根据会话确定，不由页面输入或请求参数决定。
