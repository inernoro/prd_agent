# 第 2 章 找到入口并登录

## 你在做什么

你将从 MAP 左下角用户菜单进入 Gateway 控制台。MAP 管理员会通过一次性授权自动登录，不再填写另一套密码；独立租户用户仍使用 Gateway 控制台账号登录。进入后先确认当前用户、当前租户和角色，不急着创建配置。

## 为什么要做

MAP 管理员已经在 MAP 完成身份校验，不应为了进入内部模型网关再保存或输入一套共享密码。MAP 只签发 60 秒、单次消费的随机授权码，Gateway 消费后建立自己的短会话；MAP access token、密码和 `gwk_` 应用接入密钥都不会跨系统传递。独立租户的控制台账号仍是另一种身份，不能与应用 key 混用。

## 开始前检查

- 本书所说的“从全空开始”是指租户内还没有 Provider、模型、模型池成员、appCaller 和业务 key，不是指整个平台连第一个人类账号都没有。出于安全原因，公网登录页不允许任何人自助领取首个 Owner。
- MAP 内部管理员：当前 MAP 会话仍有效，左下角用户菜单中能看到“模型网关”。普通 MAP 角色默认看不到该入口，也不能直接请求一次性授权。
- 独立租户用户：已得到本租户 Owner 或 Admin 单独提供的控制台用户名与初始密码。初始密码只用于首次验证，不在聊天、截图或工单中转发。
- 浏览器地址栏显示可信的 MAP 域名；不要使用别人发来的陌生登录页。

## 看到什么算成功

MAP 管理员从左下角入口点击一次即可进入 Gateway 内部租户，地址栏不再保留一次性授权码；Gateway 用户菜单提供“返回 MAP”。普通 MAP 用户看不到入口，即使直接请求签发接口也会被服务端拒绝。独立租户账号仍可从登录页进入，且只看到已有成员关系的租户。

## 失败怎么办

- 整个平台还没有首个 Owner：停止在登录页尝试。由平台运维按受控初始化流程创建首个租户与唯一 Owner，再把一次性初始凭据单独交给负责人；普通用户不能通过请求参数或注册按钮把自己提升为 Owner。
- MAP 用户菜单没有“模型网关”：先核对当前角色；该入口只对 MAP 管理员显示，普通角色默认没有此功能。
- 一键登录提示链接无效或已过期：返回 MAP，再从左下角用户菜单点击一次。授权码只能使用一次且 60 秒过期，不要刷新旧过渡页。
- 提示账号或密码错误：检查是否误用了 MAP 密码或 `gwk_` 密钥；连续失败时联系租户管理员核对账号，不要要求重置其他人的密码。
- 登录后要求改密：这是首次登录保护，按页面完成；若旧口令未知，让创建该账号的管理员重新走成员管理流程，不能猜公共默认口令。
- 登录后没有目标租户：你尚未加入该租户，联系 Owner 或 Admin 创建成员关系；请求里填写 `tenantId` 不能解决。
- 页面反复跳回登录：记录出现时间和页面提示，联系管理员检查账号状态或会话是否被强制失效。

## 本章小结

MAP 管理员通过一次性授权无缝进入内部 Gateway，独立租户用户使用控制台账号，应用使用接入密钥。三条身份链彼此隔离，租户范围始终由服务器会话确定。
