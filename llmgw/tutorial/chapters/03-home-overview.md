# 第 3 章 看懂首页

## 你在做什么

你将把“概览”当作当前租户的仪表盘，依次看健康状态、快速接入、最近请求、关键指标、常用模型、Top appCaller、成员与密钥，而不是把它当成系统发布后台。

## 为什么要做

每天进入 Gateway，最先要回答四件事：网关是否可用、现在该从哪里开始、最近有没有失败请求、费用数据是否可信。模型迁移、容器拓扑和发布检查属于“系统运维”，不应挤占租户首页。先学会读首页，后面每次配置后都能回到这里确认整体变化。

## 开始前检查

- 顶部租户切换器显示“教程咖啡店”或你准备创建它之前的当前租户。
- 左侧“工作区”下可以看到“概览”。
- 空租户时很多数字为 0 是正常的，但费用未知不应显示为金额 0。

## 看到什么算成功

你能区分“Gateway 健康”和“业务路由已就绪”，也能解释空状态为什么合理。首页没有把 unknown cost 写成 0；CNY 与 USD 没有可信汇率时不会被相加成一个总额。你知道第一条请求成功后，最近请求和 appCaller 统计会在这里出现。

## 失败怎么办

- 首页一直显示加载：刷新一次；仍失败时记录页面错误和时间，不要反复创建对象，交给管理员检查 API 健康。
- 健康状态不可用：先停止真实调用；只有内部租户的 Owner 或 Admin 才查看“系统运维”，其他用户把首页状态和时间交给管理员。
- 最近请求为空：确认当前租户和时间范围；尚未做 Quickstart 时这是预期结果。
- 费用显示未知：说明价格覆盖或供应商账单不足，不要手工把它改写成 0，第 22 至 23 章会处理。

## 本章小结

概览是租户工作台：第一屏关心可用性、上手路径、最近请求和费用可信度，深层运行信息留在治理页面。
