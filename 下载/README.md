# 产品管理系统 · 本地测试资料

本目录存放产品管理智能体（product-agent）的说明与测试数据，可直接用于 MAP 导入验收。

## 文件说明

| 文件 | 用途 |
|------|------|
| `产品管理系统设计理念.md` | 基于当前系统已落地能力整理的设计理念与对象关系说明 |
| `产品管理系统-需求测试数据.csv` | 50 条需求测试数据，主题为「产品管理系统」 |

## CSV 导入格式（与系统一致）

首行表头必须为：

```text
外部 ID,标题,描述,等级,状态,计划发布时间,实际发布时间
```

| 列 | 说明 |
|----|------|
| 外部 ID | TAPD 风格纯数字或带前缀 ID；相同 ID 重复导入会**更新**原记录 |
| 标题 | 必填 |
| 描述 | 选填 |
| 等级 | `p0` / `p1` / `p2` / `p3`（小写） |
| 状态 | 中文工作流名：待评审、待规划、已立项、开发中、已排期、已上线、已拒绝 等 |
| 计划发布时间 | 选填，如 `2026-07-01` |
| 实际发布时间 | 选填；已上线需求可填完成日期 |

## 导入步骤

1. 登录 MAP 管理后台，进入 **产品管理**。
2. 确保已有归属产品（建议先创建名为「产品管理系统」的产品）。
3. 打开 **需求** 总览（或单产品内需求 Tab 的管理员导入入口）。
4. 点击 **导入历史数据** → 选择归属产品 → 上传 `产品管理系统-需求测试数据.csv`。
5. 预览确认 50 条后点击 **确认导入**。

需 **产品管理应用管理员** 权限。

## 编码

CSV 为 **UTF-8 with BOM**，Excel 直接打开中文不乱码。
