# 缺陷管理 Agent 使用教程

> **类型**：guide | **适用角色**：测试工程师、开发人员、项目经理 | **最后更新**：2026-03-19

---

## 1. 什么是缺陷管理 Agent

缺陷管理 Agent 是智能缺陷分析与管理助手。提供可视化缺陷看板、智能优先级评估、自动分配、完整的生命周期追踪，以及 AI 驱动的缺陷分析和修复建议。

**适用场景**：
- 测试阶段需要高效管理大量缺陷
- 需要 AI 辅助分析缺陷原因和修复建议
- 跨团队协作，需要缺陷状态可视化
- 需要缺陷报告和趋势分析

---

## 2. 快速上手

### 步骤一：创建缺陷项目

1. 登录后进入缺陷管理 Agent
2. 点击「创建项目」，填写项目信息
3. 邀请团队成员加入项目

### 步骤二：提交第一个缺陷

1. 点击「提交缺陷」按钮
2. 填写缺陷标题和详细描述
3. 上传截图或录屏（支持图片、视频）
4. 选择严重程度和优先级
5. AI 自动分析并补充缺陷分类标签

### 步骤三：管理和跟踪

- 在看板视图中拖拽卡片切换状态
- 收到的缺陷在「收到的」标签页中查看
- 处理完成后更新状态并添加处理说明

## 3. 核心功能详解

### 3.1 缺陷看板

可视化管理所有缺陷的状态流转。

**看板列**：
- **待处理**：新提交的缺陷
- **处理中**：已分配并开始处理
- **已处理**：已修复等待验证
- **已关闭**：验证通过关闭

**操作方式**：
- 拖拽卡片在列之间移动
- 点击卡片查看详情
- 使用筛选器按严重程度、负责人等过滤

### 3.2 AI 智能分析

AI 自动分析缺陷内容，提供：
- **缺陷分类**：自动识别缺陷类型（UI、功能、性能、安全等）
- **优先级建议**：基于影响范围和严重程度评估
- **相似缺陷检测**：自动关联历史相似缺陷，避免重复提交
- **修复建议**：AI 分析可能的原因并给出修复方向

### 3.3 缺陷分享与外部协作

将缺陷分享给外部团队或客户。

**使用方法**：
1. 在缺陷详情页点击「分享」
2. 生成分享链接（可设置有效期）
3. 外部人员通过链接查看缺陷详情
4. 支持外部 Agent 分析缺陷并生成修复报告

### 3.4 修复报告

AI 生成结构化的缺陷修复报告：
- 问题定位过程
- 根因分析
- 修复方案
- 验收标准

## 4. 缺陷生命周期

```
提交 → 待处理 → 处理中 → 已处理 → 验收 → 已关闭
                    ↓                    ↓
                  挂起 ←────────── 打回重修
```

### 状态说明

| 状态 | 说明 | 操作者 |
|------|------|--------|
| 待处理 | 新提交，等待分配 | 提交者 |
| 处理中 | 已开始修复 | 开发人员 |
| 已处理 | 修复完成等待验证 | 开发人员 |
| 验收 | 测试验证中 | 测试人员 |
| 已关闭 | 验证通过 | 测试人员 |
| 挂起 | 暂缓处理 | 项目经理 |

## 5. 进阶用法

### 5.1 缺陷模板

创建常用的缺陷描述模板，统一团队的缺陷提交规范。

### 5.2 Webhook 集成

配置 Webhook 将缺陷事件推送到其他系统（如企业微信、钉钉）。

### 5.3 批量操作

选择多个缺陷进行批量状态变更、批量分配等操作。

### 5.4 数据分析

查看项目的缺陷趋势图、分布统计，帮助评估质量状况。

## 6. 常见问题

**Q：可以从其他缺陷管理工具导入数据吗？**
A：目前支持通过 API 导入，后续将支持常见格式的直接导入。

**Q：缺陷通知机制是怎样的？**
A：缺陷状态变更时，相关人员会收到站内通知。配置 Webhook 后还可以推送到外部渠道。

**Q：AI 分析缺陷需要多长时间？**
A：通常几秒内完成分析。批量分析时会有进度提示。

**Q：外部分享链接安全吗？**
A：分享链接可以设置有效期，过期自动失效。不包含敏感的项目内部信息。
