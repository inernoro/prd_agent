# 工作流引擎使用教程

> **类型**：guide | **适用角色**：开发人员、运维人员、自动化爱好者 | **最后更新**：2026-03-19
> **预计阅读**：10 分钟 | **难度**：中级

---

## 1. 功能概览

工作流引擎提供可视化流程编排，通过拖拽「胶囊」节点串联自动化任务。

| 能力 | 说明 |
|------|------|
| 画布编辑器 | 拖拽式节点编排，所见即所得 |
| 5 种胶囊节点 | HTTP 请求、延时、条件判断、格式转换、通知推送 |
| 执行管理 | 一键运行、实时日志、执行历史 |
| Artifact 预览 | 每个节点的输入/输出数据可视化 |
| 定时调度 | Cron 表达式定时执行 |

---

## 2. 操作路径

**入口**：首页 → 点击「工作流引擎」卡片 → 进入工作流列表

---

## 3. 完整操作流程

### 步骤 1：创建工作流

1. 在工作流列表页，点击创建按钮
2. 输入工作流名称和描述
3. 进入画布编辑器

> **Pro Tip**：首次使用可以选择内置模板快速体验。DEMO 模板包含完整的 5 步流程：获取数据 → 延时等待 → 条件判断 → 格式转换 → 通知。

### 步骤 2：编排节点

在画布编辑器中：

1. 从侧边栏拖拽胶囊节点到画布（或双击画布添加）
2. 从一个节点的输出端口 **拖线** 到下一个节点的输入端口
3. 点击节点打开配置面板，填写参数

**5 种胶囊节点**：

| 节点类型 | 标识 | 用途 | 输入/输出示例 |
|---------|------|------|-------------|
| **HTTP 请求** | Globe 图标 | 调用外部 API | 输入：API 地址 → 输出：JSON 响应 |
| **延时** | Clock 图标 | 等待指定时间 | 输入：等待秒数 → 输出：透传上游数据 |
| **条件判断** | GitBranch 图标 | 根据条件分支 | 输入：判断表达式 → 输出：TRUE/FALSE 两路 |
| **格式转换** | RefreshCw 图标 | 数据格式转换 | 输入：JSON → 输出：CSV（或反向） |
| **通知推送** | Bell 图标 | 发送提醒通知 | 输入：消息内容 → 输出：发送结果 |

**节点配置要点**：
- **输入来源**：选择 **「无（自动请求）」** 或 **「步骤 N 数据」**（使用上游节点的输出）
- **输出格式**：如 **「JSON 用户列表」**、**「CSV 格式数据」**

> **Pro Tip**：条件判断节点有 TRUE 和 FALSE 两个输出端口，可以实现分支逻辑。例如：数据为空走通知路径，不为空走处理路径。

### 步骤 3：测试运行

1. 点击 **运行** 按钮（Play 图标）
2. 右侧显示 **「执行日志」** 面板，实时输出每个节点的执行状态：
   - 蓝色 `info` — 节点开始执行
   - 绿色 `success` — 执行成功
   - 红色 `error` — 执行失败
   - 黄色 `warn` — 警告信息
3. 执行状态依次变化：**「排队中」** → **「执行中」** → **「已完成」** / **「失败」**
4. 运行中可点击 **取消**（X 图标）中断

### 步骤 4：查看结果

- 点击任意节点查看 **「Artifact」** 面板 — 该节点的输出数据
- 点击 **「Download」** 下载 Artifact（支持 JSON、CSV 等格式）
- 如果节点有 **「?」**（HelpCircle）图标，悬停查看上下文帮助

## 4. 画布操作速查

| 操作 | 方式 |
|------|------|
| 添加节点 | 从侧边栏拖拽 / 双击画布 |
| 连接节点 | 从输出端口拖线到输入端口 |
| 编辑配置 | 点击节点 |
| 删除节点 | 选中 → Delete 键 / 右键删除 |
| 缩放 | 鼠标滚轮 |
| 平移 | 空格 + 鼠标拖拽 |

## 5. DEMO 模板解析

内置的 DEMO 模板包含 5 个步骤，帮助你理解工作流运行机制：

```
🌐 获取测试数据 → ⏳ 延时等待(3s) → 🔀 条件判断 → 🔄 格式转换(JSON→CSV) → 🔔 空数据通知
                                        ↓ TRUE
                                   数据不为空，走格式转换
                                        ↓ FALSE
                                   数据为空，走通知提醒
```

## 6. 常见问题

**Q：工作流可以定时执行吗？**
A：可以。在工作流设置中配置 Cron 调度计划。

**Q：执行失败怎么调试？**
A：在执行日志中定位失败节点（红色 `error` 条目），点击节点查看输入/输出数据和错误信息。

**Q：一个工作流最多多少节点？**
A：理论无限制，建议 50 个以内保持可维护性。复杂流程建议拆分为多个工作流。
