# 工作流引擎使用教程

> **类型**：guide | **适用角色**：开发人员、运维人员、自动化爱好者 | **最后更新**：2026-03-19

---

## 1. 什么是工作流引擎

工作流引擎提供可视化的流程编排能力。通过拖拽「胶囊」节点，串联 HTTP 请求、条件判断、数据转换、通知推送等操作，实现自动化任务编排。

**适用场景**：
- 定时从外部 API 抓取数据并推送通知
- 自动化测试流水线
- 数据格式转换与分发
- 多步骤审批流程自动化

---

## 2. 快速上手

### 步骤一：创建工作流

1. 进入工作流引擎页面
2. 点击「创建工作流」
3. 输入工作流名称和描述

### 步骤二：编排节点

1. 进入画布编辑器
2. 从左侧面板拖拽胶囊节点到画布
3. 连接节点之间的数据流
4. 配置每个节点的参数

### 步骤三：测试运行

1. 点击「测试运行」验证流程
2. 查看每个节点的输入/输出
3. 确认无误后启用工作流

---

## 3. 胶囊节点类型

| 节点类型 | 图标 | 用途 |
|---------|------|------|
| HTTP 请求 | Globe | 调用外部 API |
| 延时 | Clock | 等待指定时间 |
| 条件判断 | GitBranch | 根据条件分支（TRUE/FALSE） |
| 格式转换 | RefreshCw | JSON ↔ CSV 等数据格式转换 |
| 通知推送 | Bell | 发送提醒通知 |

---

## 4. 画布操作

- **添加节点**：从侧边栏拖拽或双击画布
- **连接节点**：从输出端口拖线到下一节点的输入端口
- **编辑参数**：点击节点打开配置面板
- **删除节点**：选中后按 Delete 或右键删除
- **缩放/平移**：滚轮缩放，空格+拖拽平移

---

## 5. 常见问题

**Q：工作流可以定时执行吗？**
A：可以。在工作流设置中配置调度计划（Cron 表达式）。

**Q：一个工作流最多支持多少个节点？**
A：理论上无限制，建议保持在 50 个节点以内以确保可维护性。

**Q：执行失败后如何调试？**
A：在执行历史中查看每个节点的日志，定位失败节点的输入/输出数据。
