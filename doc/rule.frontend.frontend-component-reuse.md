# 前端组件复用 · 规则

> **版本**：v2.0 | **日期**：2026-07-17 | **状态**：已落地

同一业务概念在两个以上页面出现时，应优先使用共享组件和共享选项事实源，避免各页面复制枚举、标签和交互。

## 1. 必须复用的内容

- 模型类型、平台、状态、权限等稳定枚举选择器。
- 同语义的状态徽章、筛选栏、上传器和空状态。
- 已进入全局设计系统的按钮、弹窗、表单和反馈组件。
- 同一领域在多个页面使用的数据格式化和展示映射。

仅外观相似但业务语义不同的元素不强行合并。

## 2. 放置规则

| 范围 | 位置 |
|---|---|
| 全应用通用 UI | `src/components/` 对应领域目录 |
| 单 Agent 多页面复用 | 该 Agent 的 `components/` |
| 选项、标签和颜色映射 | 领域注册表或共享常量文件 |
| 仅单页使用 | 页面局部组件 |

组件只接收展示所需的值和事件，不读取另一页面的私有状态。

## 3. 禁止事项

- 多个页面各自维护同一组选项。
- 让用户手输已有枚举值。
- 为绕过共享组件限制复制一份“临时版本”。
- 共享组件内硬编码某个页面的业务流程。
- 仅抽 JSX 外壳，却保留多份不一致的数据映射。

## 4. 变更要求

修改共享选项或组件时必须搜索全部消费者，检查：

- 字段和枚举兼容性。
- 默认值、空值和禁用状态。
- 键盘操作、可访问名称和错误提示。
- 深色与浅色主题。
- 移动和宽屏布局。

如共享组件无法覆盖合理差异，应扩展明确的属性，而不是复制组件。属性超过职责边界时再拆成基础组件和领域包装层。

## 5. 验收

- 同一业务概念只有一处选项与标签事实源。
- 页面之间的选中值和禁用规则一致。
- 修改共享组件后相关消费者的类型检查和测试通过。
- 没有新增纯文本枚举输入或重复映射。

具体已注册组件以源码目录和引用关系为准，本规则不维护容易过时的组件清单。
