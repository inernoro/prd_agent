# 工作流引擎 · 设计

> **版本**：v2.1 | **日期**：2026-07-17 | **状态**：已落地

> **appKey**：`workflow-agent`

## 管理摘要

- **问题**：数据采集、分析、报告、通知等跨系统任务依赖人工串联，难以复用、追踪和定时执行。
- **决策**：以可视化 DAG 定义工作流，由服务端 Run/Worker 执行，使用制品管道传递节点数据，并通过 SSE 提供持续进度。
- **价值**：Agent 获得外部系统能力，业务流程获得 AI 能力；一次执行可追溯、可续跑、可分享。
- **范围**：工作流编辑器、执行引擎、调度、制品、分享，以及与 LLM Gateway、Automation Hub 和开放平台的集成。

## 用户场景

| 场景 | 编排结果 |
|---|---|
| 周报数据采集 | 按计划采集外部项目与代码数据，作为周报素材 |
| 缺陷通知 | 由领域事件触发，分析缺陷后向目标人员发送通知 |
| 内容生产 | 外部内容解析、语音转写、AI 改写和报告渲染串联为一条管线 |
| 网页发布 | 将工作流产出的 HTML 交给托管服务，生成可访问链接后通知相关人 |

## 核心决策

1. **DAG 而非线性脚本**：节点和连线描述依赖关系，编排器只执行依赖已满足的节点。
2. **服务端权威执行**：浏览器负责配置、启动和观察；Run、节点状态、产物和失败结论由服务端写入。
3. **制品优先传递**：节点通过具名制品消费上游结果，避免在节点之间复制大文本或临时文件。
4. **能力经统一网关调用**：涉及模型的节点通过 LLM Gateway，并使用 `workflow-agent` 的应用身份。
5. **可恢复且可审计**：执行保留定义快照、节点记录和产物；符合条件的失败可从指定节点续跑或重放。

## 架构与边界

| 层级 | 职责 | 约束 |
|---|---|---|
| 工作流定义 | 节点、连线、变量和触发条件 | 发布后执行使用快照，避免编辑影响运行中任务 |
| 执行引擎 | 依赖调度、取消、续跑和状态聚合 | 后台 Worker 处理，不能由前端模拟执行 |
| 节点能力 | 采集、转换、控制流、AI、渲染、通知和发布 | 每类能力声明输入、输出和配置校验 |
| 制品管道 | 文本、结构化数据、图片、HTML 和文件 | 大制品进入对象存储，节点间传递受控引用 |
| 观察与分享 | SSE、日志、执行详情和分享链接 | 分享遵循访问控制、过期和撤销策略 |

节点类型按能力分为触发器、数据采集、HTTP 与脚本处理、AI 分析或生成、控制流、渲染与输出、通知与发布。具体舱类型、字段和编辑器表现以 `capsuleRegistry.tsx` 与后端类型定义为准，不在设计文档复制实现代码。

## 数据与制品契约

| 对象 | 责任 |
|---|---|
| Workflow | 保存可发布的 DAG 定义、变量和触发配置 |
| WorkflowExecution | 保存一次运行的定义快照、总体状态、时间和发起来源 |
| NodeExecution | 保存节点状态、输入摘要、输出引用、错误和重放信息 |
| Artifact | 保存节点产物的类型、位置、摘要和访问控制 |
| Schedule / Trigger | 保存定时、事件或人工触发条件 |
| ShareLink | 管理可公开或需登录访问的执行产物链接 |

小型产物可内联保存；大文件进入对象存储并以受控引用传递。任何节点都不能将密钥、完整敏感上下文或未脱敏日志作为可分享制品。

## 执行、调度与实时反馈

1. 用户、计划任务、领域事件或开放接口创建执行实例。
2. Worker 校验工作流定义和节点配置，按依赖关系执行可运行节点。
3. 节点产出写入制品管道；下游节点按声明读取制品或引用。
4. 引擎持续写入执行状态并通过 SSE 推送阶段、节点进度、等待审批、错误和完成事件。
5. 取消、续跑、重放和分享均以执行记录为准，不修改历史输入。

定时任务由调度 Worker 驱动；事件触发由 Automation Hub 等发布方提供。需要人工确认的控制流节点应明确进入等待状态，不能以静默超时替代决策。

### 控制流与 SSE 契约

| 能力 | 约束 |
|---|---|
| 延时节点 | 等待期间保持执行可取消，完成后透传上游制品 |
| 条件节点 | 按声明的字段与运算符求值，只激活命中输出槽 |
| 未命中分支 | 标记为跳过并传播到其独占下游，不伪装成成功执行 |
| 分支合流 | 只等待实际激活且可到达的上游，避免被跳过分支永久阻塞 |
| SSE 事件 | 至少覆盖阶段、节点开始、节点完成、跳过、失败和执行终态 |
| 断线恢复 | 客户端按事件序号续读；不可恢复时回读服务端执行快照 |
| 降级 | SSE 不可用时允许有限轮询，但服务端状态仍是唯一权威 |

控制流语义由后端执行引擎裁决，前端连线槽只表达用户配置。条件求值、跳过传播和合流判断必须保留节点日志，便于解释为什么某条路径没有执行。

## 接口与界面

| 能力 | 稳定入口 |
|---|---|
| 工作流与模板 | `/api/workflow-agent/workflows` |
| 启动、取消、续跑、重放 | `/api/workflow-agent/executions` 相关端点 |
| 实时进度 | `/api/workflow-agent/executions/{executionId}/stream` |
| 调度与分享 | `/api/workflow-agent/schedules`、`/api/workflow-agent/shares` |
| 画布与编辑器 | `/workflow-agent`、`/workflow-agent/{workflowId}`、`/workflow-agent/{workflowId}/canvas` |

前端入口、权限和路由以 `prd-admin/src/app/navRegistry.tsx` 与 `App.tsx` 为准。接口的完整字段和错误语义以 Controller 与前端契约文件为准。

## 安全与风险

| 风险 | 处理原则 |
|---|---|
| 脚本或生成代码执行 | 使用受限执行环境，限制网络、资源、挂载和运行时长 |
| 凭证泄露 | 加密保存，日志脱敏，列表和分享结果不返回明文 |
| 大文件放大 | 制品按大小转入对象存储，限制下载和注入方式 |
| 分享误用 | 支持访问模式、撤销和有效期；分享前检查制品敏感性 |
| 长任务无反馈 | SSE 持续推送阶段与节点状态，前端不得静态等待 |
