# 技能系统统一设计 · 设计

> **版本**：v2.0 | **日期**：2026-07-17 | **状态**：已落地

## 一、管理摘要

- **解决什么问题**：历史提示词与技能双轨维护，管理入口、执行读取和用户概念不一致。
- **当前决策**：账户技能统一保存于 `skills` 集合，由技能服务、管理后台和客户端共同消费；`SKILL.md` 是跨平台交换格式。
- **兼容边界**：旧提示词数据和旧客户端接口只承担迁移兼容，不再形成可写的第二事实源。
- **范围区分**：账户技能、官方技能包和海鲜市场条目用途不同，不能因为都含 `SKILL.md` 就混为同一集合。

## 1. 目标

- 管理、执行和客户端展示统一读取账户技能事实源。
- 系统、公共和个人技能使用同一模型表达可见性与所有权。
- 支持从对话沉淀、人工创建、导入、编辑和导出技能。
- 通过 `SKILL.md` 与外部 Agent 工具交换，不泄露平台内部字段。
- 保留可审计的兼容路径，并让新写入不再进入旧提示词集合。

## 2. 领域边界

| 领域 | 事实源 | 用途 |
|------|--------|------|
| 账户技能 | `skills` | 系统、公共和个人技能的管理与执行 |
| 官方技能 | 代码或官方目录 | 平台维护、版本固定、可下载安装的技能包 |
| 海鲜市场 | `marketplace_skills` | 用户发布、发现、版本和分享技能包 |
| 本地项目技能 | 仓库 `.claude/skills` 或兼容目录 | 当前代码库的 Agent 工作规则，项目内置优先 |
| 旧提示词 | `promptstages` | 迁移和旧版本兼容，不接受新产品能力依赖 |

账户技能统一不意味着所有技能载体强制入同一集合。每个领域保留自身生命周期，但交换格式和解析器应尽量复用。

## 3. 账户技能模型

账户技能至少表达：

| 信息 | 作用 |
|------|------|
| `skillKey` | 稳定标识和执行查找键 |
| 标题与描述 | 管理、搜索和展示 |
| 执行模板 | 模型或工具执行所需的指令内容 |
| 输入与输出 | 描述调用契约和结果预期 |
| 所有者与可见性 | 区分系统、公共和个人范围 |
| 角色与排序 | 控制客户端呈现和推荐顺序 |
| 来源与内置标记 | 区分迁移、用户创建和系统提供 |

服务端按当前用户和可见性过滤。客户端不能通过修改 `ownerId`、`visibility` 或 `isBuiltIn` 获得额外管理权限。

## 4. 旧提示词迁移

迁移遵循以下原则：

1. 以旧 `promptKey` 作为目标 `skillKey`，保持调用标识稳定。
2. 提示词正文进入技能执行模板，角色和排序按现有语义映射。
3. 目标技能已存在时不覆盖，避免启动迁移破坏管理员新配置。
4. 迁移操作幂等，重复启动不会产生重复技能。
5. 原集合在兼容期保留，但新管理操作只写 `skills`。

旧接口若仍被客户端使用，应从 `skills` 读取并转换旧 DTO。兼容层不得继续维护独立业务规则，退役条件由实际客户端版本覆盖决定。

## 5. 管理与使用入口

| 入口 | 责任 |
|------|------|
| Admin Skills | 管理系统和公共技能、可见性、排序及执行模板 |
| Skill Agent | 使用 AI 创建、编辑、预览和打包个人技能 |
| Desktop 技能栏 | 发现当前用户可见技能并在对话中触发 |
| 导入与导出 | 在账户技能和 `SKILL.md` 之间转换 |
| 海鲜市场 | 发布和获取版本化技能包，不直接替代账户技能权限 |

旧 `/prompts` 页面不再承担配置入口。路由兼容只能重定向或只读展示，不允许管理员以为修改成功但执行层读取不到。

## 6. SKILL.md 交换契约

`SKILL.md` 用于跨平台交换技能名称、描述和指令正文。平台扩展字段应保持可选，外部工具忽略扩展字段后仍能理解核心用途。

导入时必须：

- 校验 front matter、正文大小和必要字段。
- 生成或校验账户内唯一 `skillKey`，冲突时不静默覆盖。
- 把所有权固定为当前用户，忽略文件伪造的内部身份字段。
- 拒绝危险路径、超大附件和不受支持的包结构。
- 返回解析错误的具体位置，不把无效文件保存成可执行技能。

导出时不得包含访问密钥、内部用户 ID、私有模型凭据和运行日志。ZIP 包中的 README、示例和资源只作为辅助内容，`SKILL.md` 仍是默认入口。

## 7. 执行与安全

- 执行层通过 `ISkillService` 查找当前用户可见技能。
- 系统与公共技能的编辑需要管理权限，个人技能只能由所有者修改。
- 项目内置技能优先于远程市场同名技能，防止仓库规则被覆盖。
- 技能文本是模型输入，不自动获得文件、网络或平台管理权限。
- 任何工具调用仍经过 Agent 执行器和权限边界裁决。
- 技能执行和导入错误应可观测，但日志不得输出完整私有提示词或凭据。

## 9. 验收标准

- Admin 修改技能后，后端执行与客户端读取同一条 `skills` 数据。
- 旧提示词迁移重复执行不会覆盖或复制目标技能。
- 个人、公共和系统技能的读取与编辑权限符合所有权。
- `SKILL.md` 可导入、编辑、导出并在支持核心字段的平台使用。
- 同名市场技能不能覆盖项目内置技能或他人的账户技能。
- 旧接口只做适配，不再向 `promptstages` 写入新配置。
