# 总裁面板 · 设计

> **版本**：v3.1 | **日期**：2026-07-17 | **状态**：已落地

## 管理摘要

- **问题**：管理层需要快速理解团队协作、AI 使用、成本与风险，但原始运行日志和周报数据分散且短期数据会过期。
- **决策**：总裁面板只消费已聚合、可解释的数据快照；周报业务能力由 Report Agent 负责，面板不复制周报生成和团队管理逻辑。
- **价值**：为管理者提供团队级趋势、使用与成本的统一观察入口，同时保持指标来源可追溯。

## 产品边界

| 产品 | 责任 |
|---|---|
| 总裁面板 | 汇总指标、趋势、排行、成本和跨团队洞察 |
| Report Agent | 团队、成员、日报、周报、AI 汇总与周报互动 |
| 工作流引擎 | 采集外部数据、按计划生成或补充统计素材 |
| LLM Gateway 与日志 | 提供模型调用、成本和运行原始事实 |

总裁面板是观察和决策界面，不承担编辑周报、直接修改模型配置或绕过领域权限的职责。

## 核心决策

1. **聚合优先**：面板读取按时间和团队聚合的事实，不能依赖短 TTL 原始日志直接拼长期趋势。
2. **指标可解释**：每项指标应能追溯到数据来源、统计周期和筛选条件；无数据时显示未知而非补零。
3. **领域数据分治**：团队和周报事实仍由 Report Agent 权威维护，面板只消费稳定读模型。
4. **权限先行**：团队范围、成本可见性和排行榜范围均由服务端根据用户身份裁决。

## 信息架构

| 维度 | 管理问题 |
|---|---|
| 全局概览 | 当前周期的使用、活动和风险是否异常 |
| 团队洞察 | 哪些团队投入、产出或阻塞显著变化 |
| Agent 使用 | 智能体能力是否被采用，失败是否集中 |
| 成本中心 | 成本、模型与调用量的变化是否可解释 |
| 外部协作 | 外部数据源和自动化采集是否健康 |

展示顺序遵循“结论先于明细”。趋势、排行和异常状态均应能进入对应团队或数据明细，而不是把原始日志直接堆在首页。

## 数据与刷新

面板读取的主要事实包括团队周报聚合、模型调用与成本聚合、智能体运行统计和领域事件摘要。需要长期趋势的指标必须通过周期性快照或聚合表保存；原始日志仅用于追溯和诊断。

工作流可为团队补充外部项目、代码提交或质量数据，但采集失败必须标注数据新鲜度与来源，不能将旧快照表示为实时数据。

## 风险与验收

- 原始日志 TTL、缺失用户归属或未映射外部身份会影响趋势可信度，界面必须提供时间范围和数据完整性说明。
- 成本和排行榜属于敏感数据，服务端应先完成权限与团队过滤。
- 指标变更需同时更新口径说明和测试数据，避免同名指标在不同页面采用不同算法。
