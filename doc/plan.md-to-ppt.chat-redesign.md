# MD 转 PPT 对话工作台收尾 · 计划

> **版本**：v2.0 | **日期**：2026-07-17 | **状态**：规划中

## 目标

把已经落地的“对话、大纲、生成、局部修改和 artifact 预览”闭环补成可恢复、可直接编辑、可安全发布的完整工作台。本文只保留尚未完成的增量；现有生成、patch、SSE、主题和运行记录不得重做。

## 已有事实

- `MdToPptRun` 已支持 `convert`、`patch` 和 `outline`，大纲由服务端持久化。
- `md-to-ppt-agent.outline::chat` 已进入 AppCaller 注册表。
- 前端已有对话工作台、下一步状态栏、选区反馈和 artifact 预览。
- 生成 HTML 可以发布为网页，但预览隔离与多轮状态恢复仍要按 `debt.md-to-ppt.md` 收口。

## 未完成范围

### P1：大纲右侧直接编辑

- 大纲状态下，右侧显示每页可编辑卡片，支持标题、要点、增删页和排序。
- 编辑立即写入服务端权威状态；左侧对话只承担解释、AI 修改和确认动作。
- “让 AI 改”必须返回结构化大纲 patch，并保留用户手工修改。
- 页数调整后重新校验页号、当前选中页和后续生成参数。

### P2：显式状态机与刷新恢复

- 状态至少区分 `input`、`clarifying`、`outline`、`generating`、`preview`、`patching` 和 `failed`。
- 刷新后从运行记录恢复当前状态、大纲、消息、HTML 版本和选中页，不依赖 `sessionStorage` 作为事实源。
- 生成或 patch 断线后可以重新附着 SSE；服务端完成状态不能被旧浏览器状态覆盖。
- 用户未确认大纲时保持在大纲阶段，不自动继续生成。

### P3：澄清问卷

- 输入歧义较高时输出 2 至 5 个可快速选择的问题；信息充分时直接给大纲。
- 回答与原始输入共同进入大纲生成，问题、答案和跳过动作可恢复。
- 问卷不能成为固定阻塞步骤，用户始终可以跳过并继续。

### P4：编辑与模板闭环

- 修复编辑模式中不可选、字号调整异常和局部修改误伤其他页面的问题。
- 用户确认的模板可发布到海鲜市场，并保留作者、预览、来源和版本。
- 官方模板扩容只增加经过视觉验收的模板，不把模板数量当完成标准。
- 套图背景保持候选能力；没有稳定的裁切、对比度和版权策略前不进入默认流程。

### P5：隔离预览与验收

- LLM 生成 HTML 不再以应用同源的 `allow-scripts allow-same-origin` 组合运行。
- 预览使用独立源或 opaque origin，并验证脚本不能读取认证信息或调用 MAP 私有 API。
- 覆盖输入、澄清、大纲编辑、刷新恢复、生成、patch、失败重试、发布和移动端。
- 双主题取证；大纲和生成期间持续展示阶段、进度和可取消状态。

## 验收门

1. 用户可以只编辑右侧大纲，不必通过自然语言完成每一次小修改。
2. 任意阶段刷新后状态、消息、大纲和 artifact 都不丢失，也不会自动跨阶段。
3. 同一页连续 patch 可以撤销或追溯，其他页面内容不被意外改写。
4. 生成 HTML 无法访问 MAP token、storage 或鉴权 API。
5. 失败有阶段、原因和重试入口，不出现静止等待超过两秒的空白状态。
6. 旧运行记录仍可打开，现有 convert、patch 和发布接口不回归。
