# Remotion 视频创作提示词模板

## 简介

此模板旨在帮助用户清晰地向 AI 表达他们使用 Remotion 创建视频的需求。一个结构良好、信息完整的提示词是 AI 理解任务并提供高质量、可执行代码和指导的关键。

## 如何使用

请复制以下模板,并用您项目的具体信息填充 `[占位符]` 部分。提供的信息越详细、越具体,AI 生成的结果就越准确、越有用。

---

### **填写示例**

**1. 项目核心目标:** 我想为 GitHub 用户生成一个个性化的年度代码回顾视频,时长约 20 秒。

**2. 视频规格:** 20秒, 1280x720, 30 FPS。

**3. 视频场景分解:**
    *   **场景 1 (0-4秒):** 显示用户的 GitHub 头像和 "{{username}}'s 2025 Year in Code" 标题。
    *   **场景 2 (4-12秒):** 一个动态增长的条形图,展示过去12个月的提交活动。
    *   **场景 3 (12-18秒):** 循环展示该用户贡献最多的3个仓库的图标。
    *   **场景 4 (18-20秒):** 结尾画面,显示一个大的年度总提交数和一个感谢信息。

**4. 动态数据与参数 (Input Props):**
    *   `username`: "JohnDoe"
    *   `avatarUrl`: "https://github.com/johndoe.png"
    *   `commitHistory`: `[30, 45, 60, 50, 70, 80, 90, 65, 55, 75, 85, 100]`
    *   `topRepos`: `[{name: 'remotion', org: 'remotion-dev'}, {name: 'react', org: 'facebook'}, {name: 'typescript', org: 'microsoft'}]`
    *   `totalCommits`: 780

**5. 资产文件:** 无,所有内容均为程序化生成或从 URL 加载。

**6. 技术与环境上下文:** 我正在构建一个 Next.js 应用,希望将其作为一项功能集成。我需要一个可以为大量用户生成视频的无服务器渲染方案。

**7. 期望的输出:** 请提供完整的代码、项目结构建议,并重点解释如何设置 AWS Lambda 进行无服务器渲染。
