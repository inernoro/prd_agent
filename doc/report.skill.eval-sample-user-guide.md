## 1. 功能概览与页面结构

周报管理 Agent 是一个完整的团队周报管理系统，支持 AI 自动生成周报内容、多平台数据采集、团队审阅协作等功能。

### 页面结构：3 个主标签页

| 标签页 | 可见范围 | 核心功能 |
|--------|----------|----------|
| **周报** | 所有用户 | 我的周报列表、每日打卡入口、本周数据概览 |
| **团队** | 仅负责人/副负责人 | 成员提交状态面板、团队汇总、趋势分析 |
| **设置** | 所有人（个人区域）+ 管理员（管理区域） | 数据源配置、模板管理、团队管理 |

### 核心模块一览

| 模块 | 说明 |
|------|------|
| 团队管理 | 创建团队、设置层级关系、配置可见性 |
| 成员管理 | 添加/移除成员、设定角色、多平台身份映射 |
| 模板配置 | 系统预置模板 + 自定义模板、多种板块类型 |
| 数据源 | 团队级 Git 仓库 + 个人级多平台数据源 |
| 每日打卡 | 按分类记录工作内容，支持自定义标签和时长 |
| 周报生成 | 选择团队+模板+周次，AI 一键生成内容 |
| 审阅流程 | 提交 -> 审阅/退回，含计划对比功能 |
| 团队总结 | AI 聚合成员周报生成团队级摘要 |
| 评论协作 | 按章节评论，支持回复线程 |
| 导出与趋势 | Markdown 导出、个人/团队历史趋势图 |

## 2. 权限与角色说明

### 2.1 系统权限（5 项）

需要系统管理员在「用户管理 -> 角色管理」中分配：

| 权限标识 | 说明 | 适用人群 |
|----------|------|----------|
| `report-agent.use` | 基础访问（查看入口、写周报、打卡） | 所有需要使用周报的用户 |
| `report-agent.view.all` | 跨团队查看所有周报 | 高级管理员 |
| `report-agent.team.manage` | 创建/编辑/删除团队 | 管理员 |
| `report-agent.template.manage` | 管理周报模板 | 管理员 |
| `report-agent.datasource.manage` | 管理团队级数据源 | 管理员 |

### 2.2 团队角色（3 种）

| 角色 | 标识 | 权限范围 |
|------|------|----------|
| 负责人 | `leader` | 审阅/退回周报、标记请假、查看团队面板、生成团队汇总 |
| 副负责人 | `deputy` | 与负责人权限相同 |
| 成员 | `member` | 编写和提交自己的周报、每日打卡 |

### 2.3 关键规则

- 一个用户可以同时属于多个团队，每个团队各一份独立周报
- 没有 `report-agent.use` 权限的用户无法看到侧边栏的周报入口
- `report-agent.view.all` 可以跨越团队可见性限制查看任何人的周报

## 3. 管理员首次搭建

> 以下步骤存在严格的先后依赖关系，请按顺序执行。

### 步骤 1：分配权限

**路径**：侧边栏 -> 用户管理 -> 角色管理

1. 找到或创建一个管理角色（如"周报管理员"）
2. 勾选全部 5 项周报权限
3. 将该角色分配给管理员用户
4. 对普通用户，分配包含 `report-agent.use` 的角色即可
5. 刷新页面，确认侧边栏出现周报入口

### 步骤 2：初始化系统模板

**路径**：周报页面 -> 设置 -> 模板管理

1. 进入「设置」标签页
2. 点击「模板管理」卡片
3. 若列表为空，点击「初始化预置模板」按钮
4. 系统将创建 3 个预置模板（详见第 6 节）
5. 确认模板列表中至少显示一个模板

### 步骤 3：创建团队

**路径**：周报页面 -> 设置 -> 团队管理

1. 点击「新建团队」
2. 填写团队名称、选择负责人、可选填写描述
3. 设置周报可见性和自动提交计划
4. 保存后确认负责人已自动成为团队成员

### 步骤 4：添加成员

**路径**：团队管理 -> 选择团队 -> 成员管理

1. 点击「添加成员」
2. 从用户列表中选择用户
3. 设定角色和岗位
4. 保存

### 步骤 5（可选）：配置数据源

**路径**：设置 -> 团队数据源 或 设置 -> 我的数据源

1. 添加团队级 Git 仓库或个人级多平台数据源
2. 配置用户映射
3. 测试连接并同步数据

## 4. 团队管理

### 4.1 创建团队

**路径**：设置 -> 团队管理 -> 新建团队

1. 点击「新建团队」按钮
2. 填写以下字段：

| 字段 | 必填 | 说明 |
|------|------|------|
| 团队名称 | 是 | 如"产品研发组" |
| 负责人 | 是 | 从用户列表选择，该用户自动加入团队 |
| 描述 | 否 | 团队简介 |
| 父团队 | 否 | 选择上级团队，形成树形层级 |
| 周报可见性 | 否 | `all_members`（所有成员互相可见）或 `leaders_only`（仅负责人可见） |
| 自动提交计划 | 否 | 格式如 `friday-18:00`，到时间自动提交草稿 |
| 自定义打卡标签 | 否 | 输入标签名按 Enter 添加，如"需求评审"、"代码复查" |

3. 点击保存

### 4.2 编辑团队

1. 在团队列表中点击目标团队
2. 修改需要变更的字段
3. 点击保存

### 4.3 删除团队

1. 在团队详情中点击「删除」
2. 确认删除操作

> 注意：团队下存在周报记录时无法删除，需先清理所有关联周报。

### 4.4 团队层级

通过设置「父团队」字段可以构建团队的层级关系。子团队继承父团队的管理结构，方便大型组织管理多个团队。

## 5. 成员管理

### 5.1 添加成员

**路径**：团队管理 -> 选择团队 -> 添加成员

1. 点击「添加成员」按钮
2. 从用户下拉列表中选择用户
3. 设置角色：`member`（成员）、`leader`（负责人）或 `deputy`（副负责人）
4. 可选填写岗位名称（如"前端开发"、"产品经理"）
5. 点击保存

> 注意：同一用户不能重复添加到同一团队；被添加用户需在系统中已激活。

### 5.2 移除成员

1. 在成员列表中找到目标成员
2. 点击移除按钮
3. 确认操作

### 5.3 修改成员角色与岗位

1. 在成员列表中点击目标成员的编辑按钮
2. 修改角色或岗位
3. 保存

### 5.4 多平台身份映射

身份映射用于将成员在不同平台上的账号关联到系统用户，实现跨平台数据自动归集。

**配置方法**：

1. 在成员管理中找到目标成员
2. 点击「身份映射」编辑按钮
3. 按平台填写对应的账号标识：

| 平台 | 映射值示例 |
|------|-----------|
| GitHub | `zhangsan`（用户名） |
| GitLab | `zhangsan`（用户名） |
| Gitee | `zhangsan`（用户名） |
| 语雀 (Yuque) | `zhangsan`（用户名） |
| TAPD | `zhangsan@company.com`（邮箱） |
| Jira | `zhangsan@company.com`（邮箱） |

4. 保存

映射配置后，从对应平台采集的数据会自动关联到该成员的周报统计中。

## 6. 模板配置

### 6.1 系统预置模板

系统提供 3 个预置模板，通过「初始化预置模板」按钮一键创建：

| 模板 | 标识 Key | 适用场景 |
|------|----------|----------|
| 研发通用 (Dev-General) | `dev-general` | 研发团队，含代码统计和技术产出板块 |
| 产品通用 (Product-General) | `product-general` | 产品团队，侧重需求进展和用户反馈 |
| 极简模式 (Minimal) | `minimal` | 快速填写，仅含本周完成和下周计划 |

系统模板不可删除，但可以编辑其名称、描述和章节内容。初始化接口幂等，重复调用不会创建重复模板。

### 6.2 创建自定义模板

**路径**：设置 -> 模板管理 -> 新建模板

1. 点击「新建模板」
2. 填写模板名称和描述
3. 添加章节，每个章节需配置以下属性：

**板块类型 (sectionType)**：

| 类型 | 说明 |
|------|------|
| `auto-stats` | 自动统计数据（如提交数、任务数），系统自动填充 |
| `auto-list` | 自动列表（如提交记录列表），系统从数据源拉取 |
| `manual-list` | 手动列表，用户逐条填写 |
| `free-text` | 自由文本，用户自由输入 |

**输入类型 (inputType)**：

| 类型 | 说明 |
|------|------|
| `bullet-list` | 逐条列举（最常用） |
| `rich-text` | 富文本长文本输入 |
| `key-value` | 键值对形式 |
| `progress-table` | 进度表格，含百分比 |

4. 设置各章节的排序、是否必填、最大条数等
5. 可选绑定特定团队或岗位
6. 可勾选「设为默认模板」
7. 保存

### 6.3 编辑和删除模板

- 编辑：点击模板进入详情，修改后保存
- 删除：仅自定义模板可删除，系统预置模板不可删除
- 排序：支持拖拽调整章节顺序

## 7. 数据源管理

周报系统支持两级数据源：团队级和个人级。

### 7.1 两级数据源对比

| 维度 | 团队数据源 | 个人数据源 |
|------|-----------|-----------|
| 配置入口 | 设置 -> 团队数据源 | 设置 -> 我的数据源 |
| 权限要求 | `datasource.manage` 或团队负责人 | `report-agent.use`（所有用户） |
| 数据归属 | 整个团队共享 | 仅配置者本人 |
| 支持类型 | Git / SVN | GitHub / GitLab / Gitee / 语雀 / TAPD / Jira |
| 用途 | 团队统计和活动概览 | 个人打卡面板和周报 AI 填充 |

### 7.2 配置团队数据源

**路径**：设置 -> 团队数据源

1. 点击「新增数据源」
2. 填写以下信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 名称 | 是 | 如"前端仓库"、"后端仓库" |
| 类型 | 是 | `git` 或 `svn` |
| 仓库地址 | 是 | 完整 URL，如 `https://github.com/org/repo` |
| 访问令牌 | 是 | Personal Access Token，保存后脱敏显示 |
| 分支过滤 | 否 | 逗号分隔，如 `main,develop`，不填则采集全部分支 |
| 轮询间隔 | 否 | 最小 10 分钟，默认 60 分钟 |
| 用户映射 | 否 | Git 提交邮箱 -> 系统用户 ID 的映射关系 |

3. 点击「测试连接」验证配置是否正确
4. 保存数据源
5. 点击「同步」手动拉取数据

**用户映射**是团队数据源的核心配置。格式为 JSON 键值对：

```
{
  "zhangsan@company.com": "系统用户ID",
  "lisi@gmail.com": "系统用户ID"
}
```

未配置映射的 Git 提交将无法归属到具体成员。

### 7.3 配置个人数据源

**路径**：设置 -> 我的数据源

1. 点击「新增数据源」
2. 选择平台类型（GitHub / GitLab / Gitee / 语雀 / TAPD / Jira）
3. 填写显示名称
4. 根据平台填写配置信息：
   - **GitHub/GitLab/Gitee**：仓库 URL + 访问令牌
   - **语雀**：空间 ID + Token
   - **TAPD/Jira**：项目标识 + 访问凭据
5. 点击「测试连接」
6. 保存后点击「同步」拉取本周数据

**各平台令牌获取方式**：

| 平台 | 获取路径 | 所需权限 |
|------|----------|----------|
| GitHub | Settings -> Developer Settings -> Personal Access Tokens | `repo` |
| GitLab | Preferences -> Access Tokens | `read_repository` |
| Gitee | 设置 -> 私人令牌 | `projects` |
| 语雀 | 个人设置 -> 开发者 -> Token | 读取权限 |

### 7.4 数据源管理操作

| 操作 | 说明 |
|------|------|
| 测试连接 | 验证令牌和 URL 是否有效 |
| 手动同步 | 立即拉取最新数据，返回同步的提交数 |
| 启用/禁用 | 控制是否参与自动轮询 |
| 查看提交 | 查看已同步的 Git 提交记录列表 |
| 编辑 | 修改配置信息（令牌可选更新） |
| 删除 | 删除数据源及其已同步的数据 |

## 8. 每日打卡

每日打卡是轻量级的工作记录工具，记录的内容会在 AI 生成周报时自动汇总。

### 8.1 记录打卡

**路径**：周报标签页 -> 「打卡」按钮

1. 点击「打卡」进入打卡面板
2. 在输入框中填写工作内容
3. 选择分类标签
4. 可选填写时长（分钟）
5. 可选选择自定义标签（团队配置的额外标签）
6. 按 Enter 或点击发送按钮提交
7. 记录显示在下方时间线中

### 8.2 分类标签

系统预置 6 个固定分类，不可修改或删除：

| 分类 | 标识 | 适用场景 |
|------|------|----------|
| 开发 | `development` | 编码、调试、代码审查 |
| 会议 | `meeting` | 各类会议 |
| 沟通 | `communication` | 跨团队沟通、需求对接 |
| 文档 | `documentation` | 文档编写与维护 |
| 测试 | `testing` | 测试相关工作 |
| 其他 | `other` | 不属于以上分类的工作 |

### 8.3 自定义标签

团队管理员可以在「团队管理 -> 编辑团队」中配置自定义打卡标签。这些标签作为分类的补充，用户打卡时可以同时选择一个分类和多个自定义标签。

### 8.4 时长追踪

每条打卡记录可以填写 `durationMinutes`（耗时分钟数），用于统计各分类的时间分布。

### 8.5 数据源集成

打卡面板会自动展示从个人数据源同步的 Git 提交记录：
- 未记录的提交以提示形式出现
- 点击单条提交旁的「+」可快速添加为打卡记录
- 点击「一键补充」可批量添加所有未记录的提交

### 8.6 存储规则

- 每个用户每天一条记录（`userId + date` 唯一）
- 保存为 Upsert 操作：同一天重复保存会覆盖已有内容
- 支持日期切换，可补填历史日期的打卡

## 9. 周报编写与提交

### 9.1 创建周报

**路径**：周报标签页 -> 「写周报」按钮

1. 点击「写周报」按钮
2. 在弹窗中选择：
   - **团队**：若属于多个团队，选择目标团队
   - **模板**：选择周报模板（团队绑定模板时会自动选中）
   - **周次**：默认当前周，可选择历史周次进行补填
3. 点击「开始编写周报」

> 唯一约束：每个用户在每个团队的每一周只能创建一份周报。重复创建会返回错误。

### 9.2 编辑周报

1. 进入编辑器后，按模板章节逐项填写
2. 每个章节有标题和引导说明
3. 根据输入类型进行填写（列表、文本、键值对或进度表）
4. 右上角显示完成进度
5. 随时点击「保存」存为草稿

### 9.3 AI 自动生成

**路径**：编辑周报时 -> 「AI 填充」按钮

1. 确保周报处于可编辑状态（草稿/已退回/逾期）
2. 点击「AI 填充」按钮
3. AI 综合以下数据自动生成内容：
   - 本周每日打卡记录
   - 个人数据源的 Git 提交
   - 系统内活动数据（缺陷提交、文档编辑、创意产出等）
4. 生成的内容会覆盖当前所有章节
5. 人工审阅和调整 AI 生成的内容
6. 确认无误后保存或提交

> 注意：AI 填充会覆盖现有内容。数据越丰富（打卡记录 + 提交记录），生成质量越高。

### 9.4 提交周报

1. 确认所有必填章节已填写
2. 点击「提交」按钮
3. 状态从「草稿」变为「已提交」
4. 提交后不可再编辑，除非被负责人退回

### 9.5 周报状态流转

```
未开始 (not-started) --创建--> 草稿 (draft) --提交--> 已提交 (submitted)
                                                        |
                                              +---------+---------+
                                              |                   |
                                      已审阅 (reviewed)    已退回 (returned) --修改后重新提交--> 已提交
```

特殊状态：
- **逾期 (overdue)**：超过自动提交时间仍未提交
- **请假 (vacation)**：由负责人手动标记

### 9.6 各状态下的可用操作

| 状态 | 本人可做 | 负责人可做 |
|------|----------|-----------|
| 未开始 | 创建周报 | 标记请假 |
| 草稿 | 编辑、保存、提交、删除 | 标记请假（自动删除草稿） |
| 已提交 | 只读 | 审阅通过、退回、添加评论 |
| 已审阅 | 只读 | 只读 |
| 已退回 | 编辑、重新提交 | -- |
| 请假 | 只读 | 取消请假 |
| 逾期 | 编辑、提交 | -- |

### 9.7 统计快照

周报提交时，系统会自动生成一份统计快照（statsSnapshot），记录提交时刻的各项数据指标。该快照不可被后续数据变更影响，确保审阅时看到的数据与提交时一致。

## 10. 周报审阅

### 10.1 审阅流程

**路径**：团队标签页 -> 点击已提交的成员周报

1. 在团队面板中找到状态为「已提交」的成员
2. 点击进入周报详情
3. 阅读各章节内容
4. 可在各章节下方添加评论反馈
5. 执行操作：
   - 「审阅通过」：状态变为「已审阅」（终态，不可撤销）
   - 「退回」：弹窗填写退回原因，状态变为「已退回」

### 10.2 退回与重新提交

1. 负责人退回周报时需填写退回原因
2. 用户收到退回通知后，可以修改周报内容
3. 修改完成后重新提交
4. 负责人再次审阅

### 10.3 计划对比

**路径**：周报详情页 -> 「计划对比」标签

系统自动对比上周计划与本周实际完成情况：

| 对比项 | 数据来源 |
|--------|----------|
| 上周计划 | 上一周周报中的"下周计划"章节 |
| 本周实际 | 本周周报中的"本周完成"章节 |

若上周没有周报数据，计划对比页面会提示无数据。

### 10.4 标记请假

**路径**：团队标签页 -> 成员卡片 -> 标记请假

1. 在成员状态网格中找到请假的成员
2. 点击「标记请假」
3. 可选填写请假原因
4. 系统创建一条 `vacation` 状态的特殊周报

规则：
- 成员本周有草稿/逾期周报时，草稿会被自动删除
- 成员本周已提交或已审阅的周报时，不能标记请假
- 取消请假会删除 vacation 记录，成员状态恢复为「未开始」

### 11.2 AI 团队总结

**路径**：团队标签页 -> 「一键汇总」按钮

1. 确保团队成员已提交周报
2. 点击「一键汇总」
3. AI 聚合所有已提交/已审阅的周报，生成结构化汇总
4. 汇总包含多个段落（整体进展、重要成果、风险与问题等）

规则：
- 仅负责人/副负责人或拥有 `view.all` 权限者可生成
- 至少需要一份已提交的周报
- 每个团队每周只有一份汇总，重新生成会覆盖旧版本

| 打卡天数 | 每周有打卡记录的天数 |

### 11.4 团队趋势

展示最近 12 周的团队整体数据：

| 指标 | 说明 |

| 已审阅数 | 已审阅通过的成员数 |
| 逾期数 | 逾期未提交的成员数 |
| 提交率 | 已提交 / 总人数 的百分比 |
| 团队提交数 | 团队所有成员的 Git 提交总数 |

## 13. 导出功能

### 13.1 导出个人周报

**路径**：周报详情页 -> 右上角「导出」按钮

- 格式：Markdown (.md)

- 内容：包含周期、成员数、提交数、各汇总段落
- 权限：仅负责人/副负责人或拥有 `view.all` 权限者

## 14. 常见问题

**Q1：一个用户可以属于多个团队吗？**

可以。每个团队独立管理，用户在每个团队中各有一份周报。一个用户在 A 团队的周报和 B 团队的周报互不影响。

**Q2：已提交的周报还能修改吗？**

不能直接修改。需要团队负责人先执行「退回」操作，用户收到退回后才能重新编辑和提交。

**Q3：已审阅的周报能退回吗？**

不能。审阅通过是终态操作，不可撤销。请在审阅前仔细检查内容。

**Q4：同一周能创建多份周报吗？**

在同一个团队内不能。每个用户在每个团队的每一周只能有一份周报（唯一约束：userId + teamId + weekYear + weekNumber）。但如果属于多个团队，每个团队各一份。

**Q5：AI 生成的周报内容会覆盖我手动填写的内容吗？**

会。AI 填充会覆盖当前所有章节的内容。建议在手动填写前先使用 AI 填充，然后在 AI 生成的内容基础上进行修改。

**Q6：团队数据源和个人数据源有什么区别？**

团队数据源由管理员配置，数据属于整个团队，用于团队统计面板；个人数据源由用户自行配置，数据仅属于个人，主要用于打卡面板和 AI 周报生成。两者可以同时使用。

**Q7：系统预置模板可以删除吗？**

不可以。系统预置模板标记为 `isSystem = true`，不允许删除，但可以编辑其名称、描述和章节结构。如需不同模板，建议创建自定义模板。

**Q8：打卡的 6 个固定分类可以修改吗？**

不可以。6 个系统分类（开发、会议、沟通、文档、测试、其他）是固定的。但管理员可以在「团队管理」中为团队配置额外的自定义打卡标签，作为分类的补充。

**Q9：如何补填历史周次的周报？**

创建周报时可以指定 `weekYear` 和 `weekNumber` 参数，选择目标周次即可创建历史周报。

**Q10：团队汇总可以重新生成吗？**

可以。每个团队每周只有一份汇总，重新点击「一键汇总」会覆盖之前的版本。

## 15. 故障排除

### 15.1 权限相关

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 看不到周报功能入口 | 缺少 `report-agent.use` 权限 | 管理员在角色管理中添加权限后刷新页面 |
| 设置中看不到管理区域 | 缺少对应管理权限 | 按需添加 `template.manage`、`team.manage`、`datasource.manage` |
| 看不到团队标签页 | 不是任何团队的负责人或副负责人 | 由管理员在团队中设置为 leader 或 deputy |
| 无法查看他人周报 | 非团队成员或团队可见性为 `leaders_only` | 加入团队、联系负责人、或获取 `view.all` 权限 |

### 15.2 创建与编辑相关

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 创建周报时找不到团队 | 未加入任何团队 | 联系管理员将你添加到团队 |
| 创建周报时找不到模板 | 模板未初始化或被删除 | 管理员初始化预置模板或创建新模板 |
| 提示"该周已存在周报" | 同一团队同一周已有周报 | 编辑已有周报，不要重复创建 |
| 无法编辑周报 | 状态不是草稿/已退回/逾期 | 联系负责人退回后再编辑 |
| 无法删除周报 | 状态不是草稿 | 已提交/已审阅的周报不可删除 |

### 15.3 数据源相关

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 测试连接失败 | 令牌无效或仓库 URL 格式错误 | 检查令牌权限范围和 URL 完整性 |
| 同步后没有数据 | 用户映射未配置 | 配置 Git 提交邮箱到系统用户 ID 的映射 |
| 打卡面板看不到提交记录 | 个人数据源未配置或未同步 | 在「我的数据源」中添加并手动同步 |
| 提示"不支持的数据源类型" | 类型拼写错误 | 使用正确的类型标识：`github`/`gitlab`/`gitee`/`yuque` |

### 15.4 AI 相关

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| AI 填充按钮不可用 | 周报状态不是草稿/已退回/逾期 | 确保周报处于可编辑状态 |
| AI 生成内容质量差 | 打卡记录和提交数据不足 | 先补充打卡记录，配置并同步数据源 |
| AI 生成失败 | LLM 服务异常 | 检查后台 LLM Gateway 配置和模型可用性 |
| 团队汇总生成失败 | 没有已提交的成员周报 | 至少需要一份已提交或已审阅的周报 |

### 15.5 团队管理相关

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 无法删除团队 | 团队下存在周报记录 | 先清理所有关联的周报数据 |
| 提示"该用户已是团队成员" | 重复添加同一用户 | 该用户已在团队中，无需重复添加 |
| 负责人用户不存在 | 选择的用户 ID 无效 | 确认用户在系统中存在且状态为 Active |
