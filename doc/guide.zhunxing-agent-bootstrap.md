# 准星智能体首版落地指南

> **版本**：v1.0 | **日期**：2026-06-03 | **状态**：草案

---

## 一、适用范围

本指南用于将“准星”快速落地为首版可用智能体，目标是先跑通“制度问答闭环”：

1. 能回答问题
2. 有条款依据
3. 无依据时可兜底升级

---

## 二、首版落地清单

### 2.1 必备输入

- 应用信息：`appId=zhunxing-agent`、名称“准星”
- 知识源：首批制度文件（已知至少包含《米多公司考勤管理办法》）
- 责任人：至少一位 HR 或制度管理员负责口径复核

### 2.2 通过标准

- 连续回答 20 个高频问题，且每条答案可回溯条款
- 无命中时输出标准兜底话术，不出现“编造条款”

---

## 三、注册应用（App Registry）

### 3.1 注册准星

```bash
curl -X POST http://localhost:5000/api/app-registry/apps \
  -H "Content-Type: application/json" \
  -d '{
    "appId": "zhunxing-agent",
    "appName": "准星",
    "description": "公司规范与流程问答中枢",
    "version": "1.0.0",
    "capabilities": {
      "inputTypes": ["text"],
      "outputTypes": ["text"],
      "supportsAttachments": false,
      "triggerKeywords": ["规章", "制度", "规范", "流程", "交接", "考勤", "请假"]
    },
    "endpoint": "https://your-domain/api/zhunxing/handle",
    "authType": "Bearer"
  }'
```

### 3.2 创建路由规则

```bash
curl -X POST http://localhost:5000/api/app-registry/rules \
  -H "Content-Type: application/json" \
  -d '{
    "name": "准星-规范流程问答",
    "priority": 20,
    "condition": {
      "type": "Keyword",
      "keywords": ["规章", "制度", "规范", "流程", "交接", "考勤", "请假", "旷工"]
    },
    "targetAppId": "zhunxing-agent"
  }'
```

## 四、知识录入规范（最小字段）

每条知识最少包含以下字段：

| 字段 | 示例 |
|---|---|
| docTitle | 米多公司考勤管理办法 |
| effectiveDate | 2026-02-01 |
| chapter | 8.2 |
| ruleText | 一日两次打卡，上午 9:00，下午 18:30 |
| keywords | 打卡、上下班、考勤 |
| ownerDepartment | HR |

## 六、首批 FAQ（20 条）

2. 午休是否算工时？

5. 在公司范围外打卡是否有效？  
6. 迟到和早退如何判定？  
7. 请假必须提前申请吗？  
8. 请假 40 小时以内谁审批？  
9. 请假超过 40 小时谁审批？  
10. 事假最小申请单位是多少？  
11. 单次事假最长可以请多久？  
12. 每季度和全年事假上限是多少？  
13. 婚假有几天？  
14. 产假和奖励假如何计算？  
15. 陪产假有几天？  
16. 哺乳假如何使用？  
17. 丧假按亲属关系如何区分天数？  
18. 病假期间工资如何计算？  
19. 什么情况会被认定为旷工？  
20. 旷工累计到什么程度会触发严重违纪？

## 七、兜底话术（无依据时）

```text
当前知识库未找到与你问题完全匹配的有效条款。为避免误导，先不给出结论。
建议你补充场景条件（部门、请假类型、时长、是否已有审批），
或联系对应责任部门（HR/制度管理员）进行人工确认。
```

## 八、上线后运营建议

1. 每周复盘 Top 20 高频问题，补齐盲区条款
2. 制度更新当天同步知识版本并标注生效时间
3. 对“高风险问答”（薪资、处罚、法务）开启强制复核标记
