# PRD Agent 使用教程

> **类型**：guide | **适用角色**：产品经理、测试工程师、开发人员 | **最后更新**：2026-03-19

---

## 1. 什么是 PRD Agent

PRD Agent 是一个智能需求文档解读与问答系统。上传 PRD（产品需求文档）后，即可与 AI 对话，快速理解需求细节、发现文档缺失、生成测试用例。

**适用场景**：
- 新加入项目，需要快速熟悉需求
- 评审 PRD 时，想检查是否有遗漏
- 需要从 PRD 中提取测试用例或验收标准
- 需要快速生成需求摘要给团队同步

---

## 2. 快速上手

### 步骤一：创建团队

1. 登录后进入 PRD Agent 页面
2. 点击左侧「创建团队」按钮
3. 填写团队名称，邀请团队成员

### 步骤二：上传 PRD 文档

1. 在团队空间中，点击「上传文档」
2. 支持 PDF、Word、Markdown 等格式
3. 系统自动解析文档内容并建立索引

### 步骤三：开始对话

1. 点击已上传的文档进入会话页面
2. 在输入框中提问，例如：「登录流程的异常处理逻辑是什么？」
3. AI 将基于文档内容精准回答，并标注引用位置

---

## 3. 核心功能详解

### 3.1 智能问答

直接向 AI 提问关于 PRD 的任何问题，AI 会：
- 定位到文档的相关章节
- 给出精准回答并标注来源
- 对模糊描述进行补充说明

**提问技巧**：
- 具体化：「用户登录失败后重试次数限制是多少？」优于「登录怎么做？」
- 对比类：「v2 和 v1 的权限模型有什么区别？」
- 假设类：「如果用户在支付过程中断网，系统如何处理？」

### 3.2 内容缺失检测

AI 自动扫描 PRD 文档，发现以下类型的遗漏：
- 未定义的异常处理流程
- 缺少边界条件说明
- 遗漏的用户角色或权限说明
- 缺失的非功能性需求（性能、安全等）

**使用方法**：在会话中输入「检查这份文档有哪些遗漏」或点击工具栏的「缺失检测」按钮。

### 3.3 测试用例生成

基于 PRD 内容一键生成测试用例：
- 正向流程测试
- 异常流程测试
- 边界值测试
- 兼容性测试

**使用方法**：输入「为登录模块生成测试用例」，AI 将输出结构化的测试用例表格。

### 3.4 需求摘要

快速生成 PRD 核心要点总结，适用于：
- 团队周会同步
- 新成员入职阅读
- 跨部门沟通

---

## 4. 进阶用法

### 4.1 补充文档

除主 PRD 外，可上传补充文档（UI 设计稿说明、接口文档等），AI 会综合所有文档回答问题。

### 4.2 PRD 评论

在文档的特定位置添加评论，团队成员可以看到并回复。适合异步评审场景。

### 4.3 会话归档

重要的问答会话可以归档保存，方便后续查阅。

---

## 5. 常见问题

**Q：支持哪些文档格式？**
A：PDF、Word (.docx)、Markdown (.md) 格式。

**Q：文档大小有限制吗？**
A：单个文档建议不超过 50MB，超大文档建议拆分上传。

**Q：AI 回答不准确怎么办？**
A：尝试更具体的提问方式，或检查文档是否解析完整。也可以上传补充文档提供更多上下文。

**Q：团队成员能看到我的对话吗？**
A：同一团队空间内的对话对所有成员可见，适合协作评审。
