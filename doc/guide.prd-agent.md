# PRD Agent 使用 · 指南

> **类型**：guide | **适用角色**：产品经理、测试工程师、开发人员 | **最后更新**：2026-03-19
> **预计阅读**：8 分钟 | **难度**：入门

---

## 1. 功能概览

PRD Agent 是智能需求文档解读与问答系统。上传 PRD 文档后，AI 基于文档内容精准回答问题，帮你发现需求遗漏、生成测试用例。

| 能力 | 说明 |
|------|------|
| 智能问答 | 基于 PRD 内容回答任何问题，标注引用出处 |
| 缺失检测 | 自动扫描需求文档中的逻辑漏洞和缺失描述 |
| 用例生成 | 从需求描述中提取测试用例和验收标准 |
| 需求摘要 | 快速生成 PRD 核心要点总结 |
| PRD 评论 | 在文档特定位置添加评论，支持团队异步评审 |
| 补充文档 | 上传额外文档（UI 设计稿、接口文档）丰富上下文 |

---

## 2. 操作路径

**入口**：首页 → 点击「PRD 分析师」卡片 → 进入 PRD Agent 页面

页面包含两个 Tab：
- **「PRD 对话」**（MessagesSquare 图标）— 与 AI 基于文档对话
- **「群组管理」**（Users2 图标）— 管理团队和文档

---

## 3. 完整操作流程

### 步骤 1：创建群组

1. 切换到 **「群组管理」** Tab
2. 点击 **「创建」** 按钮
3. 输入群组名称（如「v3.0 需求评审」）
4. 点击确认创建

> **Pro Tip**：群组相当于一个项目空间，建议按版本或模块创建，例如「登录模块重构」「v2.5 新增功能」。

### 步骤 2：上传 PRD 文档

1. 进入已创建的群组
2. 点击上传区域，选择文件
3. 支持格式：**PDF**、**Word (.docx)**、**Markdown (.md)**
4. 系统自动解析文档结构并建立索引（耗时取决于文档大小）

> **Pro Tip**：除了主 PRD，还可以上传补充文档（UI 标注、接口定义）。AI 会综合所有文档回答问题，覆盖面更全。

### 步骤 3：开始智能问答

1. 切换到 **「PRD 对话」** Tab
2. 在底部输入框中提问
3. AI 回答时会以流式输出逐字显示，同时标注引用的文档章节

**高效提问技巧**：

| 提问类型 | 示例 | 效果 |
|---------|------|------|
| 精确查询 | 「用户登录失败后的重试次数限制是多少？」 | 直接定位到具体描述 |
| 对比分析 | 「v2 和 v1 的权限模型有什么区别？」 | 跨章节对比总结 |
| 假设推演 | 「如果用户在支付中途断网，系统如何处理？」 | 发现潜在遗漏 |
| 缺失检测 | 「这份文档有哪些遗漏的异常处理流程？」 | 自动扫描逻辑漏洞 |
| 用例生成 | 「为登录模块生成测试用例」 | 输出结构化用例表格 |

### 步骤 4：使用 PRD 评论

1. 在对话中找到需要标注的需求点
2. 添加评论，团队成员可以看到并回复
3. 适合异步评审场景——产品写完 PRD，开发和测试各自留评审意见

## 4. 实用场景

### 场景 A：新人快速熟悉项目

```
提问：「请用 500 字总结这份 PRD 的核心功能和目标用户」
提问：「列出所有涉及的用户角色及其权限差异」
提问：「画出主流程的状态转换图」
```

### 场景 B：测试用例提取

```
提问：「为第 3.2 节的登录流程生成测试用例，包含正向、异常和边界场景」
提问：「列出所有需要进行性能测试的接口」
```

### 场景 C：需求评审检查

```
提问：「检查这份文档是否遗漏了以下方面：异常处理、并发控制、数据权限、缓存策略」
提问：「哪些需求描述存在歧义或自相矛盾？」
```

## 5. 常见问题

**Q：文档很大，解析会很慢吗？**
A：解析时间取决于文档大小，通常 10 页以内的 PDF 在 10 秒内完成。解析过程中有进度提示，不会出现空白等待。

**Q：AI 回答不准确怎么办？**
A：尝试更具体的提问方式。如果文档本身缺少相关描述，AI 会说明「文档中未找到相关信息」，这正是发现遗漏的信号。

**Q：可以同时分析多份文档吗？**
A：可以。在同一群组中上传多份文档，AI 会综合所有文档内容回答。

**Q：对话记录会保存吗？**
A：是的，所有对话自动保存。重要会话还可以归档，方便后续查阅。
