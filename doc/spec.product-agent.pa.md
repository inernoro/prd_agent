# 私人执行助理（PA Agent）产品方案

> **版本**：v1.1 | **日期**：2026-05-10 | **状态**：开发中 | **appKey**：pa-agent

面向个人与知识工作者的对话式执行助理：把模糊目标拆成可执行步骤，用四象限排序，并在多会话中持续跟进。本文档可作为独立 Markdown 下载与评审材料。

---

## 一、产品定位

**一句话**：像主流 AI 聊天一样自然对话，但输出能**落盘为任务**，并在一套**艾森豪威尔四象限看板**上管理优先级与进度。

**与「纯聊天」的差异**：

| 维度 | 纯聊天 | PA Agent |
|------|--------|----------|
| 产出 | 一次性建议 | 可持久化的任务与子步骤 |
| 优先级 | 靠用户自己记 | 模型建议象限 + 用户调整 |
| 上下文 | 单线程易混 | 多会话 + 会话标题与摘要 |
| 附件 | 常需复制粘贴 | 支持上传文档抽取正文参与对话 |

**边界（当前阶段）**：不替代项目协作工具（Jira 等）的全员流程；聚焦**个人或小型闭环**的规划与执行辅助。

---

## 二、目标用户与场景

**用户画像**

- 管理者、产品经理、独立开发者：事项多、打断多，需要快速「想清楚 + 排优先级 + 知道下一步」。
- 备考、求职、副业规划：目标模糊，需要 MECE 拆解与阶段推进。

**典型场景**

1. 用户说「下周要交付 X，帮我拆一下」：得到子任务列表 + 象限建议，**明确意图**时自动写入看板。
2. 用户闲聊里提到「最近有个会要准备材料」：**潜在任务**以「加入看板」按钮呈现，由用户确认。
3. 用户上传 PDF/Word：抽取文本进入上下文，在此基础上继续拆解与答疑。

## 三、核心能力（Phase 1 已具备）

### 3.1 对话与流式体验

- SSE 流式回复，避免长时间空白等待。
- 用户消息与助手消息分区展示；空状态提供快捷意图示例（如「拆解任务」）。
- 支持附件上传（后端抽取文本，对话中携带 `attachedText` / `attachedFileName`）。

### 3.2 多会话

- 会话列表：标题、最后一条预览、消息条数、更新时间。
- 新建、重命名、删除会话；进入页面时优先同步服务端会话，失败时降级为本地会话 ID，避免页面卡死。

### 3.3 智能任务识别（显式 / 隐式）

模型在回复末尾按约定输出 JSON 块（由后端解析，前端展示前剥离）：

- **auto（明确任务）**：系统自动创建 `PaTask` 并推送 SSE `task` 事件；前端展示「已加入看板」类反馈。
- **suggest（潜在任务）**：仅创建建议态展示，用户点击确认后再落库。
- **普通对话**：不输出任务 JSON。

任务字段包含：标题、象限（Q1–Q4）、理由、子步骤列表、可选的 `sessionTitle`（用于首次自动命名会话）。

### 3.4 四象限任务看板

- Q1 紧急重要 / Q2 重要不紧急 / Q3 紧急不重要 / Q4 不紧急不重要。
- 任务卡片展示子任务进度；支持完成、归档等操作（与后端状态一致）。

### 3.5 权限与模型

- 权限键：`pa-agent.use`；内置角色 operator / viewer / agent_tester 已包含。
- LLM 经 `ILlmGateway`，`AppCaller`：`pa-agent.chat::chat`；启动时若未绑定模型组，可自动绑定首个可用非视频 chat 模型组，降低「未注册 / 未绑定」导致的不可用。

## 四、信息架构与关键流程

### 4.1 页面结构（管理端）

- **左侧**：会话列表 + 底部入口「任务看板」。
- **中间**：当前会话的聊天区；输入区支持文本、附件、拖拽。
- **看板**：从侧栏切换，全屏或主区展示四象限。

### 4.2 端到端数据流（简述）

1. 用户发送消息（可选附件）→ API 组装 System Prompt（MECE + 四象限 + 任务识别规则）→ 流式 LLM。
2. 流结束后解析 JSON → 写 `PaTask` / 更新 `PaSession` 元数据 → SSE 推送任务事件。
3. 前端刷新会话列表、看板列表；聊天区仅展示剥离 JSON 后的正文。

## 五、数据模型要点（概念层）

| 概念 | 说明 |
|------|------|
| PaSession | 用户维度会话；含 Title、LastMessagePreview、MessageCount、时间戳 |
| PaMessage | 会话内消息持久化 |
| PaTask | 任务与子步骤、象限、状态；含置信度语义（auto / suggest） |
| PaTaskConfidence | 常量：`auto`、`suggest` |

详细字段与索引以代码与数据字典为准；后续可在 `design.pa-agent.md` 中单开技术设计。

## 六、路线图（建议）

**Phase 1（当前）**：多会话 + 流式聊天 + 附件 + 智能任务落库 + 四象限看板 + 权限与模型绑定兜底。

**Phase 2（建议）**

- 与日历 / 截止日联动；任务提醒与逾期视觉提示。
- 任务编辑、拖拽改象限、批量归档。
- 会话级「本周焦点」摘要与跨会话检索。

**Phase 3（可选）**

- 团队共享看板只读链接；导出 Markdown / PDF 周报式总结。
- 与 PRD / 缺陷 Agent 的浅集成（从文档一键生成行动项）。

---

## 七、风险与原则

- **模型遵循度**：任务 JSON 依赖提示词约束，需监控坏例并迭代 System Prompt 与解析容错。
- **成本与延迟**：长附件抽取 + 长上下文增加 token；应对单文件大小与截断策略产品化。
- **隐私**：附件内容过 LLM 前应在产品说明中明确；企业部署可考虑「仅本地抽取 + 用户确认上传」。
