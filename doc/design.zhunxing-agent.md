# 准星智能体技术设计

> **版本**：v1.0 | **日期**：2026-06-03 | **状态**：规划中

---

## 一、设计目标

为“准星”提供可快速落地且可扩展的首版架构，满足以下目标：

1. 以最小改造接入现有 App Registry 调度能力
2. 先实现“制度问答可用”，再演进为“知识库 + AI 平台能力”
3. 严格遵循“有依据才回答”的风险控制策略

---

## 二、总体架构

```text
用户问题
  -> App Registry 路由
  -> zhunxing-agent 问答服务
  -> 检索知识条款（制度/流程）
  -> 生成结构化答案（结论 + 依据 + 操作建议）
  -> 返回统一响应
```

组件划分：

1. 路由层：通过 App Registry 的规则将规范类问题转发至准星
2. 应答层：准星核心服务（检索、排序、回答生成、兜底）
3. 知识层：知识条款库（文档、章节、条款、版本、生效时间）
4. 审计层：问答日志、命中记录、风险标签

## 五、回答引擎策略

### 5.1 检索排序流程

1. 关键词召回（制度词 + 场景词）
2. 条款语义匹配（问题与条款语义相似度）
3. 生效与版本过滤（只取当前生效版本）
4. 风险规则过滤（敏感内容最小化输出）

### 5.2 生成策略

- 主模板：`结论 -> 依据 -> 操作建议 -> 风险提示`
- 多命中时按“明确性优先”返回，避免混合口径
- 无命中时返回标准兜底，不猜测、不编造

---

## 六、提示词与防幻觉约束

系统提示词关键约束：

1. 只能依据知识库中命中的条款作答
2. 必须输出至少一个引用来源
3. 缺少条件时先反问，不直接裁决
4. 涉及处罚、薪资、法务争议时提示人工确认

---

## 七、安全与合规设计

1. 文档分级入库：公开/内部/敏感/机密
2. 默认不返回原文大段内容，仅返回必要摘要和条款号
3. 敏感字段（账号、密钥、个人隐私）入库前脱敏
4. 审计日志保留：问答请求、命中文档、响应版本

---

## 八、可观测性与运营指标

核心观测指标：

- 命中率（按知识域）
- 引用覆盖率（带出处回答占比）
- 兜底率（无依据返回占比）
- 反馈通过率（用户认为“答案有帮助”的比例）

---

## 九、演进路线

### 阶段 1：规则问答

- 基于结构化知识条款，支持稳定制度问答

### 阶段 2：多文档联动

- 支持跨制度与跨部门流程联合回答

### 阶段 3：知识中枢平台化

- 抽象统一知识 schema，支持更多业务域智能体复用
