> **版本**：v4.0 | **日期**：2026-06-02 | **状态**：no-fake + 参数选择器 + 出站动作(巧思第一波) + 智能体接力(E9 配图) 已落地，下面为剩余波次技术债务
> **关联设计**：`design.agent-universe.md`

记录"智能体宇宙"主动声明的已知边界、剩余技术债务、需用户判断的分叉，避免下一次 session 没人记得。

## 一、已落地并验收

- **no-fake**：统一信封 `invoke` 只路由真实 `IAgentAdapter`，找不到报 `NO_REAL_AGENT`，绝不仿冒。注册表只登记有真实组件的 4 个（visual/literary/defect/prd）。
- **自定义智能体零代码接入**：`custom:{itemId}` 实时读库 systemPrompt 跑真实网关，新建即用。
- **参数选择器**：`GET agents/{key}/parameters` 从智能体自己原有的池拉真实可选项（尺寸/模型），`invoke` 的 `parameters` 透传到真实引擎。
- **真人体验修复**：抽屉 z-index 盖过铃铛、发送按钮等高、引用 chip、空状态预期说明、新对话按钮、思考态秒数+慢模型提示。
- **出站动作机制（巧思第一波）**：契约新增 `outboundActions[]`；选中智能体显示「智能涌现」提示；**缺陷智能体「创建缺陷」** 一键建入缺陷库（复用 `POST defects`，HTTP 实测 success:true）。L1 验收报告已归档（分享链 + verify-open exit 0）。
- **智能体接力 E9（文学→视觉 配图）**：文学产出旁「为这段配图」→ 前端编排两次 `invoke`：literary `generate_illustration`（构思插画描述）→ visual `text2img`（生图）→ 图片可一键插文档。全程进度托管在一条消息里。

## 二、已知边界（诚实记录）

| 边界 | 说明 |
|------|------|
| 尺寸非像素精确 | 当前池模型 `gpt-image-2-all`（apiyi）对尺寸松散解释（选 1344×768 实出 1254×1254），模型特性；要精确就在池里换严格支持尺寸的模型（前端零改动）。 |
| 多轮上下文 | 适配器单轮（single-shot），chat 每次提交独立一轮、不带前几轮上下文。再加工场景可接受。 |
| 创建缺陷无项目/预填 | create-defect 当前不带项目（落默认/未分组），严重度/标题未 AI 预填（标题由后端 `DefectTitleNormalizer` 归一）。见 §四 E11。 |
| 插入位置 | 文学"插入文档"走既有 追加末尾/替换原文，非光标处插入。见 §四。 |
| 配图接力较慢 | E9 串两次 LLM（文学描述 ~16s + 视觉生图 ~20s）≈ 40s，已分阶段显示进度（①构思 → ②生图），非卡死。 |
| toast 瞬时 | 出站成功 toast ~3s，自动化截图未必每次捕获；已用 HTTP 直测兜底。 |

## 五、验收标准（用户硬性要求：通过才算数）

每个增量交付前必须：后端 CDS 编译通过；真实路径验证（SSE/日志看到确实调真实 adapter）；视觉出图预览域名实测；写明自测路径与结论。涉及 UI 的走 Playwright 真人路径盯帧复验（审布局/盯流式逐字/查预期），不拿"200/有元素"交差。
