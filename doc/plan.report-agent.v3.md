# Report Agent 采集优先升级 · 计划

> **版本**：v3.1 | **日期**：2026-07-17 | **状态**：规划中

## 目标

把周报从“空模板等待填写”升级为“打开时已有可审阅草稿”。系统优先聚合可授权的数据，AI 负责归纳和表达，人只补充机器无法知道的计划、判断和感受。

本计划描述 v3 的未完成增量，不重述已经落地的团队、模板、工作流、数据源、周报状态机和海报能力。

## 原则

1. 零配置也有基础内容：先使用系统内活动，不以外部集成为前置条件。
2. 渐进增强：GitHub、TAPD 等连接只增加信息，不决定页面是否可用。
3. 证据可追：每条 AI 归纳能回到来源类别和时间范围。
4. 人工可编辑：AI 草稿不是提交结果，用户可以修正和删除。
5. 稀疏诚实：数据不足时明确说明，不编造“本周成果”。

## 阶段

### P0：采集增强与首屏预填

- 统一系统内活动的时间范围、去重和用户归属。
- 在周报编辑器首屏展示统计摘要和可编辑完成事项。
- 明确来源、采集时间和未连接数据源，不显示假成功。
- 建立同一周重复生成的幂等与人工修改保护。

### P1：管理者摘要与风险

- 团队汇总从内容拼接升级为结构化完成项、阻塞和变化。
- 风险先用确定性规则识别，再用 AI 归纳语义；两者来源可区分。
- 管理者可下钻到成员周报和原始活动，不能只看黑盒分数。

### P2：主动触达

- 支持受控 webhook 或通知渠道，在草稿可审阅和逾期时提醒。
- 通知配置按团队与用户权限管理，带频率限制、重试和停用。
- 推送只含必要摘要，敏感正文通过登录后的页面查看。

### P3：历史查询

- 支持按成员、团队、时间和主题查询历史周报与活动。
- 先做结构化筛选，再让 AI 总结结果；无证据时不回答确定结论。
- 保存查询范围和引用，支持复核。

### P4：质量闭环

- 记录草稿来源、实际模型、fallback 原因和用户编辑差异。
- 质量评估按脱敏样本和明确指标执行，不用批量付费调用试模型。
- 模型池治理继续由 `plan.report-agent.model-governance.md` 负责。

## 数据边界

- 只采集当前用户或团队授权范围内的数据。
- 周报保存来源摘要和必要引用，不复制其它业务集合全文。
- AI prompt 不包含无关附件、密钥、私有链接或跨团队信息。
- 统计快照与可编辑正文分离，避免刷新数据覆盖人工修改。

## 验收门

- 新用户在无外部连接时也能看到基于系统活动的诚实草稿或明确空态。
- 同一周重复生成不会覆盖用户已编辑内容。
- 每条归纳能看到来源类别，管理者只能查看授权团队。
- 模型失败时显示规则草稿及稳定原因，不伪装成 AI 成功。
- 风险、通知和查询各自有权限、负向测试和关闭路径。
- 移动端与宽屏均能完成审阅、编辑和提交。

## 风险

| 风险 | 控制 |
| --- | --- |
| 聚合范围越权 | 服务端按用户、团队和时间统一过滤 |
| AI 编造成果 | 来源约束、结构化输出和可复核引用 |
| 自动刷新覆盖人工内容 | 快照与正文分离，采用版本或显式确认 |
| 通知轰炸 | 频率限制、去重和团队级停用 |
| 指标被误当绩效 | 页面解释数据边界，不输出单一人员排名 |
