# 视觉创作配图标记人工控制 · 计划

> **版本**：v2.0 | **日期**：2026-07-17 | **状态**：规划中

## 目标

让文章配图标记从依赖模型随机输出，升级为可持久化、可精确编辑、可复现的用户控制流程。

位置策略、段落 gutter、右键插入、锚点视觉反馈和首次教程已经落地。本计划只保留尚未完成的 workspace 持久化、服务端锚点强约束和单标记操作。

## P2：策略与锚点权威化

1. 将位置策略保存到文章配图 workspace，不再依赖浏览器 session。
2. 服务端在调用模型前扫描受支持的用户锚点。
3. 明确锚点存在时，标记位置由确定性解析器决定；模型只补描述与视觉建议。
4. 工作区版本与更新接口覆盖策略变化，避免多标签页静默覆盖。

支持的锚点语法必须有限且可展示。无法识别的写法应提示用户，不得猜测位置。

## P3：单标记操作

- 在指定段落上方或下方新增标记；
- 删除单个标记；
- 仅重新生成一个标记的描述或图片；
- 操作后保持其它标记顺序和已编辑内容不变；
- 每次修改更新 workspace 版本，并返回服务端权威结果。

前端 hover 菜单只是入口，所有变更必须经过服务端权限、版本和索引校验。

## 数据与兼容

- 历史 workspace 没有位置策略时按 `auto` 读取。
- 锚点身份不能只用当前数组下标；重排后仍需定位原标记。
- 模型输出不能删除用户显式锚点。
- 单标记重生成失败时保留旧内容，不影响整篇文章。

## 验收门

- 换浏览器或设备后位置策略仍保持。
- 用户锚点在多次生成中位置稳定，不因模型变化漂移。
- 新增、删除和重生成一个标记不会改动其它标记。
- 并发编辑触发版本冲突提示，不发生最后写入静默覆盖。
- 自动模式与历史 workspace 行为不退化。
- 前后端行为测试、真实预览和移动端操作通过。

## 风险

| 风险 | 控制 |
| --- | --- |
| 数组下标漂移 | 使用稳定标记身份或版本内定位合同 |
| 模型忽略锚点 | 服务端确定性预扫描并锁定位置 |
| 乐观更新失败丢内容 | 失败回滚到服务端结果 |
| 旧数据无策略 | 缺省 `auto`，迁移按读取兼容 |
