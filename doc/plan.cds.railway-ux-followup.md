## 下一步计划

| 优先级 | 问题 | 修复方向 | 验收方式 |
|--------|------|----------|----------|
| P0 | 从 skill 扫描和 CDS 内置扫描未完全统一 | 把本地 cdscli 的包管理器判断迁入服务端扫描链路 | `package-lock.json` 项目生成 npm 命令 |
| P1 | 数据库卡片还不能直接浏览数据 | Mongo/MySQL 节点增加库、表、集合、存储量、初始化脚本入口 | 点击数据库卡片可查询和初始化 |
| P1 | 卡片视觉不够 Railway | 统一卡片信息密度、来源 chip、更新时间含义、可点击区域 | 视觉截图对照 Railway 风格无明显遮挡 |
| P1 | 部署失败归因仍不够细 | 把日志诊断结果挂到卡片和部署页签 | 能标明代码侧、配置侧、CDS 侧 |
| P2 | E2E 覆盖不足 | 增加从 compose、skill、0 创建三条 Playwright/Bridge 冒烟 | 三条路径至少各跑通一次 |
