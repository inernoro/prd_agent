> **版本**：v1.0 | **日期**：2026-04-02 | **状态**：当前

## 一、需求与方案阶段

| # | 技能名 | 调用方式 | 作用 | 使用场景 | 具体用法 |

| 3 | **plan-first** | `/plan-first`、`给出方案`、`先给方案` | 先出方案再动手，300 字内结构化分析，等用户确认后才执行 | 任何需要先规划再执行的场景 | `说"给出方案"或"方案确认后再动手"` |

## 推荐工作流（质量保障技能链）

```
需求 → /validate → 设计 → /risk → /trace → 实现 → /verify → /scope-check → /cds-deploy → /preview → /handoff → /weekly
```

| 阶段 | 推荐操作 |
|------|----------|
| 新需求提出 | `/validate` 验证需求质量 |
| 方案评审 | `/risk` 风险评估 → `/trace` 链路追踪 |
| 开发完成 | `/verify` 交叉验证 → `/cds-deploy` 部署+冒烟 |
| 人工验收 | `/preview` 生成预览地址 |
| 提 PR 前 | `/resolve` 预合并 main → `/scope-check` 边界审计 |
| 准备上线 | `/handoff` 交接清单 |
| 周五收尾 | `/weekly` 周报 → 自动 `/doc-sync` |
| 迁移/重构后 | `/hygiene` 代码卫生 |

## 技能文件位置

所有技能定义位于 `.claude/skills/*/SKILL.md`，共 37 个目录。部分技能含 `reference/` 子目录（补充文档）和 `data/` 子目录（设计数据）。
