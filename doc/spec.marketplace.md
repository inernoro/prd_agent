# 海鲜市场产品规格 · 规格

> **版本**：v4.0 | **日期**：2026-07-17 | **状态**：已落地

海鲜市场是平台内可共享配置的发现与复制入口。用户可以发布本人拥有的配置，其他用户通过 Fork 获得独立副本。市场不转移原配置所有权，也不直接暴露敏感字段。

## 1. 目标与边界

目标：

- 让用户发现可复用的 Agent 配置。
- 通过复制而非共享写权限降低协作风险。
- 为发布、下架、复制和统计建立统一规则。
- 允许新配置类型通过注册表受控接入。

非目标：

- 不作为插件二进制、代码包或任意文件市场。
- 不提供跨租户的隐式数据共享。
- 不允许市场消费者修改发布者的原配置。

## 2. 用户角色

| 角色 | 能力 |
|---|---|
| 发布者 | 发布本人拥有且允许共享的配置，更新展示信息，取消发布 |
| 消费者 | 浏览、筛选、查看详情并 Fork 配置 |
| 管理员 | 处理违规内容、控制允许发布的配置类型 |

## 3. 配置类型

可发布类型由配置类型注册表统一声明。每种类型必须定义：

- 稳定类型标识与展示名称。
- 来源实体的所有权校验。
- 可复制字段白名单。
- 敏感字段排除规则。
- Fork 后新实体的默认状态。
- 详情页和目标使用入口。

未登记类型不能通过仅传字符串的方式发布。新增类型时必须同时覆盖后端注册、复制策略、前端展示和测试。

## 4. 发布

发布前必须满足：

- 当前用户拥有来源配置。
- 配置类型允许进入市场。
- 必填展示信息完整。
- 来源配置不含不可脱敏的秘密或私人数据。
- 同一来源不存在冲突的有效发布记录。

发布记录至少包含来源引用、类型、标题、简介、作者、发布时间、状态和统计信息。市场只保存展示快照与来源关联，不复制完整业务配置作为新的事实源。

发布者更新原配置后，市场展示是否同步由该类型策略决定；必须避免用户无法判断详情对应哪个版本。

## 5. 浏览与详情

用户可按类型、关键词、发布时间和热度浏览。列表卡片展示：

- 标题、简介和配置类型。
- 作者与发布时间。
- Fork 次数等可公开统计。
- 明确的详情与 Fork 入口。

详情页只展示允许公开的配置摘要、预览和使用说明。任何密钥、内部标识、私有附件地址或未经授权的引用都不得返回前端。

## 6. Fork

Fork 是创建独立副本，不是引用共享：

1. 服务端确认市场条目有效且当前可见。
2. 根据配置类型读取来源实体。
3. 通过字段白名单创建归当前用户所有的新实体。
4. 清除标识、审计字段、秘密和发布状态。
5. 记录来源市场条目，并原子更新统计。
6. 返回新配置的标识和可进入的使用位置。

Fork 后：

- 发布者后续修改或下架不改写既有副本。
- 消费者可以独立编辑和删除副本。
- 不得复制来源用户的权限、令牌和私有资源所有权。
- 重试应避免因同一请求意外创建多份副本。

## 7. 下架与治理

发布者可以取消发布。下架后条目不再出现在公开列表，也不能继续 Fork，但已创建副本保持不变。

管理员可因内容违规、安全风险或来源失效强制下架。治理操作必须记录操作者、原因和时间。

来源实体被删除时，市场条目应转为不可 Fork 状态并从公开列表移除，不能返回空详情或在 Fork 时才暴露内部错误。

## 8. 权限与安全

- 发布、编辑和下架必须校验市场条目及来源实体所有权。
- 列表和详情只返回公开字段。
- Fork 在服务端执行，不接受客户端提交的完整来源配置。
- 所有配置类型使用显式字段白名单，禁止通用序列化复制。
- 图片和附件只能复制允许共享的引用或创建受控副本。
- 管理操作和异常 Fork 应可审计。

## 9. 接口职责

具体路由和字段以实现为准，接口至少覆盖：

| 资源 | 操作 |
|---|---|
| 市场列表 | 分页、搜索、类型筛选和排序 |
| 市场详情 | 获取公开详情与可 Fork 状态 |
| 发布记录 | 创建、更新展示信息、取消发布 |
| Fork | 创建当前用户拥有的配置副本 |
| 我的发布 | 查询本人发布记录及统计 |
| 治理 | 管理员下架与原因记录 |

统一要求：

- 使用标准认证和响应包装。
- 服务端执行所有权限与类型判断。
- 列表查询有稳定排序和分页边界。
- 错误区分条目不存在、已下架、来源失效、无权限和类型不支持。

## 10. 验收标准

- 用户只能发布本人拥有且已登记类型的配置。
- 列表和详情不泄露秘密、内部字段或私人附件。
- Fork 后产生独立所有权副本，且字段符合该类型白名单。
- 发布者修改原配置不会静默覆盖消费者副本。
- 下架后不能新建 Fork，既有副本仍可使用。
- 来源删除后条目不会继续显示为可用。
- 新增配置类型时，缺少注册或复制策略会被测试阻止。
