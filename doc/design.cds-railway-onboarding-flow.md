# CDS Railway 式部署向导设计

> **版本**：v1.0 | **日期**：2026-05-12 | **状态**：已落地

## 背景

用户希望 CDS 能像 Railway 一样完成从仓库到运行环境、基础设施、部署、日志和数据库查看的连续页面流程。此前 CDS 已具备这些能力，但入口分散：项目列表负责创建，项目设置负责配置，拓扑页只负责展示，基础设施创建入口不够直接，新用户很难从 0 自助部署一个陌生仓库。

## Railway 首次部署路径

| 步骤 | Railway 行为 | 用户获得的信息 |
|------|--------------|----------------|
| 1 | 点击 New Project | 明确进入部署工作流 |
| 2 | 选择 Deploy from GitHub repo | 来源是哪个仓库 |
| 3 | 选择仓库和分支 | 默认分支、仓库权限是否可用 |
| 4 | Railway 自动识别运行环境 | 服务镜像、命令、端口有默认值 |
| 5 | 在项目画布添加数据库 | 数据库是同项目内的服务节点 |
| 6 | 自动注入变量并触发部署 | 应用能拿到数据库连接串 |
| 7 | 查看服务详情、日志、变量、数据 | 失败时知道看哪里 |

## CDS 旧流程差距

| 问题 | 旧表现 | 后果 |
|------|--------|------|
| 创建入口分散 | 新建项目、GitHub 选择、环境变量、基础设施在不同位置 | 新用户不知道下一步点哪里 |
| 运行环境不显性 | 主要依赖自动扫描或 `cds-compose.yml` | 没配置文件的仓库会落到空白拓扑 |
| 基础设施入口隐藏 | 主要在设置或 API 层 | 用户无法像 Railway 一样在画布里加数据库 |
| 连接变量不连续 | infra 和 env 的关系靠配置导入体现 | 手动创建项目时容易漏 DATABASE_URL |
| 失败归因弱 | 红色状态容易被误解为 CDS 故障 | 代码错误、配置错误、平台错误混在一起 |

## 已落地方案

### 一键部署项目弹窗

项目列表的“新建项目”升级为“一键部署项目”，把三件事放到同一个弹窗：

| 区域 | 内容 | 后端落点 |
|------|------|----------|
| 选择仓库 | 项目名称、Git URL、GitHub 选择器、默认分支 | `POST /api/projects` |
| 选择运行环境 | 自动识别、Node.js、Python、.NET、Java、自定义 | `Project.onboardingRuntime` |
| 选择基础设施 | MongoDB、PostgreSQL、MySQL、Redis | `InfraService` + 项目级 env |

当仓库没有 `cds-compose.yml` 或 `docker-compose.yml` 时，若用户选择了运行环境，clone 完成后 CDS 会用运行环境提示创建一个根 BuildProfile。这样用户不会被留在空白拓扑页。

### 基础设施预设

| 预设 | 镜像 | 端口 | 自动环境变量 | 默认持久化 |
|------|------|------|--------------|------------|
| MongoDB | `mongo:7` | 27017 | `MONGODB_URL` | `/data/db` |
| PostgreSQL | `postgres:16-alpine` | 5432 | `DATABASE_URL`、`POSTGRES_URL` | `/var/lib/postgresql/data` |
| MySQL | `mysql:8` | 3306 | `DATABASE_URL`、`MYSQL_URL` | `/var/lib/mysql` |
| Redis | `redis:7-alpine` | 6379 | `REDIS_URL` | `/data` |

### 拓扑页新增基础设施

拓扑页的“基础设施”区域新增“新增基础设施”按钮。用户可以选择数据库预设或自定义镜像，创建后节点直接出现在拓扑图，右侧详情继续负责启动、停止、重启、日志和数据库操作入口。

## 关键设计约束

| 约束 | 原因 |
|------|------|
| 不删除旧入口 | 老用户仍需要高级配置、导入和设置页 |
| 默认持久化卷 | 数据库重启不能丢数据 |
| 运行环境提示优先于启发式扫描 | 用户主动选择的运行环境比猜测更可信 |
| 只在无精确配置时使用提示 | `cds-compose.yml` / `docker-compose.yml` 仍是最高优先级 |
| 自定义运行环境必须填镜像和命令 | 避免创建不可部署的半成品 |

## 验收清单

| 场景 | 预期结果 |
|------|----------|
| 从 GitHub 创建 Node 项目并选择 Redis | 项目创建成功，clone 后生成 Node BuildProfile，Redis 节点和 `REDIS_URL` 存在 |
| 从 GitHub 创建 Python 项目且仓库只有 README | clone 后仍生成 Python BuildProfile |
| 创建项目时选择 PostgreSQL | infra 默认带 `/var/lib/postgresql/data` 持久化卷 |
| 拓扑页手动新增 MongoDB | 节点立即出现在基础设施区，详情可启动和查看日志 |
| 自定义运行环境不填命令 | 后端返回校验错误，不创建半成品 |
