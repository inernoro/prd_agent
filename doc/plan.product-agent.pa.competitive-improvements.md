# PA Agent 可执行任务升级 · 计划

> **版本**：v2.0 | **日期**：2026-07-17 | **状态**：规划中

## 目标

把 PA Agent 从“对话生成四象限任务”升级为可编辑、可安排、可提醒和可纠正的个人行动工作台。本文只保留尚未实施的产品增量，不再保存易过时的竞品宣传摘要。

## 产品边界

- 首期帮助用户规划和执行个人任务，不扩成团队项目管理系统。
- 先做建议时间块与单向导出，不直接进入自动重排日历和多工具双向同步。
- AI 结果始终可编辑、可拒绝；模型解释不能替代用户决策。
- 外部日历、提醒和语音数据需要明确授权、保存范围和删除方式。

## P2：任务模型与编辑效率

- 增加截止时间、预估时长、标签、来源会话和可选项目。
- 看板支持拖拽象限、标题与子步骤内联编辑、删除确认和筛选。
- 用户可以选择“仅建议”或“自动创建”；自动创建策略按用户保存。
- 卡片展示简短归类理由，并把用户拖拽修正记录为可审计反馈。

## P3：今日焦点与提醒

- 从 Q1/Q2、截止时间和可用时长生成一至三条“今日焦点”，用户可以替换或关闭。
- 支持至少一种到期提醒渠道，并能查看、暂停和删除提醒。
- 支持周期复盘，汇总完成、延期、象限变化和用户修正，不编造未记录原因。
- 定时任务有时区、幂等、重试和取消，不依赖浏览器页面保持打开。

## P4：输入与日历

- 语音先转写到输入区，用户确认后再生成任务。
- 提供少量结构化场景模板，如周计划、会议行动项和学习计划。
- 首阶段支持 ICS 导出或日历只读忙闲，用于避免建议与已知会议冲突。
- 双向写入日历、Todo 工具或团队分配必须另行设计冲突解决与权限模型。

## 验收门

1. 新增字段从 API、数据库到前端一致，旧任务仍可读取和编辑。
2. 拖拽、内联编辑和删除不需要回到聊天，也不会被下一轮 AI 静默覆盖。
3. “仅建议”模式不产生任务；自动模式的范围清楚可见并可撤销。
4. 提醒在时区、重复触发、服务重启和取消后行为正确。
5. AI 结构化失败时保留原始文本和重试入口，不生成残缺任务。
6. 日历或语音权限拒绝后核心任务功能仍可用。
