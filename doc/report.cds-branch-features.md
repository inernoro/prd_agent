## 2. 域名路由与分支预览 (12 commits)

| 功能 | 说明 |
|------|------|
| **子域名预览** | 支持通过子域名（如 `feature-a.dev.example.com`）直接访问对应分支 |
| **Switch 域名切换** | 访问 `/_switch/<branch>` 设置 cookie，后续请求自动路由到指定分支 |
| **Cookie 分支绑定** | `cds_branch` cookie 存储原始分支名，支持 URL 解码 |
| **域名配置化** | `SWITCH_DOMAIN` / `MAIN_DOMAIN` 从环境变量读取，不再硬编码 |
| **分支解析优先级** | X-Branch header > cookie > 域名路由规则 > 默认分支 |
| **预览按钮** | 卡片上直接显示预览按钮，一键在新标签页打开分支预览 |

**核心逻辑**：`resolveBranch()` 返回原始分支名，`handleRequest()` 负责 slugify 后查找对应服务。

**关键文件**：`src/services/proxy.ts`, `src/services/state.ts`, `web/app.js`

## 5. UI 设计演进 (20+ commits)

Dashboard 经历了 4 轮主要设计迭代：

### 5.1 初始优化期

- 字体可读性优化
- 图标尺寸统一（12→14px, 16→18px）
- SVG 在 flex 容器中被压缩的修复
- Octicon 图标体系引入（分支、提交、操作按钮）

### 5.2 Bento + Glassmorphism 阶段

- 毛玻璃效果卡片
- 层叠上下文问题（z-index）多轮修复
- Portal 方案解决下拉菜单被遮挡

### 5.3 Apple-Inspired 阶段

- 苹果风格设计语言全面改版
- 端口徽章改为轻量文字 + 圆点指示器
- 卡片去噪、层次感、呼吸感重设计

### 5.4 最终收敛

- 实心卡片背景替代毛玻璃（性能 + 兼容性）
- 左边框 2px + 状态色标识
- 齿轮下拉菜单收纳 header 按钮
- commit 信息内联到 actions 行

**设计原则**：去噪 > 层次 > 呼吸感。减少视觉元素数量，通过颜色和间距建立信息层级。

## 技术债与后续建议

| 类别 | 说明 | 优先级 |
|------|------|--------|
| **WebSocket 替代轮询** | 当前 10 秒 `setInterval` 轮询分支状态，改为 WebSocket 推送可降低延迟 | P2 |
| **部署日志持久化** | 部署日志目前仅内存保存，刷新页面后丢失 | P3 |
| **分支卡片虚拟滚动** | 分支数量超过 50 时可能出现性能问题 | P3 |
| **E2E 测试** | 缺少前端 E2E 测试，所有测试均为后端单元测试 | P2 |
