# 准星智能体产品规格

> **版本**：v1.0 | **日期**：2026-06-03 | **状态**：规划中 | **appKey**：`zhunxing-agent`

---

## 一、背景与问题定义

公司内部规章制度、产研规范、跨部门交接流程普遍篇幅较长，员工在实际工作中常见问题包括：

- 知道“有规定”，但不清楚适用条款
- 知道流程名称，但不清楚先后步骤和审批边界
- 跨部门协作时，对责任边界和时限口径理解不一致

“准星”定位为公司统一的规范与流程问答中枢，先解决“快速问到准确答案”的问题，再逐步演进为“知识库 + AI”能力底座。

---

## 二、产品定位

### 2.1 一句话定位

准星是面向公司内部员工的规范问答智能体，提供“带依据、可追溯、可升级”的制度与流程解答。

### 2.2 核心价值

1. 降低理解成本：把万字制度转为可问可答的知识服务
2. 统一口径：以文档条款作为唯一依据，减少口头传递误差
3. 提升协作效率：对产研、市场、销售等跨部门交接问题给出标准流程路径

### 2.3 非目标（MVP 阶段）

- 不替代最终审批人和制度解释权部门
- 不生成超出制度依据的“推测性裁决”
- 不对外提供公开访问能力

---

## 三、目标用户与典型场景

| 用户类型 | 典型问题 | 期望结果 |
|---|---|---|
| 新员工 | 请假、打卡、迟到早退如何计算 | 1 分钟内拿到明确规则与出处 |
| 产研成员 | 需求评审、提测、交接按什么规范执行 | 给出步骤清单和责任边界 |
| 市场/销售 | 跨部门交接材料、时限、升级路径是什么 | 给出标准交接流程与异常处理 |
| 管理者 | 团队常见制度盲区有哪些 | 输出高频问答与培训建议 |

---

## 四、MVP 范围

### 4.1 知识域范围（首批）

1. 公司规章制度（示例：考勤管理办法）
2. 产研侧工作规范（需求、研发、测试、发布）
3. 市场与销售交接流程（交接物、SLA、升级路径）

### 4.2 能力范围（首批）

- 问答检索：自然语言提问，返回结构化答案
- 依据引用：答案必须附文档名与章节条款
- 追问澄清：当问题条件不完整时先补问关键条件
- 兜底升级：无明确依据时提供责任部门与人工确认建议

### 4.3 首批知识种子（已给样例）

- 文档：`米多公司考勤管理办法`
- 适用范围：全体部门
- 执行日期：2026-02-01
- 当前已覆盖章节：作息时间、打卡类别与规则、异常考勤、请假管理、旷工管理、出勤时间计算、基础素养考评（节选）

---

## 五、回答策略（准星口径）

### 5.1 标准回答结构

1. 结论：先给可执行结论
2. 依据：给出“文档名 + 章节条款”
3. 操作建议：下一步怎么做
4. 风险提示：边界条件或例外情况

### 5.2 输出约束

- 无依据不下结论
- 不输出超范围推断
- 涉及薪资、处罚、法务争议时强制提示人工复核

---

## 六、权限与安全要求

1. 准星仅用于内部员工访问
2. 敏感原文默认不外发，回答以摘要和条款引用为主
3. 访问日志需记录（用户、时间、问题、命中文档）
4. 文档入库前执行脱敏与分级（公开/内部/敏感/机密）

---

## 七、关键指标（MVP）

| 指标 | 目标值 |
|---|---|
| 首次响应时延 | < 2 秒 |
| 命中问题可追溯率（含出处） | >= 95% |
| 无依据误答率 | <= 3% |
| 员工满意度（问答后反馈） | >= 4.5/5 |

---

## 八、版本路线

### V1（当前）

- 规则问答 + 依据引用 + 人工兜底
- 首批覆盖 HR/考勤与核心协作流程

### V2

- 多文档聚合问答（跨制度联动）
- 部门级权限与答案可见范围控制

### V3

- 企业知识图谱与流程自动导航
- 从“问答工具”升级为“知识库 + AI”中枢

---

## 九、验收标准（MVP）

- [ ] 员工可通过自然语言提问获得明确结论
- [ ] 每条有效答案均包含条款出处
- [ ] 无命中时返回标准兜底话术和责任部门建议
- [ ] 至少覆盖 20 条高频制度问题
- [ ] 管理端可持续新增/更新知识条款且可追踪版本
