# Desktop 资产中心 · 计划

> **版本**：v2.0 | **日期**：2026-07-17 | **状态**：规划中

## 目标

在 Tauri Desktop 客户端提供跨 Agent 的只读资产中心，让用户在大屏环境中搜索、筛选、预览和打开已有图片、文档、附件与视频产物。

Web 管理端已有 `DesktopAssetsPage`，但它不等于 Tauri Desktop 已完成。本计划只处理桌面客户端接入，不重建后端资产聚合模型。

## 范围

| 包含 | 不包含 |
| --- | --- |
| 复用现有资产聚合 API | 新建第二套资产数据库 |
| 网格、列表、搜索、排序和详情 | 跨模块批量删除 |
| 复制链接、下载和外部打开 | 在资产中心上传或编辑源资产 |
| 分页与缩略图懒加载 | 第一阶段内嵌复杂视频编辑器 |

## 阶段

### P1：入口与基础浏览

- 在 Desktop 导航和路由中注册“我的资产”。
- 建立桌面端资产状态层，调用现有聚合 API。
- 支持分类、分页、加载、错误和空状态。
- 提供自适应网格与基础详情预览。

### P2：大屏效率

- 增加列表视图、标题搜索和时间、名称、大小排序。
- 详情面板提供复制链接、下载和系统浏览器打开。
- 保存视图偏好，支持键盘选择、打开和关闭详情。

### P3：质量与性能

- 缩略图懒加载与失败占位。
- 深浅主题、窄窗口和高分屏验证。
- 大量资产下的分页、取消和重复请求去重。

## 数据与权限

资产列表继续由服务端按当前用户权限聚合。Desktop 不在本地重算可见范围，也不缓存真实下载凭据。下载或打开动作使用服务端返回的受控 URL；过期时重新请求，不把临时 URL 当永久资产身份。

## 验收门

- 用户可从 Desktop 导航进入资产中心并返回原工作区。
- 四类资产的列表、分页、搜索、排序和空状态可用。
- 点击资产能预览元数据并执行允许的打开或下载动作。
- 无权限资产不会因本地缓存或前端过滤缺陷泄露。
- 390px 等效窄窗与桌面宽屏均无横向溢出，深浅主题可读。
- 类型检查、Lint、Desktop 构建和至少一条真实资产 API 行为验证通过。

## 风险

| 风险 | 控制 |
| --- | --- |
| Web 与 Desktop 重复实现 | 共享契约和视觉模式，不共享平台特定副作用 |
| 大图和视频拖慢列表 | 缩略图、懒加载和按需预览 |
| 搜索只覆盖已加载页 | UI 明示范围，后续需要全量搜索时扩展服务端查询 |
| URL 过期 | 用资产 ID 重新获取，不持久化临时 URL |
