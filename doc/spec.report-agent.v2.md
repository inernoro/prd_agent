# 周报 Agent v2.0 产品需求 · 规格

> **版本**：v2.1 | **日期**：2026-07-17 | **状态**：已落地

周报 Agent 将代码、任务和知识产出整理为可确认的周报。产品重点是减少手工整理，不承担审批、评论和通知等管理流程。应用标识为 `report-agent`。

## 1. 产品目标

- 自动采集用户在指定周期内的有效产出。
- 按团队模板组织内容，并允许用户生成后修改。
- 保留来源和生成记录，避免模型杜撰工作内容。
- 支持个人确认和团队汇总，但不引入复杂审批状态。

非目标：

- 不替代项目管理或绩效系统。
- 不自行实现 TAPD、Git 等通用连接器。
- 不根据无来源的模型推测生成完成项。

## 2. 用户角色

| 角色 | 能力 |
|---|---|
| 团队负责人 | 配置团队、采集工作流、模板和成员映射，查看团队汇总 |
| 团队成员 | 配置个人来源、生成并编辑本人周报 |
| 系统管理员 | 管理应用权限、模型组和系统级默认模板 |

用户只能查看本人或有团队权限的数据。

## 3. 核心概念

| 概念 | 职责 |
|---|---|
| 团队 | 成员、负责人、采集工作流和默认模板的作用域 |
| 采集工作流 | 复用 Workflow Engine 获取外部数据 |
| Artifact | 工作流执行产生的标准化采集结果 |
| 个人数据源 | 用户级轻量补充来源 |
| 模板 | 周报板块、顺序和生成要求 |
| 周报 | 周期、来源快照、生成内容和确认状态 |
| 手动打点 | 用户主动补充的当周事实 |

## 4. 数据采集

### 4.1 团队采集

团队绑定一个已有工作流。周报生成时触发或读取该工作流的有效 Artifact，再按成员身份映射拆分数据。周报 Agent 只消费标准化结果，不复制连接器逻辑。

工作流失败时：

- 明确展示失败的数据源和可重试入口。
- 已成功来源仍可继续使用。
- 用户可选择在数据不完整的情况下生成，但结果必须标记缺失来源。

### 4.2 个人补充

个人可配置轻量数据源或添加手动打点。个人来源只进入本人周报，不修改团队工作流。相同外部记录应按来源标识和业务标识去重。

### 4.3 时间与身份

- 周期以团队时区为准，默认按自然周计算。
- 外部账号通过显式成员映射关联用户。
- 无法映射的数据进入待处理清单，不得自动归给名称相似的成员。

## 5. 模板

模板定义板块名称、顺序、可见性和生成提示。系统提供可用默认模板，团队可复制后修改。

最低支持板块：

| 板块 | 内容 |
|---|---|
| 本周完成 | 有来源支撑的交付、任务和文档 |
| 关键数据 | 可验证的数量、状态或里程碑 |
| 风险与阻塞 | 未解决问题及影响 |
| 下周计划 | 用户确认的后续动作 |

模板变化不应静默改写已生成周报。生成记录保存当时的模板和来源快照。

## 6. 生成流程

1. 用户选择周期并检查数据源状态。
2. 系统执行团队采集并合并个人来源与手动打点。
3. 系统展示采集进度、来源数量和失败项。
4. LLM 根据事实数据和模板流式生成草稿。
5. 用户编辑、补充或删除内容。
6. 用户确认周报，系统保留确认时间和最终正文。

生成约束：

- 无来源内容不得写成已完成事实。
- 数字应来自采集结果或用户输入。
- 失败重试不得重复创建同周期周报。
- 用户编辑后的内容优先于再次生成结果。

## 7. 状态

周报只保留必要状态：

| 状态 | 含义 |
|---|---|
| 草稿 | 已创建，可继续生成或编辑 |
| 已确认 | 用户确认当前内容 |

采集和生成过程使用运行状态，不混入周报业务状态。取消确认、审批流和通知不属于 v2 范围。

## 8. 团队视图

团队负责人可按周期查看：

- 已确认、草稿和未创建成员数量。
- 每位成员的周报摘要和来源完整性。
- 团队级完成项、风险与下周重点汇总。

团队汇总必须保留到成员周报的追溯路径，不应把多人的内容合并成无法核对的模型总结。

## 9. 配置流程

### 9.1 团队负责人首次配置

1. 创建或选择团队。
2. 选择已有采集工作流。
3. 完成成员与外部账号映射。
4. 选择或复制模板。
5. 运行一次采集预览并处理未映射数据。

### 9.2 成员首次使用

1. 检查本人身份映射。
2. 连接允许的个人来源。
3. 添加必要的手动打点。
4. 生成、编辑并确认本周周报。

## 10. 接口职责

接口按以下资源分组，具体路径和字段以实现为准：

| 资源 | 操作 |
|---|---|
| 团队 | 查询、创建、更新、成员管理 |
| 采集配置 | 绑定工作流、预览运行、查看映射问题 |
| 个人数据源 | 查询、连接、更新、删除 |
| 模板 | 查询、复制、更新、启停 |
| 周报 | 创建或获取周期周报、生成、更新、确认 |
| 团队汇总 | 查询周期状态和汇总结果 |

所有接口使用统一认证、权限和响应包装。生成接口必须提供流式进度。

## 11. 验收标准

- 团队负责人能完成工作流、成员映射和模板配置。
- 成员能看到采集进度、来源明细和失败来源。
- 同一周期重复生成不会重复写入相同事实。
- 生成正文中的完成项可追踪到来源或用户输入。
- 用户可以编辑并确认，刷新后内容与状态保持一致。
- 团队负责人只能查看有权限团队的成员周报。
- 模型未绑定或采集失败时，页面给出可执行的恢复指引。
