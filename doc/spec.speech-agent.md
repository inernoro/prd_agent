# 演讲智能体 (Speech Agent) · 产品规格

> **版本**：v0.1 (MVP) | **日期**：2026-06-06 | **状态**：开发中

## 一、管理摘要

- **解决什么问题**：用户手上有长文档（文章、报告、笔记、PRD），需要快速准备一次"能上台讲"的分享，但从零搭 PPT 太重。
- **方案概述**：粘贴/上传/选库一段文本 → AI 拆成"思维导图"风格的演讲大纲（主题 → 章节 → 要点）→ 可编辑、可播放、可发布为静态站点。
- **业务价值**：把"理解 → 整理 → 上台"三步压成一步；复用平台 LLM Gateway / 视觉生图 / 网页托管这些已有砖块，0 新基础设施。
- **影响范围**：新增 `/speech-agent` 路由（智能体分组）、`speech_decks` + `speech_nodes` 两个 MongoDB 集合、4 个 LLM AppCallerCode。
- **预计风险**：低（MVP 仅含 LLM 文本调用 + Mongo CRUD，无新外部依赖）。

## 二、产品定位

「演讲智能体」是一个**演讲场景下的 AI 助手**。它不是 PPT 编辑器（无母版、无动画 timeline），也不是知识库（不接管文档原始存储），而是"长文本 → 可演讲产物"的转换器。

首期内置 1 个**模式**（mode）：`mindmap`（思维导图演讲）。后续可平滑扩展：
- `outline`：纯大纲文字稿
- `story`：故事化转换（开场钩子 / 起承转合）
- `data`：数据型演讲（自动拆图表点）

所有模式共享同一棵 `SpeechDeck` + `SpeechNode` 数据结构。

## 三、用户场景

### 场景 1：产品经理准备评审

PM 写完 PRD，需要在评审会上 15 分钟讲完。打开演讲智能体 → 粘贴 PRD 全文 → 选受众"管理层"、风格"专业"、深度 3 → 一键生成思维导图 → 简单调整后上台。

### 场景 2：工程师做技术分享

工程师写完一篇技术博客，想转成内部分享材料。粘贴博客 → 选受众"工程师"、风格"故事化" → 生成 → 编辑节点要点 → 播放态用空格键翻屏（Phase 2）。

### 场景 3：运营做客户讲解

运营从知识库挑一篇产品介绍 → 选受众"客户"、风格"简洁" → 生成对外讲解材料 → 发布到 hosted_sites 拿分享链发给客户（Phase 2）。

## 四、核心能力

### 4.1 MVP（v0.1，本次交付）

| 能力 | 说明 |
|------|------|
| 创建演讲 | 粘贴文本（≥30 字）+ 选受众/风格/深度 → 入库 draft |
| 流式生成 | SSE 推送 model / thinking / typing / node / done 事件，前端边收边渲染 |
| 思维导图视图 | 按 depth 分列展示 root → 章节 → 要点，节点可点击查看详情 |
| 节点编辑 | 改标题 / 改要点列表 |
| 重新生成 | 一键清空节点重跑 |
| 删除演讲 | 级联清节点 |
| 模型可见 | 顶栏展示当前调用的 `{model} · {platform}`（ai-model-visibility 规则） |

### 4.2 Phase 2（下一版本）

- 知识库选 / 文件上传两条输入通道
- 每节点配图（复用 VisualAgent ImageGen）
- 演讲备注（口播稿）生成
- 播放态：逐节点展开 + 配图淡入 + 键盘控制
- 发布到 hosted_sites 出分享链
- ReactFlow 画布替代当前列式视图（手势走 `.claude/rules/gesture-unification.md` 标准 B）
- Run/Worker + afterSeq 断线续传

### 4.3 Phase 3（远期）

- 多模式：outline / story / data
- 渲染成视频（接 Remotion + 视频 Agent）
- 海鲜市场分享演讲模板

## 六、与其他智能体的边界

| 智能体 | 边界 |
|---|---|
| 涌现探索器 | 涌现是"发散探索创意"，演讲是"收敛成可讲材料"。共用 ReactFlow 后可共享画布手势库 |
| 视觉创作 | 视觉创作做"独立画作"；演讲智能体的节点配图通过 AppCaller `speech-agent.node-image::generation` 走 ImageGen，但展示和管理在 deck 内 |
| 知识库 | 知识库是文档原始存储；演讲是从知识库挑文档作为输入，不修改原文 |
| 视频 Agent | Phase 3 演讲可"导出为视频"，调用视频 Agent 渲染成 MP4 |
