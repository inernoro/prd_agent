# 前端搭档智能体产品规格

## 1. 定位

前端搭档智能体面向需要独立完成前端交付的后端同事。它不是替代资深前端的自动编码器，而是把 API、报错、截图现象和页面需求转成可复制代码、接入步骤和自测清单。

## 2. 目标用户

| 用户 | 典型困难 | 智能体提供的帮助 |
|------|----------|------------------|
| 后端开发 | API 写完后缺少前端调用示例 | 生成 TypeScript 类型、service、页面调用样例 |
| 后端开发 | 不熟悉 React/Vue 组件组织 | 生成组件骨架、状态边界和接入步骤 |
| 后端开发 | 遇到浏览器或构建报错难以定位 | 解析报错、列定位路径、给最小修复代码 |
| 后端开发 | 页面样式异常但描述不清 | 按截图现象输出 CSS/布局诊断与验收标准 |

## 3. 首版能力范围

### 3.1 API 接入助手

- 输入：接口路径、请求体、响应 JSON、Controller 片段或 OpenAPI 片段。
- 输出：前端类型定义、请求函数、React/Vue 调用示例、loading/empty/error 处理。
- 验收：用户能复制输出代码并知道应该放到哪些文件。

### 3.2 组件生成助手

- 输入：页面或组件的自然语言描述、字段列表、已有组件约束。
- 输出：可读的 TSX/Vue 组件、状态说明、样式建议。
- 验收：输出包含文件路径、代码块、接入步骤和基础自测。

### 3.3 前端报错诊断

- 输入：浏览器 console、构建输出、接口错误、堆栈信息。
- 输出：现象、根因候选、定位步骤、最小修复和回归验证。
- 验收：用户能按步骤缩小问题范围或直接尝试修复。

### 3.4 截图视觉诊断

- 输入：截图现象描述、设计稿与实际页面差异描述。
- 输出：问题区域、CSS/布局原因、修复代码和验收标准。
- 验收：用户知道该改哪个样式、改完看哪几个视觉点。

## 4. 首版不做

- 不自动提交代码。
- 不假装已经扫描用户完整仓库。
- 不新增数据库集合保存会话。
- 不承诺直接解析图片二进制；首版基于用户填写的截图现象和差异描述。

## 5. 信息架构

入口位置：百宝箱。

页面结构：

1. 任务类型卡片：接 API、写组件、修报错、看截图。
2. 前端项目表：按项目名、仓库地址、技术栈、构建地址、备注搜索前端项目位置。
3. 输入区：需求描述、目标技术栈、样式约束、文本材料上传。
4. 输出区：阶段提示、模型信息、思考过程、流式结果、复制按钮。

## 6. 技术约束

- appKey：`front-end-agent`。
- 权限：`front-end-agent.use`。
- LLM 调用：统一通过 `ILlmGateway`，AppCallerCode 为 `front-end-agent.assistant::chat`。
- 流式体验：后端 SSE 推送 `phase`、`model`、`thinking`、`typing`、`done`、`error`。
- 前端入口：`/front-end-agent`，通过 `NAV_REGISTRY` 和 `BUILTIN_TOOLS` 注册。
- 项目表事实源：首版内置上传清单中的项目地址，保存在前端注册表 `frontEndProjectRegistry.ts`。

## 7. 验收标准

- 用户拥有 `front-end-agent.use` 权限时，能从百宝箱进入页面。
- 四个任务类型都能切换，并根据类型切换上下文输入框。
- 前端项目表可搜索项目名、技术栈、Coding/GitHub/SVN、构建地址，并复制项目地址摘要。
- 点击生成后，页面显示阶段状态和模型信息，并流式展示结果。
- 输出包含结论摘要、交付文件、可复制代码、接入步骤、自测清单、风险与假设。
- 权限管理页能看到并配置 `front-end-agent.use`。
