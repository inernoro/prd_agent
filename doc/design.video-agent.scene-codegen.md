# 视频场景代码生成 · 设计

> **版本**：v1.1 | **日期**：2026-07-17 | **状态**：已落地

## 管理摘要

- **问题**：固定场景模板难以覆盖不同脚本、节奏和视觉主题，视频成片容易同质化。
- **决策**：为分镜生成受约束的 Remotion 场景实现；生成结果必须经过校验并可回退到稳定模板。
- **价值**：在保持可渲染、可追溯和可回滚的前提下，让不同分镜获得更贴合内容的视觉表达。

## 核心决策

1. **分镜是最小生成单元**：每个分镜独立保存其叙述、视觉描述、时长、媒体引用和生成状态。
2. **生成受技能与契约约束**：模型调用使用 `video-agent.scene.codegen::code`，注入 Remotion 能力边界和项目组件知识；不允许生成任意运行时行为。
3. **校验先于采用**：生成代码只有在结构、依赖和渲染前提满足时才成为当前场景；失败不影响其它分镜。
4. **稳定模板兜底**：生成、校验或加载失败时回退到已验证模板，并向用户明确展示回退状态。
5. **服务端权威记录**：Worker 保存生成状态、错误摘要、产物位置和使用的回退路径；浏览器只展示进度和预览。

## 场景契约

| 输入 | 说明 |
|---|---|
| 分镜主题与旁白 | 决定内容重点和字幕语义 |
| 视觉描述与场景类型 | 决定构图、信息层次和可选组件 |
| 时长与帧率 | 决定时间轴和退场边界 |
| 背景图、音频与素材引用 | 由已保存的媒体资源提供，不由生成代码私自下载 |

场景输出必须遵循项目的 Remotion 运行环境、帧驱动动画和本地可用组件。具体 API、动画工具和组件示例属于 `remotion-scene-codegen` 技能，不重复写入设计文档。

## 执行流程

1. 视频生成 Worker 创建或恢复分镜任务。
2. 对需要差异化视觉的分镜，构造受控上下文并调用场景代码生成能力。
3. 服务端校验生成结果，写入受管理的位置并记录分镜状态。
4. 渲染器加载已采用的场景；若加载、编译或渲染失败，切换到稳定模板。
5. Worker 通过阶段事件持续反馈生成、校验、渲染和回退结果。

## 回退与风险

| 风险 | 处理 |
|---|---|
| 生成结果语法或依赖不合法 | 拒绝采用，保留错误摘要并回退模板 |
| 渲染期失败 | 隔离到当前分镜，整条视频继续使用稳定场景 |
| 素材缺失或不可访问 | 标记分镜素材问题，不让生成代码伪造资源 |
| 视觉质量不符合预期 | 允许重新生成或选择模板，保留上一版可回退结果 |
| 长时间生成 | 持续推送阶段和进度，不显示静态加载状态 |

## 验收标准

- 单个分镜的生成失败不会阻断其它分镜渲染。
- 用户能区分“已生成场景”和“已回退模板”。
- 场景调用经过统一模型网关并保留可追溯的运行记录。
- 成片预览和最终渲染均不依赖浏览器临时状态。
