## 目标与价值

让普通用户第一次部署时不需要理解数据库初始化细节：CDS 先识别 Prisma、Drizzle、TypeORM、Sequelize、Django、Alembic、Rails、SQL 脚本等信号，给出推荐初始化方式；部署时在数据库 ready 后自动执行；失败或高级场景才显示 SQL 导入、自定义迁移命令、重新初始化入口。

## 范围与不覆盖

本轮覆盖远程 CDS 控制面已部署状态、项目扫描推荐、数据库服务卡片入口、迁移命令高级入口、环境变量向导的 SQL 兜底位置。没有在生产 `mytapd` 数据库上真实执行迁移或 SQL，因为这属于破坏性操作；自动执行链路由后端单测和类型检查覆盖，远程只验证入口、路由和 UI 状态。
