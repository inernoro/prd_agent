## 步骤 1 · 打开分支列表并查看资源 chip

进入分支列表后，分支卡片直接展示 Node.js、.NET、Redis 等运行时图标、端口和状态，不再把 api/admin 作为主标签。

![分支卡片资源 chip：证明 api/admin 已升级为 Node.js、.NET、MySQL、Redis 统一资源展示](./acc-prd-agent-202606092242-cds-分支资源控制台升级/01-branch-resource-chips.png)

## 步骤 2 · 打开分支详情并切到「资源」页

点击分支卡片打开详情抽屉，再点「资源」页签，可以看到应用、数据库、缓存按统一 Resource 模型分组展示。

![资源页入口：证明分支详情抽屉中“服务”已升级为“资源”，并展示 4 个统一 Resource](./acc-prd-agent-202606092242-cds-分支资源控制台升级/02-resource-tab-overview.png)

## 步骤 3 · 选择 MySQL 资源并验证单独控制入口

在资源列表选择 MySQL 后，右侧详情面板展示 MySQL :31306，并提供启动、重启、停止三个独立控制入口。

![数据库资源控制：证明 MySQL 与应用容器同模型展示，并具备单独启动、重启、停止入口](./acc-prd-agent-202606092242-cds-分支资源控制台升级/03-mysql-resource-control.png)

## 步骤 4 · 打开「连接」并验证动态连接信息

切到连接页后，面板展示内部地址、外部 TCP 地址、MySQL 连接串和复制按钮，同时提示数据库公网访问需要 allowlist、有效期和凭据记录。

![数据库连接管理：证明面板展示内部地址、外部 TCP 地址、连接串和外部访问风险提示](./acc-prd-agent-202606092242-cds-分支资源控制台升级/04-mysql-connection.png)

## 步骤 5 · 打开「备份」并验证分支数据库隔离入口

切到备份页后，面板展示创建分支独立数据库、从 main/prod 克隆、从备份恢复、连接已有数据库，以及 MySQL 快速复制的实现策略入口。

![数据库隔离与复制：证明 MySQL 面板给出分支独立库、主库克隆、备份恢复和快速复制能力入口](./acc-prd-agent-202606092242-cds-分支资源控制台升级/05-mysql-backup-clone.png)
