# AI 竞技场使用 · 指南

> **类型**：guide | **适用角色**：AI 研究者、开发人员、产品经理 | **最后更新**：2026-03-19
> **预计阅读**：6 分钟 | **难度**：入门

---

## 1. 功能概览

AI 竞技场（Arena）让多个大语言模型匿名回答同一问题——你在不知道模型身份的情况下评判回答质量，消除品牌偏见。

| 能力 | 说明 |
|------|------|
| 盲测对比 | 模型身份匿名（标记为助手 A、B、C...），评判后揭晓 |
| 实时流式 | 所有模型同时开始回答，流式输出实时可见 |
| 性能指标 | TTFT（首字延迟）、总响应时长实时显示 |
| 思考过程 | 支持显示模型的推理链（如 CoT） |
| 附件支持 | 上传图片/文件作为提问上下文 |
| 对战历史 | 所有对战记录自动保存，按日期分组浏览 |

---

## 2. 操作路径

**入口**：首页 → 点击「AI 竞技场」卡片 → 进入竞技场页面

---

## 3. 完整操作流程

### 步骤 1：创建竞技组

1. 进入竞技场页面
2. 创建一个新的竞技组
3. 添加 2-4 个模型到竞技组
4. 模型身份会被随机打乱，标记为 **「助手 A」**、**「助手 B」**、**「助手 C」**...

> **Pro Tip**：建议每次对比 2-3 个模型。超过 4 个会导致页面拥挤，影响对比体验。

### 步骤 2：发起对战

1. 在底部输入框输入测试问题
2. 可选点击附件按钮（Paperclip 图标）上传图片或文件
3. 点击发送按钮（Send 图标）
4. 所有模型 **同时** 开始回答

### 步骤 3：观察回答过程

每个模型的回答面板实时显示：

| 信息 | 说明 |
|------|------|
| 状态指示 | **「thinking...」** → **「streaming...」** → **「done」** / **「error」** |
| **「思考过程」**（Brain 图标） | 模型的推理链，可折叠展开 |
| 回答内容 | 流式输出，逐字渲染 |
| TTFT | 首字延迟时间 |
| 总时长 | 完整回答生成时间 |

### 步骤 4：评判与揭晓

1. 阅读各个匿名助手的回答
2. 根据回答质量、准确度、速度综合评判
3. 点击 **「Reveal」**（Eye 图标）揭晓每个助手对应的真实模型
4. 结果会自动保存到对战历史

---

## 4. 对战历史

对战记录按日期分组：**「今天」** / **「昨天」** / **「更早」**

每条记录展示：
- 提问内容摘要
- 参战模型数量
- 是否已揭晓
- 可展开查看完整对话

支持操作：
- **「Download」**（Download 图标） — 导出对战结果
- **「Copy」**（Copy 图标） — 复制回答内容

## 5. 实用场景

### 场景 A：选择最适合的模型

```
提问：「请解释 Transformer 的自注意力机制，用类比方式让非技术人员理解」
对比维度：解释清晰度、类比恰当性、语言流畅性
```

### 场景 B：测试代码能力

```
提问：「用 Python 实现一个线程安全的 LRU 缓存，要求 O(1) 读写复杂度」
对比维度：代码正确性、边界处理、注释质量
```

### 场景 C：测试中文理解

```
提问：「解释"道可道，非常道"的哲学含义，并与维特根斯坦的语言哲学做对比」
对比维度：理解深度、知识广度、逻辑连贯性
```

## 6. 常见问题

**Q：模型的顺序每次都不一样吗？**
A：是的。每次对战模型标记（助手 A/B/C）都会随机打乱，确保无法通过位置记忆模型身份。

**Q：对战记录会保存多久？**
A：永久保存，随时可回顾。

**Q：可以只测某个模型吗？**
A：竞技场设计为对比场景。如果只想用单个模型，请使用对应的 Agent 对话功能。
