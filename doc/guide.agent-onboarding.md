# Agent 开发入门指南

> **适用对象**：产品经理、业务负责人、首次在 PRD Agent 平台开发 Agent 的任何人
> **阅读时间**：约 25 分钟
> **前置要求**：能打开终端、能与 AI 对话即可
> **最后更新**：2026-03-27

---

## 一、PRD Agent 平台是什么

PRD Agent 是一个 **AI Agent 工厂**——你描述想要什么样的 Agent，AI 帮你把它造出来。

平台已经运行着 6 个 Agent，覆盖文档解读、视觉创作、文学配图、缺陷管理、视频生成、周报管理。你即将创造第 7 个。

### 已有 Agent 速览

| Agent | 一句话说明 | 复杂度参考 |
|-------|-----------|-----------|
| PRD Agent | 上传 PRD 文档，AI 逐章解读并回答问题 | 中等 |
| 视觉创作 Agent | 全屏画布，文生图/图生图/批量生成 | 高 |
| 文学创作 Agent | 文章智能配图，分阶段创作 | 高 |
| 缺陷管理 Agent | 提交缺陷 → AI 分析 → 分派 → 修复跟踪 | 中等 |
| 视频 Agent | 文章一键转视频教程 | 中等 |
| 周报管理 Agent | 接入 Git/日志，AI 生成周报和团队总结 | 高 |

**你的 Agent 会和它们并列出现在系统菜单中，享有相同的权限体系、AI 能力和部署基础设施。**

## 二、30 秒理解系统架构

你不需要关心技术细节，但需要知道系统有三层：

```
┌─────────────────────────────────────────────────┐
│                  用户看到的                        │
│         网页管理后台 + 桌面客户端                    │
├─────────────────────────────────────────────────┤
│                 AI 大脑层                          │
│    统一 AI 网关 → 自动选模型 → 流式输出              │
├─────────────────────────────────────────────────┤
│                 数据存储层                          │
│          MongoDB + Redis + 文件存储                │
└─────────────────────────────────────────────────┘
```

**你需要关心的只有两件事**：
1. **你的 Agent 做什么**（需求、流程、界面）
2. **AI 怎么辅助**（哪些环节需要 AI 介入、用什么方式交互）

技术实现交给 AI 编程助手完成。你只需守住原则，创意和决策留给自己。

## 三、开发一个 Agent 的完整旅程

### 旅程地图

```
 你在这里
    ↓
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  阶段 0  │───▶│  阶段 1  │───▶│  阶段 2  │───▶│  阶段 3  │───▶│  阶段 4  │
│  破冰    │    │ 需求打磨 │    │ 方案设计 │    │  建造   │    │ 验收上线 │
│         │    │         │    │         │    │         │    │         │
│ 取名+说想法│   │ 验证可行性│    │ 评估风险  │    │ AI 实现  │    │ 测试交付 │
│         │    │ 排查重复  │    │ 梳理数据流│    │ 你来验收  │    │ 生成清单 │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     📋             ✅             📐            🔨             🚀
```

### 阶段详解

#### 阶段 0：破冰——给你的 Agent 取个名字

**你要做的**：
- 输入 `/help`，AI 会问你想做什么
- 给 Agent 取一个英文名（格式：`xxx-agent`，如 `review-agent`、`hr-agent`）
- 用一两句话描述它要解决什么问题

**完成标志**：`.agent-workspace/{你的agent名}/progress.md` 生成

**小贴士**：不用想得太完美，先有个方向就行，后面随时调整。

---

#### 阶段 1：需求打磨——确认值得做

**你要做的**：
- 和 AI 聊你的想法，它会帮你整理成结构化需求
- AI 会自动检查：需求是否清晰？是否和已有 Agent 重复？价值有多大？

**AI 会使用的技能**：

| 技能 | 触发方式 | 它会做什么 |
|------|---------|-----------|
| 需求验证 | `/validate` | 检测需求中的模糊点、遗漏项，给出 7 维度评分 |

**完成标志**：需求验证评分 ≥ 7.0，没有致命的模糊项

**常见问题**：
- *"我的想法还很模糊怎么办？"*——直接说，AI 会通过提问帮你理清
- *"和现有 Agent 功能有重叠怎么办？"*——验证技能会自动排查并告诉你差异点

---

#### 阶段 2：方案设计——想清楚再动手

**你要做的**：
- 和 AI 讨论核心流程：用户打开你的 Agent 后，第一步做什么？第二步呢？
- 确认哪些环节需要 AI 介入（如：自动分析、智能推荐、内容生成）
- 确认需要哪些页面/面板

**AI 会使用的技能**：

| 技能 | 触发方式 | 它会做什么 |
|------|---------|-----------|
| 风险评估 | `/risk` | 从 6 个维度（技术/数据/性能/安全/依赖/运维）评估风险 |
| 链路追踪 | `/trace` | 画出数据从输入到输出的完整路径，确保没有断链 |

**完成标志**：方案设计文档生成，关键风险已识别并有应对方案

---

#### 阶段 3：建造——AI 编码，你来验收

**你要做的**：
- 告诉 AI "开始实现"，它会按照方案逐步编码
- 你负责在关键节点验收：页面长得对不对？流程走得通不通？
- 遇到不满意的地方直接说，AI 会调整

**AI 会使用的技能**：

| 技能 | 触发方式 | 它会做什么 |
|------|---------|-----------|
| 交叉验证 | `/verify` | 从多个角度检查实现是否正确（魔鬼辩护、边界测试、用户模拟） |
| 代码卫生 | `/hygiene` | 检查代码质量，清理技术债务 |

**重要原则**：
- 你不需要看代码，但需要打开页面验收效果
- AI 实现时会自动遵守系统的所有技术规范（权限隔离、API 规范、AI 网关等）
- 如果涉及 AI 对话，所有 AI 响应都会以流式方式展示（打字机效果），不会让用户干等

**完成标志**：核心流程在页面上走通

---

#### 阶段 4：验收上线——最后一公里

**你要做的**：
- 完整走一遍用户流程，从进入页面到完成任务
- 确认新旧数据都能正常显示
- 确认权限控制正确（管理员能做什么，普通用户能做什么）

**AI 会使用的技能**：

| 技能 | 触发方式 | 它会做什么 |
|------|---------|-----------|
| 边界检查 | `/scope-check` | 扫描分支变更，检测是否越界修改了其他 Agent 或核心代码 |
| 部署测试 | `/cds-deploy` | 一键部署到测试环境 + 自动冒烟测试 |
| 预览地址 | `/preview` | 生成可访问的预览链接，直接在浏览器验收 |
| 冒烟测试 | `/smoke` | 自动生成并执行端到端测试命令 |
| 交接清单 | `/handoff` | 生成 8 维度交接报告，确保无遗漏 |

**完成标志**：交接清单生成，冒烟测试通过，**质量评分 ≥ 3.5**，代码已合并

**质量评分**：交付前 AI 会对你的 Agent 做 10 维度打分（满分 5.0）：

| 维度 | 权重 | 关注点 |
|------|------|--------|
| 使命对齐 | 15% | 是否解决真实痛点？是否围绕共同目标？ |
| 生态互补 | 10% | 与已有 Agent 是互补还是重叠？ |
| 用户体验 | 12% | 操作流畅、学习成本低、视觉一致 |
| 功能完整度 | 12% | 核心流程跑通、边界情况处理 |
| AI 交互品质 | 12% | 流式输出、进度反馈、无空白等待 |
| 架构合规 | 10% | 权限隔离、统一网关、标准模式 |
| 代码质量 | 8% | 零错误、无越界、命名规范 |
| 安全与数据 | 8% | 权限控制、数据隔离 |
| 创新性 | 8% | 新交互模式、独到 AI 应用 |
| 文档与可维护 | 5% | 设计文档、交接清单 |

**等级**：⭐ 标杆(4.5+) / ✅ 优秀(3.5+) / 🟡 合格(2.5+) / ⚠️ 待改进(<2.5)

## 五、必须知道的 5 条铁律

这些规则 AI 会自动遵守，但你了解它们有助于理解 AI 的行为。

### 1. 每个 Agent 都有独立身份

你的 Agent 会有一个唯一的 `appKey`（如 `review-agent`），所有 API、权限、菜单都围绕它隔离。这意味着：
- 你的 Agent 不会影响其他 Agent
- 其他 Agent 也不会影响你的
- 权限可以精确控制到"谁能用你的 Agent"

### 2. AI 调用走统一网关

你的 Agent 如果需要 AI 能力（聊天、分析、生图），不需要自己对接模型。系统有统一的 AI 网关，自动选择最合适的模型、自动负载均衡、自动记录日志。

### 3. AI 响应必须有视觉反馈

用户在等待 AI 回复时，屏幕上必须有持续变化的内容。禁止让用户面对空白等待。AI 实现时会自动做到：
- 流式输出（打字机效果）
- 阶段提示（"分析中 → 生成中 → 完成"）
- 进度反馈（"正在处理第 3/10 项..."）

### 4. 前端不做业务逻辑

前端只负责"发指令"和"展示结果"。所有业务判断、数据处理都在后端完成。这样保证了：
- 逻辑集中，不会前后端不一致
- 安全性——用户无法绕过前端直接篡改

### 5. 变更必须留痕

每次代码变更都会在 `changelogs/` 目录留下记录碎片。这不需要你操心，AI 会自动完成。

## 七、常见问题

**Q：我不懂技术，能开发 Agent 吗？**
A：可以。你负责想清楚"做什么"和"为谁做"，技术实现交给 AI。你的核心价值是对业务的理解和创意。

**Q：开发一个 Agent 大概要多久？**
A：取决于复杂度。简单的（类似缺陷管理）可能 1-2 天对话就能完成。复杂的（类似视觉创作）需要更多轮迭代。

**Q：我可以中途改需求吗？**
A：可以，随时。进度文件会记录你的当前状态，AI 会帮你评估改动影响。

**Q：多个人可以同时开发不同的 Agent 吗？**
A：可以。每个 Agent 有独立的工作区和分支，互不干扰。

**Q：我的 Agent 上线后谁来维护？**
A：交接清单（`/handoff`）会生成完整的维护指南。后续迭代也可以用同样的方式和 AI 协作。

**Q：遇到问题怎么办？**
A：输入 `/help`，AI 会根据你的当前状态给出建议。如果是技术问题，AI 会自动调用相应的调试技能。
