# CDS 一键可视化部署 · 使用教程

> **类型**：guide（操作指南） · **版本**：v1.0 · **日期**：2026-06-02 · **状态**：现行
> **适用**：在 `cds.miduo.org`（CDS Dashboard）用纯前端方式把一个前后端项目 + 数据库/缓存/消息队列部署起来。
> **关联**：`design.cds-visual-deploy.md`（设计）、`debt.cds-visual-deploy.md`（已知边界）、`guide.cds-tutorial.md`（cds-compose 契约教程，互补）。
> **一句话**：贴仓库 → 检测填好 → 试运行验证 → 部署 → 应用已上线。全程点点点，不写 compose。

---

## 0. 你需要什么

- 一个能 `git clone` 的仓库 URL（公开，或 CDS GitHub App 有访问权限）。
- 登录 CDS Dashboard（`cds.miduo.org`）。
- 不需要：写 cds-compose.yml、准备 .env 文件、手背 Docker 命令。

---

## 步骤 1 · 打开「一键部署项目」

项目列表右上角点「一键部署」下拉 → 「一键部署项目」。弹窗内容超高时会内部滚动，不会飞出屏幕。

## 步骤 2 · 贴仓库 + 让 CDS 检测

在「Git 仓库 URL」粘贴地址（也可点「从 GitHub 选择」）。然后点 **「检测仓库并自动填好配置」**：

- CDS 浅克隆你的仓库并识别技术栈（Node/Python/Go/Rust/Java/PHP/.NET/静态站，monorepo 会逐模块识别）。
- 自动把每个服务的**运行环境 / Docker 镜像 / 启动命令 / 端口**填好。
- 给出**把握度**：`把握 高/中/低` + 识别依据（如 `deps:express`）。把握不高或未完全识别时会提示你先「试运行验证」。

> 没填仓库或检测不出：保留可编辑默认，手填即可；后端服务默认运行时是「自动识别」，克隆后按仓库推断。

## 步骤 3 · 调应用服务（可多个）

「选择应用服务」默认前端 + 后端。每个服务可：

- 改名、选角色（前端 / 后端 / 后台任务）、启用/停用。
- 选运行环境（9 种）；非 auto/dockerfile 时可改镜像/端口/启动命令（命令是最佳努力默认，可直接改）。
- 底部「添加服务」可加任意多个入口（多入口应用）。

## 步骤 4 · 选基础设施（含库名 + 初始化 SQL + 同类型多实例）

「选择基础设施」按类别分组：数据库（MongoDB/PostgreSQL/MySQL/SQL Server/ClickHouse）/ 缓存（Redis/Memcached）/ 消息队列（RabbitMQ/Kafka/NATS）/ 搜索（Elasticsearch）/ 对象存储（MinIO）。

- 选数据库后出现「数据库配置」：填**库名**（默认 app，写进连接串）+ **初始化 SQL**（随项目保存，数据库就绪后在拓扑数据面板「载入初始化 SQL」一键执行）。
- 同类型要多个：点 **「再加一个 PostgreSQL（独立容器 + 独立连接串）」**——首个注入 `DATABASE_URL`，第二个 `DATABASE_URL_2`，各自独立库名。

## 步骤 5 · 粘贴项目环境变量（不必准备文件）

「项目环境变量」直接粘贴 `.env` 文本，一行一条 `KEY=VALUE`，`#` 注释忽略；实时显示「已识别 N 个变量」。CDS 为基建注入的连接串（`DATABASE_URL` 等）会自动合并，这里只填你自己的。

## 步骤 6 · 试运行验证（强烈建议）

任一可编辑服务卡片底部点 **「试运行验证」**：

- CDS 起一个**一次性容器**，在你真实仓库上跑：浅克隆 → 拉镜像 → 跑启动命令 → 端口探活（最多 150s，有结论就提前结束）。
- 结论三档：
  - 绿 **配置可用**：容器常驻 + 端口已响应。
  - 黄 **需确认**：容器没崩但端口暂未响应（可能还在装/端口填错）。
  - 红 **不通过**：给退出码 + 失败日志 + 根因提示（如「/app 没有 package.json——代码可能在子目录」「镜像缺该命令——换镜像」）。
- **不行就照提示改命令/镜像，再点一次**。容器跑完即销毁。

## 步骤 7 · 创建 + 等就绪 + 打开预览

点「创建」。CDS 连续完成 创建 → 克隆 → 识别 → 构建配置 → 部署，并对每个服务做 TCP+HTTP 就绪探测、按依赖顺序编排（应用等基建）。

分支变为 **running** 后，进入分支详情抽屉，顶部会有绿色 **「应用已上线 · https://&lt;分支&gt;.&lt;域名&gt;」** + 「打开预览」按钮，一键访问。

## 常见问题（FAQ）

- **第一次试运行红了？** 看日志 + 提示：多半是命令/镜像与栈不匹配（如改了镜像没改命令），或代码在子目录（命令前加 `cd 子目录 &&`）。改完再试。
- **检测说「未识别」？** 冷门栈/魔改构建检测不到，手填运行环境 + 命令，靠试运行兜底。
- **同类型第二个数据库的连接串？** 是 `<原变量>_2`（如 `DATABASE_URL_2`），指向 `<preset>-2` 容器。
- **初始化 SQL 何时跑？** 当前随项目保存，数据库就绪后进拓扑数据面板「载入初始化 SQL」一键执行（自动随就绪执行为后续项，见 `debt.cds-visual-deploy.md`）。
