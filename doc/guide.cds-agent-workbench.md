# CDS Agent 工作台 · 指南

> **版本**：v1.0 | **日期**：2026-05-14 | **状态**：开发中

## 适用对象

本指南面向普通 MAP 用户。目标是让用户像使用远程 Claude Code 或 Codex Web 一样，在 MAP 里创建一个远程 Agent 会话，让它进入 CDS sandbox 执行任务、展示过程、留下日志，并在需要时停止或恢复。

## 入口

正式入口是左侧导航的 `CDS Agent` 页面。系统配置和授权入口仍在 `设置 -> 基础设施服务`。

不要把 MAP 文档空间当作 CDS Agent 工作台。文档空间只用于阅读说明，不能替代真实的远程执行页面。

## 首次使用

1. 打开 `设置 -> 基础设施服务`。
2. 确认存在一个状态为 `active` 的 CDS 连接。
3. 确认管理员已经配置至少一个模型运行配置，包含 `baseUrl`、`model`、API key 和 runtime。
4. 打开左侧导航 `CDS Agent`。
5. 选择 CDS 连接和模型运行配置。
6. 点击新建会话。
7. 输入任务提示词并发送。

## 会话状态

| 状态 | 含义 | 用户动作 |
|------|------|----------|
| `creating` | MAP 正在向 CDS 创建会话 | 等待或查看错误提示 |
| `running` | 远程 runtime 已启动 | 可以发送消息、查看事件和日志 |
| `idle` | 会话存在但暂未执行 | 可以继续发送消息 |
| `stopping` | 正在释放远程资源 | 等待停止完成 |
| `stopped` | 会话已停止 | 可查看历史，必要时新建会话 |
| `failed` | 创建、启动、执行或停止失败 | 查看错误和日志，按运行手册排查 |

## 对话与事件

用户发送消息后，页面必须展示持续变化的内容。正常情况下会看到：

- assistant 文本逐步出现。
- 工具调用卡片显示工具名称、参数摘要和审批状态。
- Hook 事件显示启动前、启动后、停止前、停止后的执行结果。
- 日志面板显示容器或 worker 输出。

如果页面超过 2 秒没有任何变化，应查看事件时间线和日志面板，而不是只等待。

## 工具审批

默认策略是 `confirm-dangerous`：

- 只读工具可以自动允许。
- shell、write、edit、delete 类危险操作需要人工确认。
- 用户允许或拒绝后，结果会持久化为事件。

审批不是一次性弹窗的临时行为，它属于会话审计记录。刷新页面后仍应能看到审批历史。

## 远程代码巡检与 PR

当任务目标是巡检 `prd_agent` 并提交 PR 时，提示词应包含：

```text
请在远程 CDS sandbox 中巡检 prd_agent 仓库，找到一个小而真实的问题，修改后运行相关测试，创建分支、提交、推送并提交 PR，最后返回 PR 链接、修改文件和测试结果。
```

完成标准不是“模型回复了一段文字”，而是：

- 远程 runtime 确实读取了仓库。
- 产生了代码 diff。
- 运行了相关测试。
- 创建了分支、提交并推送。
- 提交了 PR。
- MAP 页面能看到过程事件、日志和最终 PR 链接。

## 常见失败

| 现象 | 常见原因 | 处理 |
|------|----------|------|
| 显示 401 | CDS 连接被撤销或 token 不匹配 | 回到基础设施服务重新授权 |
| 已连接但无法启动 | 没有模型运行配置或 runtime 不可用 | 联系管理员补配置 |
| 只有 fake 输出 | 真实 runtime 未就绪，系统回退到 fake | 不可作为最终验收，必须切到真实 runtime |
| 工具一直等待 | 危险操作等待人工审批 | 在工具卡片中允许或拒绝 |
| 刷新后丢失输出 | 事件恢复失败 | 记录 sessionId 并按运行手册排查 afterSeq |
