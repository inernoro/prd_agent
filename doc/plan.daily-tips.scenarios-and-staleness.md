# 每日小贴士场景统一与过时检测 · 计划

> **版本**：v2.0 | **日期**：2026-07-17 | **状态**：规划中

## 目标

让新手教程、新功能发布和缺陷修复回执共用同一套小贴士投递、展示、引导和清理能力，同时自动发现失效锚点、死链接和长期无效内容。

现有抽屉、卡片、Spotlight、投递记录和永久 dismiss 已落地。本计划只跟踪仍未完成的场景分类、真实引导动作和 stale 治理。

## 当前待办

### P1：跳转后真正完成引导

- 每条可操作 tip 明确目标路由、目标锚点和完成条件。
- 跳转后滚动并聚焦目标；需要展开面板或切换 tab 时，通过受控动作完成。
- 多步教程保存当前步骤，用户完成最后一步后永久 dismiss。
- 目标不存在时显示失败原因并停止，不只播放光圈假装成功。
- 管理页与 PushDialog 的宽屏留白和信息密度按实际截图校正。

首版只允许有限动作注册表，例如聚焦、切 tab、展开和预填示例。禁止把任意 JavaScript 或 CSS selector 行为从数据库直接执行。

### P2：三类场景统一

| 场景 | 触发 | 目标受众 |
| --- | --- | --- |
| onboarding | 管理员或系统种子 | 新用户或未完成用户 |
| feature-release | 发版流程生成草稿，管理员确认 | 受影响用户 |
| bug-fix | 缺陷关闭后生成定向回执 | 报告人和关注者 |
| manual | 管理员手动创建 | 指定范围 |

管理页支持按场景筛选；场景决定默认受众和文案模板，但不能绕过权限与人工确认。发版和缺陷流程默认只创建待审核草稿，避免自动骚扰。

### P3：过时检测

检测信号：

- `actionUrl` 不在当前路由注册表；
- `targetSelector` 或 tour id 不存在；
- 关联 feature 已停用；
- 内容超过治理周期且长期无展示、点击或完成；
- 目标权限已不存在或受众为空。

扫描结果写稳定原因码和检测时间。系统默认标记 stale，不自动删除；管理员可修复、停用或确认删除。CI 负责静态路由与锚点检查，服务端周期任务负责参与度和功能状态。

## 数据边界

- 场景、动作和 stale 原因使用受控枚举。
- 投递与完成按用户记录，不把一个用户的完成状态影响其他人。
- tip 被编辑后按版本决定是否重新展示，非破坏性文案调整不打扰已完成人员。
- 缺陷和发版产生的草稿不复制敏感正文、内部链接或无权限内容。

## 验收门

- 四类场景可筛选、创建和定向投递。
- 跳转后的引导能完成至少一个真实页面动作，并能识别目标缺失。
- 最后一步完成后跨会话不再出现。
- 路由、锚点和低参与度扫描能生成可解释 stale 列表。
- stale 内容不会继续投递，删除仍需人工确认。
- 管理页在宽屏与移动端均可查看、修复和停用内容。

## 风险

| 风险 | 控制 |
| --- | --- |
| 数据库动作变成脚本注入 | 只执行前端注册的动作 key |
| 发版或缺陷通知轰炸 | 默认草稿、去重和频率限制 |
| DOM 重构导致锚点失效 | CI 静态扫描与运行时失败上报 |
| 旧用户被反复打扰 | 用户完成记录加内容版本语义 |
