## 文档命名规则（doc/）

### 适用范围
- `doc/` 目录下除 1~5 号文档外的所有文档

### 编号规则
1) 已有编号的文档：**序号不变**  
2) 无编号文档：**按现有顺序追加编号**

### 命名规则
1) **Agent 设计类文档**  
   - 命名：`{agent-name}-design.md`  
   - 示例：`visual-agent-design.md`

2) **基础设施类文档**  
   - 命名：`{appname}-design.md`  
   - 示例：`model-pool-design.md`

3) **原则/规范类文档**  
   - 命名：`{topic}-rule.md`  
   - 示例：`app-feature-naming-rule.md`

### appname 唯一性约束
`appname` 需要在以下三处保持一致且唯一：
- 前端路由
- `/mds` 接口
- `/api/mds` 文档

示例：  
`/mds` 对应文档命名为 `mds-design.md`

### 权限规则补充
权限相关文档包含四个 key 的合并逻辑（四 key 合并），应在命名与内容中明确体现。
