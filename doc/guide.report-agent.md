# 周报管理 Agent 使用 · 指南

> **类型**：guide | **适用角色**：团队负责人、项目经理、全体成员 | **最后更新**：2026-03-19
> **预计阅读**：8 分钟 | **难度**：入门

---

## 1. 功能概览

| 能力 | 说明 |
|------|------|
| 日志记录 | 每日记录工作内容，自动汇总到周报 |
| AI 生成 | 分析日志和 Git 提交，一键生成结构化周报 |
| 团队汇总 | 负责人一键汇总所有成员周报 |
| 评论协作 | 团队成员在周报上异步沟通 |
| 数据源集成 | 接入 Git 仓库等数据源自动采集工作记录 |

---

## 2. 操作路径

**入口**：首页 → 点击「周报管理员」卡片 → 进入周报管理页面

页面包含三个 Tab（**「团队」Tab 仅在有团队时显示**）：

| Tab | 图标 | 内容 |
|-----|------|------|
| **「周报」** | FileText | 我的周报列表、日志记录、趋势统计 |
| **「团队」** | Users | 团队看板（我管理的/我加入的） |
| **「设置」** | Settings | 数据源配置、模板管理、团队管理 |

---

## 3. 完整操作流程

### 步骤 1：创建团队

1. 切换到 **「设置」** Tab
2. 在「团队管理」区域创建团队
3. 邀请团队成员并设置角色（负责人 / 成员）

### 步骤 2：配置数据源

1. 在 **「设置」** Tab 的「数据源」区域
2. 添加 Git 仓库地址（支持 GitHub/GitLab 等）
3. 配置后 AI 生成周报时会自动拉取本周 commit 记录

### 步骤 3：每日记录

1. 切换到 **「周报」** Tab
2. 在日志区域记录每天的工作内容
3. 内容会自动保存，周报生成时 AI 自动汇总这些记录

> **Pro Tip**：养成每天花 1 分钟记日志的习惯。哪怕只写一句话「修复了登录页面的样式问题」，AI 汇总时也能据此展开。

### 步骤 4：生成周报

1. 在 **「周报」** Tab 点击创建新周报
2. AI 自动从数据源拉取本周记录：
   - 每日日志内容
   - Git commit 记录（如已配置）
3. 点击 **AI 生成**，自动汇总为结构化周报：
   - **本周完成** — 已完成的任务列表
   - **进行中** — 未完成的进行中任务
   - **下周计划** — 基于当前进度的计划
   - **风险与阻塞** — 需要关注的问题
4. 审核内容，必要时手动修改
5. 提交

### 步骤 5：团队汇总（负责人）

1. 切换到 **「团队」** Tab
2. 查看团队面板，区分「我管理的」和「我加入的」团队
3. 查看所有成员的周报提交状态
4. 一键生成团队汇总报告

---

## 4. 模板管理

在 **「设置」** → **模板管理** 中：
- 创建周报模板，定义每周必须包含的栏目
- 团队统一使用同一模板，保证格式一致
- 模板支持自定义字段（如「本周学习」「客户反馈」）

---

## 5. 常见问题

**Q：数据源支持哪些平台？**
A：目前支持 Git 仓库提交记录。后续将支持更多项目管理工具。

**Q：AI 生成的周报可以修改吗？**
A：可以。AI 生成的是初稿，你可以自由修改后再提交。

**Q：团队成员能看到我的日志吗？**
A：日志默认仅自己可见。只有提交后的周报才对团队可见。

**Q：周报可以导出吗？**
A：支持导出为 Markdown 格式。
