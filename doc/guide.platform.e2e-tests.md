# Playwright E2E 测试 · 指南

> **版本**：v2.0 | **日期**：2026-07-17 | **状态**：已落地

浏览器 E2E 用于验证真实页面渲染、导航和交互，是单元测试、接口测试和 HTTP 冒烟之外的用户路径证据。

## 1. 适用场景

- 新增或修改页面、导航、弹窗和关键交互。
- 修复只在浏览器发生的状态、样式或脚本问题。
- 验证登录后的端到端业务闭环。
- 对预览域名做发布前回归。

纯函数和服务端契约优先使用更低层测试，不把所有逻辑堆进 E2E。

## 2. 安装

```bash
cd e2e
pnpm install
pnpm install-browsers
```

仓库前端统一使用 pnpm。测试目录和现有 fixture 是写新用例前的事实源。

## 3. 执行

```bash
cd e2e
E2E_BASE_URL=https://preview.example.com pnpm test
pnpm exec playwright show-report
```

需要观察浏览器时使用 `pnpm test:headed`，调试单个用例时使用文件路径或 `-g` 过滤。预览地址必须指向本次分支的最终页面，而不是无关根域名。

## 4. 用例结构

每条用例应：

1. 从用户可到达的入口开始。
2. 用角色、标签或稳定测试标识定位元素。
3. 执行一个清晰业务场景。
4. 断言用户可感知结果和必要的服务端状态。
5. 在失败时保留截图、trace 或日志。

不要依赖任意等待时间、脆弱 CSS 层级或生产数据顺序。

## 5. 登录与数据

- 复用 `e2e/utils/` 中的认证 fixture。
- 凭据通过环境变量传入，不能写进测试或文档。
- 测试数据应可识别、可清理且不与真实用户混淆。
- 并行用例不得共享可变实体，必要时使用唯一后缀。

## 6. 视觉与主题

涉及 UI 的用例至少检查目标视口，视觉验收按项目验收技能覆盖浅色和深色主题。截图是证据，不替代对按钮、状态和数据的语义断言。

## 7. 稳定性规则

- 等待明确事件或断言，不使用固定 sleep。
- 对 SSE 和长任务等待阶段或终态。
- 用例失败必须可重复，偶发失败先定位竞态。
- 测试结束清理自身创建的数据。
- 不通过重试次数掩盖真实缺陷。

## 8. 交付证据

交付时说明测试命令、目标 URL、场景和断言。若因凭据或外部服务无法执行，列出已尝试路径和具体阻塞，不声称行为已验证。
