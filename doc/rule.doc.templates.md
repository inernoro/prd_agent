# 七类文档内容模板 · 规则

> **版本**：v4.0 | **日期**：2026-07-17 | **状态**：已落地

## 适用范围

`doc/` 下所有 Markdown 必须先选择一种职责，再按对应模板组织。文件名、状态枚举和索引要求见 `rule.doc.naming.md`；存量维护和回收规则见 `rule.doc.maintenance.md`。

## 通用要求

每份文档必须以 H1 和单行元数据开头：

```markdown
# 清晰标题 · 类型

> **版本**：v1.0 | **日期**：YYYY-MM-DD | **状态**：草案
```

所有类型共同遵守：

1. 前两段说明目标、适用对象和当前结论。
2. 一份文件只承担一种职责；完成计划不继续充当设计或报告。
3. 代码只用于说明无法用契约表表达的关键点，全文不超过 30%。
4. 命令、实现清单和一次性日志不复制进设计；引用真实文件或脚本。
5. 不写临时分支、短期预览地址、上下文交接话术或“下一个 Agent”日记。
6. 新增、删除或改名后同步 `index.yml` 与 `guide.list.directory.md`。

## spec.*：定义做什么

必需结构：

- 目标与用户
- 范围与非目标
- 用户场景或业务流程
- 数据与接口契约
- 验收标准
- 兼容与限制

不得写实施排期、逐文件改法或已完成日志。

## design.*：定义怎么做

必需结构：

- 目标与当前事实
- 核心决策
- 架构与数据流
- 关键契约
- 风险、回滚与验收
- 实现来源

设计文档不得复制完整类、组件、脚本或配置文件。已经退场的方案直接删除；有复盘价值的决策差异压缩到“替代方案”一节。

## plan.*：只跟踪未完成动作

必需结构：

- 目标
- 当前事实
- 未完成阶段
- 完成标准
- 风险与依赖

完成项只用于说明必要前置，不维护长勾选清单。全部完成后删除计划，并把长期事实更新到 design/spec/debt。

## rule.*：定义禁止与约束

必需结构：

- 适用范围
- 强制规则
- 允许的例外
- 违规示例的简短说明
- 自动校验或审查入口

规则只保留最小示例，不复制教程。能够由 lint、测试或脚本执行的约束必须给出事实源。

## guide.*：告诉用户怎样操作

必需结构：

- 适用场景与前置条件
- 最短成功路径
- 结果判定
- 常见失败与恢复
- 相关事实源

命令优先引用仓库脚本；必须展示时只写单条入口和关键参数，不内嵌整套脚本。

## report.*：记录已经发生的结果

`doc/` 中新增报告只允许 `report.YYYY-WNN.md` 周报。验收截图、测试证据和专项报告进入验收知识库，不提交 `doc/`。

存量非周报报告只有在被代码、脚本或测试作为契约读取时暂时保留，并在索引中标注 grandfather；取消引用后立即回收。

## debt.*：记录未偿还边界

必需结构：

- 当前影响
- 债务清单及状态
- 触发条件或风险
- 偿还条件
- 关联实现

已经偿还的详细过程移到 Git 历史或周报；台账只保留简短结论，避免 paid 项淹没 open 项。

## 类型转换规则

| 当前内容 | 正确去向 |
| --- | --- |
| 计划全部完成 | 删除 plan，事实进入 design/spec |
| 设计只剩旧方案 | 删除 design，必要差异写入现行设计 |
| 报告提出长期风险 | 风险进入 debt，报告进入知识库或回收 |
| 指南包含完整实现 | 实现回代码或脚本，指南只保留操作入口 |
| 规则包含长教程 | 教程拆到 guide，规则保留约束 |
| 债务已经偿还 | 删除明细或只留一行 paid 结论 |

## 审查清单

- 文件名前缀、H1 类型和内容职责一致。
- 元数据状态来自规定枚举，日期真实有效。
- 文档内没有大段实现代码、临时日志、失效链接或重复事实源。
- `index.yml` 标题与 H1 一致，人工目录只列当前文件。
- 删除文件后，全仓不存在非历史悬挂引用。
