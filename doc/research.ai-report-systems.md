# AI 周报系统市场调研报告

> **调研日期**：2026-03-08
> **目的**：为 report-agent 模块升级获取灵感，了解市面主流 AI 周报/状态更新工具的功能与创新点

---

## 一、调研总览

调研覆盖 **20+ 款产品**，横跨国际市场与中国市场，涵盖：
- 专用 AI 周报工具（Steady、Geekbot、Spinach、DailyBot、Stepsize）
- 综合平台内置周报能力（ClickUp Brain、Notion AI、Lattice、Linear Pulse）
- 中国市场方案（飞书 AI 周报、钉钉、Dify Agent、Monica）
- 开源方案（Parabol、CrewAI、Geekbot MCP Server）

---

## 二、产品详细分析

### Tier 1：专用 AI 周报/状态更新工具

#### 1. Steady（原 Status Hero）

| 维度 | 详情 |
|------|------|

#### 2. Geekbot

| 维度 | 详情 |

## 四、2025-2026 行业创新趋势

### 趋势 1：从"填表"到"一键 AI 草稿"
最大的转变是消除手动写报告。Steady 的 AI Quick Fill、飞书 AI 周报、ClickUp Brain 都从工具活动数据草拟报告，只需人工审阅和轻微编辑。

### 趋势 2：MCP / AI 模型集成
Geekbot 的 MCP Server 是前沿创新 — 通过标准化协议将 standup 数据暴露给外部 AI 模型（Claude、GPT），团队可构建自定义 AI 工作流。

### 趋势 3：会议到报告的 Pipeline
Spinach AI 从实时会议音频中提取结构化报告，完全消除异步输入步骤。

### 趋势 4：管理者侧 AI 汇总
飞书"AI 智能汇报"和 Steady Echoes 解决了管理者瓶颈 — 不再逐份阅读 20 份个人周报，AI 综合为一份可行动的摘要。

### 趋势 5：自然语言查询历史数据
Geekbot 对话式分析和 Lattice AI Agent 支持用自然语言查询过去的绩效数据，将静态报告变成交互式知识库。

### 趋势 6：从描述到预测
Stepsize 的风险预警代表着从"发生了什么"（描述性）到"可能出什么问题"（预测性）的报告进化。

### 趋势 7：多 Agent 架构
Notion Custom Agents、CrewAI 的方式 — 启动协调的 Agent 群组，跨工具采集数据并组装报告，是最前沿的架构模式。

### 趋势 8：音频消费形态
Linear Pulse 的音频摘要是全新的 UX — 适合通勤/运动时消化周报信息。

### 趋势 9：多模型支持
ClickUp Brain 提供 GPT/Claude/Gemini 选择；Dify 支持可配模型。用户越来越期待模型灵活性。

## 五、对 report-agent 的升级启示

基于调研，以下是当前 report-agent 可借鉴的高优先级改进方向：

### 🔴 高价值 — 差异化明显

| 方向 | 参考产品 | 说明 |
|------|----------|------|
| **多源数据聚合** | Steady、飞书、ClickUp | 不只拉 Git，还要拉任务管理（工作流/缺陷）、会话记录、文档变更 |
| **管理者 AI 汇总** | 飞书 AI 智能汇报 | 管理者收到所有成员周报后，AI 一键生成团队汇总+洞察，这是解决管理者痛点的杀手功能 |
| **自然语言查询历史** | Geekbot、Lattice | "上个月 @张三 做了多少 bug 修复？"— 将周报数据变为可查询的知识库 |
| **风险/阻塞器自动检测** | Stepsize、DailyBot | AI 从提交数据、任务延期中主动识别风险，而不是等人报告 |

### 🟡 中等价值 — 体验提升

| 方向 | 参考产品 | 说明 |
|------|----------|------|
| **定时自动执行** | Notion Agents、飞书 | Agent 按计划（如每周五下午）自动生成并推送，零触碰 |
| **推送到 IM** | DailyBot、Geekbot | 周报推送到飞书/钉钉/企微/Slack，而不只是在 Web 端看 |
| **情感/士气追踪** | Geekbot、Lattice | 从周报文本中分析团队情绪趋势，早期预警倦怠 |
| **音频摘要** | Linear Pulse | 周报可以"听"，适合移动端/通勤场景 |

### 🟢 长期方向 — 架构演进

| 方向 | 参考产品 | 说明 |
|------|----------|------|
| **MCP 协议开放** | Geekbot MCP | 将周报数据通过 MCP 暴露给外部 AI 模型，支持自定义查询 |
| **多 Agent 协作** | CrewAI、Notion | 采集 Agent + 分析 Agent + 分发 Agent 的协作架构 |
| **绩效系统联动** | Lattice | 周报数据连接 OKR/KPI 追踪，成为绩效评审的数据依据 |
| **多模型可选** | ClickUp Brain | 用户可选择不同 AI 模型生成周报（GPT/Claude/DeepSeek） |

## 七、总结

当前 report-agent 已具备 git 数据采集、团队结构、模板、AI 摘要等基础能力。与市场领先产品相比，最大的差距在于：

1. **数据源单一** — 主要依赖 Git，缺少任务管理/会话/文档等多源聚合
2. **缺少管理者视角** — 没有"AI 汇总所有人周报"的能力（飞书最强的功能）
3. **缺少主动推送** — 需要用户手动查看，没有 IM/邮件推送
4. **缺少交互式查询** — 不能用自然语言查询历史周报数据
5. **缺少预测能力** — 不能从数据中主动识别风险和阻塞器

建议优先级：**管理者 AI 汇总 > 多源数据聚合 > 定时推送 > 自然语言查询 > 风险检测**
