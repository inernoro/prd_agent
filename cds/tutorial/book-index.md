# CDS 权威教程

这本书带你从“第一次打开 CDS”走到“多项目隔离、自动预览、正式发布、灰度验收、故障恢复和集群运维”。它既面向第一次接触云开发环境的产品与开发人员，也面向需要维护生产发布链路的平台工程师。

## 你最终会得到什么

学完整本书后，你应当能独立完成以下事情：

1. 为一个 Git 项目建立 CDS 项目身份，连接 GitHub，并让 push 自动生成可访问的预览环境。
2. 看懂项目、分支、BuildProfile、基础设施、环境变量、RemoteHost 和发布目标之间的边界。
3. 在没有项目发布脚本时，让 CDS 根据 Compose 或静态构建事实生成可审计的正式发布脚本。
4. 用完整 commit、发布计划、脚本 SHA256、最终公网 HTML 与入口资源证明“这个版本真的发布成功”。
5. 在配置串项目、页面版本混乱、入口 500/502、构建失败、网络失联或节点迁移时找到第一原因并恢复。

## 阅读路线

### 基础篇：从项目到第一条预览

第 0 至 8 章适合所有人。你会认识 CDS 的核心对象，创建项目，连接 GitHub，理解多项目身份和环境变量边界，并完成第一条预览。

### 进阶篇：从预览到可验收发布

第 9 至 15 章面向项目负责人和交付人员。你会选择交付模式，理解三条部署路径，接入数据服务，设计冒烟测试，并建立真正的正式发布目标。

### 高级篇：生产安全与平台运维

第 16 至 24 章面向平台维护者。你会使用生产发布安全合同，完成集群和主机迁移，维护转发器、蓝绿切换、Agent SDK 与平台运行手册。

## 全书共同规则

- 项目身份先于部署。每个分支、环境变量、发布目标和运行记录都必须能回到唯一 `projectId`。
- `_global` 默认只属于 CDS 控制面。项目只有显式开启 `inheritGlobalEnv` 才继承，业务数据库、队列和第三方凭据不得默认跨项目传播。
- 预览成功不等于正式发布成功。正式发布必须锚定完整 commit，并验证最终公网 HTML、HTML 实际引用的 JS/CSS、API 和专项服务。
- 容器 `running`、Docker healthcheck 或内部 curl 只能作为诊断证据，不能替代用户最终入口。
- AI 修改发布目标必须使用项目级 Agent Key。全局 AI key 不能写入项目发布配置。
- 错误目标先归档再处理。归档保留操作者、时间、原因和历史运行证据，不把事故现场直接删除。
- 没有发布脚本不是阻塞。CDS 可以动态生成 Compose 或静态站发布脚本，但不会假装支持尚未实现的 Kubernetes、GitOps 或镜像仓库执行器。

## 连续示例

全书使用“示例商城”作为项目名，Git 仓库身份为 `acme/example-shop`，默认分支为 `main`。基础篇先产生分支预览；进阶篇再把同一个完整 commit 发布到 `shop.example.com`。实际操作时必须替换为自己的项目、仓库、服务器和域名，不要照抄示例凭据或路径。
